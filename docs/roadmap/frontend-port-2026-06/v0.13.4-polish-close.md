# v0.13.4 — Dream overlay + UX extras + initiative close

> **Status: PLANNED.** Initiative: Frontend Port — the Body (Order 6, version 6/6).
> Risk: **Low** (presentation polish on shipped seams). Depends: v0.13.0–v0.13.3. Flag: none
> (each extra is small + independently removable; nothing risky enough to gate).

## Goal

Turn a working avatar+voice chat into a *delightful* one, and close Initiative 6. This version adds the
dream overlay, the UX extras (Alan's "想想其他可以明显提高用户体验的设计"), the accessibility +
responsive pass, and the full end-to-end verification, then marks the initiative ✅.

## What ships

**Dream overlay (the 入梦 experience):**
- A soft full-screen dreamy overlay on `dream.status is_dreaming` (matching the yumi palette: moon 🌙,
  drifting stars/bubbles, gentle blur), the model posed sleeping (v0.13.1 `setState('sleeping')`),
  input disabled with "Luna 在做梦…" + a ☀️ **唤醒** button (sends `dream.wake`). Honor a
  **min-overlay-duration** so a fast dream cycle doesn't flash (Python's v0.55.3 lesson). Optionally
  surface `dream.step` as a tiny "她梦见了…" caption.

**UX extras (each small, each clearly improves the experience):**
1. **Thinking indicator** — between `turn.started` and the first token, a cute typing-dots bubble on
   Luna's side ("Luna 在想…"); the model is already in `thinking` state (v0.13.1).
2. **Proactive distinction** — when Luna speaks on her own (proactive/continuation bubbles, LD #15),
   a gentle glow + a "🌱 Luna 主动找你" tag so unsolicited messages read differently from replies.
   If she acts but stays silent, a subtle ghost note ("Luna 悄悄做了点什么").
3. **Mood pip** — a small indicator near the model showing her current affect (the last message's
   `expression`/`emotion`), cute icon + tint.
4. **Scroll affordance** — auto-scroll on new messages **only when already at the bottom**; if scrolled
   up, show a "↓ 新消息" pill instead of yanking the view.
5. **Settings popover (the kill-switches as UI)** — a small ⚙️ with toggles: voice on/off
   (`LUNA_TTS`/gain mute), proactive on/off, reduced motion. (Proactive/voice toggles are client-side
   prefs reflecting the existing env switches; wire to server only if trivial — otherwise client-only +
   noted.)
6. **Connection resilience UI** — the status pip animates reconnecting; a gentle toast on
   disconnect/reconnect (the `wsClient` already auto-reconnects).

**Accessibility + responsive:**
- Honor `prefers-reduced-motion`: freeze the lace shimmer, the dream drift, the typing dots.
- Responsive: the v0.13.0 stack-on-narrow holds with the live model (model scales / docks on top).
- Readable contrast for bubbles/timestamps over the striped background.

**Modified files:** mostly `packages/web/src/ui/*` (overlay, indicators, settings) + small `app.ts`
wiring (subscribe to `dream.status`/`dream.step`, proactive bubble tagging via the controller's
existing event routing). No protocol/controller shape changes expected — if a proactive bubble needs a
distinguishing flag the controller already has the event type; surface it through `BubbleView` if not
already (additive, verify against [`controller.ts`](../../../packages/web/src/controller.ts)).

## Tests

- Dream overlay: `dream.status is_dreaming` → overlay shown + input disabled; `dream.wake` path +
  min-duration (inject the clock) → overlay hides after the floor, not before.
- Thinking indicator: appears on `turn.started`, removed on first delta / finalize.
- Proactive tagging: a proactive/continuation bubble gets the distinct class/tag; a normal reply does
  not.
- Scroll: at-bottom → auto-scrolls; scrolled-up + new message → pill appears, view not moved.
- Reduced-motion: with the media query forced, animation classes are not applied.
- Full controller-integration replay: a scripted session (user turn → tool → reply → proactive →
  dream.enter → dream.step → dream.status → wake) renders the right bubbles, cards, overlay, and tags
  end-to-end.

## What this version explicitly does NOT include

- No new backend capability — this is the frontend close. Eval framework, event-driven proactive
  triggers, self-improvement = **separate future initiatives** (master roadmap).
- No TTS caching, no multi-session UI, no theme switcher beyond reduced-motion.
