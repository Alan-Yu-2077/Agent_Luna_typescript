## Goal

Replace the bare dev host (`app.ts` + `index.html`) with Alan's redesigned, cute, Japanese-leaning
chat UI вАФ the whole visible page **except** the live avatar and voice (those are v0.13.1/v0.13.2).
After this version the app is genuinely usable and pretty: you chat with Luna, see her replies stream
into right-side bubbles with timestamps, watch her tool actions as cute cards, and press рЯМЩ еЕ•жҐ¶. The
Live2D area is a styled placeholder (static yumi image or a cute silhouette). Crucially this is **pure
presentation**: it consumes the v0.12.0 `controller` unchanged, implementing a richer `BubbleView` and
leaving `live2d`/`audio` as the existing `consoleLive2DSink`/`noopAudioSink` stubs.

## What this version explicitly does NOT include

- **No live Live2D.** The model stage is a static placeholder. Real pixi-live2d/Cubism/yumi rendering,
  expression/emotion driving, and dragging вЖТ **v0.13.1**.
- **No audio.** `audio` stays `noopAudioSink`; no GPT-SoVITS, no lip-sync вЖТ **v0.13.2**.
- **No dream overlay animation, no proactive-glow, no thinking indicator, no settings panel,
  no reduced-motion pass.** The еЕ•жҐ¶ button sends `dream.enter` and the status reflects `dream.status`
  textually, but the full dreamy overlay + UX extras вЖТ **v0.13.3**.

## Risk

Low вАФ pure front-end on a frozen contract. Watch points: (1) CSS lace/stripe must stay readable
(contrast) and not tank paint perf вАФ keep frills as cheap pseudo-elements/SVG, not huge images;
(2) don't smuggle event interpretation into the UI вАФ every server-state decision stays in the
controller, the UI only renders what `BubbleView` is told (the v0.12.0 boundary). Mitigation:
the controller-integration test above guards the boundary.
