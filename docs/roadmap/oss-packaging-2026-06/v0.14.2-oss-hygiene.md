# v0.14.2 — OSS hygiene + Initiative 7 close

> **Status: PLANNED.** Initiative: Open-source packaging (Order 7, version 3/3). Risk: **Low**
> (docs + config + scanning). Depends: v0.14.1. Flag: none.

## Goal

Bring the repo to open-source-ready and close Initiative 7: a license, an honest README that matches
reality (it currently says "scaffolding only … No runtime code yet"), a complete env example, a
third-party-license inventory, and a secret-scan gate — so a stranger can land on the repo, understand
what it is, run it with one command, and trust there are no leaked keys.

## What ships
- **`LICENSE`** — MIT (the app code). Author/year header.
- **`README.md` rewrite** — replace the stale scaffolding text with: what Luna is (agent brain +
  memory + dream + proactivity + Live2D body + voice), a **Quickstart** that states the **Docker +
  ~5 GB voice-model prerequisite up front**, the exact `bun install` → `bun run setup` → `bun run dev`
  flow, an architecture overview (monorepo: protocol / server / web + `services/tts`), the env/config
  surface, and an honest "voice = yumi via GPT-SoVITS; heavy by design" note. Link DEVELOPMENT.md +
  the roadmap.
- **`.env.example` completion** — add `LUNA_TTS_ENGINE=gpt-sovits`, `GPT_SOVITS_URL`,
  `HF_ENDPOINT` (mirror), and a comment that the web-side toggles (`luna:tts` / `luna:live2d` /
  `luna:reduce-motion`) are browser `localStorage`, not env. Keep all values as placeholders only.
- **`THIRD_PARTY_LICENSES`** — enumerate GPT-SoVITS (MIT, code + weights), pixi.js (MIT),
  pixi-live2d-display (MIT), the Cubism Core (Live2D Proprietary — note redistribution terms), the
  **yumi model assets** (artist terms — must be stated), and any voice-dataset terms. State that
  model-weight licenses beyond the permissive defaults are the user's responsibility.
- **Secret scan** — a `gitleaks` config + a CI step (or a documented pre-commit) + a one-time scan of
  the working tree **and git history**; confirm no `ANTHROPIC_API_KEY`/keys ever committed.
- **`.gitignore`** — confirm `models/` + `.env` + sqlite are ignored; remove any stray tracked junk
  (e.g. `.DS_Store`).

## Tests
- `gitleaks detect` runs clean on the tree (and history scan reported).
- README internal links resolve (a link-check); the quickstart commands match the real `package.json`
  scripts (`setup`/`dev`/`start` exist).
- `tsc --noEmit` (web + server) + `bun test` still green (no regressions from config/docs).

## What this version explicitly does NOT include
No new runtime behavior. No CI beyond the secret-scan gate (a fuller CI pipeline — test/lint/build on
push — is a reasonable **future initiative**, noted, not built here).

## Risk
**Low.** Main watch: the README must not overpromise — the Docker/5 GB prerequisite is stated in the
first screen, not buried. The license inventory must be accurate for the **non-MIT** pieces (Cubism
Core proprietary; yumi art terms) — get those exactly right rather than blanket-"MIT".
