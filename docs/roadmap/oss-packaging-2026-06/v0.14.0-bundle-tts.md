# v0.14.0 — Bundle GPT-SoVITS as a co-located service + dev-server direct integration

> **Status: PLANNED.** Initiative: Open-source packaging (Order 7, version 1/3). Risk: **High**
> (external ML service; not live-verifiable in this env). Depends: Initiative 6 (the AudioSink +
> dev-server proxy). Flag: `LUNA_TTS_ENGINE` (defaults `gpt-sovits`); sidecar off → silent (existing
> graceful degrade).

## Goal

Move yumi's voice from "an external thing on Alan's machine" to "a service that lives in this repo and
the TS dev-server talks to directly." This version bundles GPT-SoVITS as `services/tts/gpt-sovits/`
(Docker-first) and rewrites the dev-server's `/api/gpt-sovits/*` handling to translate the app's
request into GPT-SoVITS `api_v2`, so we no longer depend on the Python project's Node proxy. It does
NOT yet add the one-command launcher (v0.14.1) — you still start things separately — but after it, a
running GPT-SoVITS container + the dev-server produce yumi's voice end-to-end.

## What ships

**New `services/tts/gpt-sovits/`:**
- `docker-compose.yml` — runs the community GPT-SoVITS image (`xxxxrt666/gpt-sovits` or pinned),
  exposes `api_v2` (`:9880`), mounts a `models/` volume (gitignored) for weights + yumi's voice model,
  and a `voice/` mount for the reference audio. CPU profile by default; GPU/MPS as documented overrides.
- `voice.json` — yumi's voice config consumed by the dev-server translation: `{ ref_audio_path,
  prompt_text, prompt_lang, text_lang, model_id }` (the `neuro-v2-e24` settings, ported from Python
  `TTS/browser/config.js` + the gpt-sovits service defaults).
- `README.md` — how the sidecar works, the weights/voice-model fetch, native-install fallback.

**Modified `packages/web/dev-server.ts`:**
- Replace the raw forward (`/api/gpt-sovits/* → LUNA_TTS_PROXY`, `dev-server.ts:19-27`) with a
  **translation handler**: on `POST /api/gpt-sovits/speak`, read `{text, voice}`
  ([`ttsClient.ts:14`](../../../packages/web/src/audio/ttsClient.ts)), merge `voice.json`, POST the
  `api_v2` body `{text, text_lang, ref_audio_path, prompt_text, prompt_lang, media_type:'wav',
  streaming_mode}` to `GPT_SOVITS_URL` (`/tts`), and stream the WAV back. 502 (→ silent) if the
  sidecar is unreachable. Keep `ttsClient` unchanged.
- Pull the translation into a small pure module (e.g. `services/tts/gpt-sovits/translate.ts` or
  `packages/web/src/audio/sovitsRequest.ts`) so it's unit-testable without Docker.

**Config (`.env.example`, fully wired in v0.14.2):** `LUNA_TTS_ENGINE=gpt-sovits`,
`GPT_SOVITS_URL=http://localhost:9880`, `HF_ENDPOINT=https://hf-mirror.com` (optional mirror).

**`.gitignore`:** add `models/` (weights land there at runtime).

## Tests
- `sovitsRequest` (pure): `{text:'你好', voice}` + `voice.json` → the exact `api_v2` body (text_lang,
  ref_audio_path, prompt_text…); empty/whitespace text → no request; unknown voice → defaults.
- dev-server handler: with a stubbed upstream `fetch`, `/api/gpt-sovits/speak` returns the upstream
  WAV bytes on 200 and 502 when the upstream throws (mirrors the `ttsClient` test pattern).
- `docker compose -f services/tts/gpt-sovits/docker-compose.yml config` parses (CI-runnable without
  pulling the image).
- **Owner-verified (needs Docker + ~5 GB):** `docker compose up` the sidecar + `dev:web` → send a
  message → hear yumi.

## What this version explicitly does NOT include
The one-command launcher (`bun run dev` starting all three) → **v0.14.1**. LICENSE / README rewrite /
`THIRD_PARTY_LICENSES` / secret-scan → **v0.14.2**. No lightweight TTS engine (locked: GPT-SoVITS only).

## Risk
**High + partially unverifiable here.** Mitigations: (1) all logic that CAN be tested without Docker
(request translation, handler error path, compose validity) is unit/CI-tested; (2) the live path is
owner-verified + documented; (3) graceful degrade (sidecar down → silent) means a misconfigured
sidecar never breaks chat; (4) the exact `api_v2` body shape is grounded in the Python
`gpt-sovits-service.js` request (cite at build time). Open: yumi voice-model hosting (see initiative
README OQ #1) — until resolved, `voice.json` + the fetch step can't be finalized.

## Acceptance criteria
- [ ] `services/tts/gpt-sovits/` exists with a valid `docker-compose.yml` + `voice.json` + README.
- [ ] dev-server translates `{text,voice}` → `api_v2` and streams WAV; `ttsClient` untouched.
- [ ] The Python Node proxy is no longer required (translation is in-repo).
- [ ] `models/` gitignored; no weights committed.
- [ ] Unit tests green; `docker compose config` valid; owner confirms live yumi voice with Docker.
