# v0.4.0 — Memory substrate foundation (L1 persistence + L2 full-text timeline)

> **Status: SHIPPED 2026-06-12.** Initiative: memory-dream-substrate (Order 2, version 1/5). Risk: **Medium**
> (first persistence of live state; swaps a shipped in-memory path). Depends: v0.3.6. Flag:
> `LUNA_PERSIST` (default on; `=0` falls back to in-memory for tests / ephemeral runs).

## Goal

Stop Luna forgetting on restart. Make `Session` SQLite-backed (L1 = the live thread, persisted as
Anthropic `MessageParam[]`), and add an append-only **L2 full-text timeline** table that records
every turn verbatim — the ground-truth archive everything else derives from. This collapses Python
v0.52 (single-writer) + v0.53 (full-text archive) into one TS version: SQLite WAL gives single-writer
for free, so there is no lock machinery to build.

## What ships

### `packages/protocol/src/memory.ts` (new)
Zod schemas: `L2Turn { id, session_id, turn_id, t_ms, user_text, assistant_text, raw_json }`
(`raw_json` = the full Anthropic content blocks, so signed thinking + tool_use survive a reload —
same fidelity discipline as v0.3's verbatim history). `SessionRow { id, turn_seq, updated_ms }`.

### `packages/server/src/memory/` (new)
- `db.ts` — opens the memory DB via `sql.ts:openDb(LUNA_DB_PATH)` (same file the trace store uses;
  one `Database` instance shared process-wide), runs `migrate`.
- `migrations/0002_memory.sql` — `CREATE TABLE sessions(...)`, `CREATE TABLE l2_turns(...)` +
  index `(session_id, t_ms)`. DDL only, no PRAGMA (per `trace/README.md` discipline).
- `sessionStore.ts` — `loadSession(id)` / `saveSession(session)` / `appendL2(turn)`. Reuses the
  shipped `sql.ts` connection.

### Modified
- `packages/server/src/turn/session.ts` — `getSession(id)` hydrates from SQLite via `loadSession`
  on first touch (falls back to a fresh in-memory `Session` when `LUNA_PERSIST=0`); a new
  `persistSession(session)` writes back. The `{ id, history, turnSeq, activeTurn, mutex }` shape is
  unchanged — `history` and `turnSeq` gain durability.
- `packages/server/src/turn/runTurn.ts` — in `append_results` / `finalize`, after history grows,
  call `persistSession` + `appendL2` for the completed turn. Writes ride the existing
  `session-serial` mutex; with WAL this is the single-writer guarantee.
- `packages/server/src/main.ts` — the memory DB is the same `openDb` already opened for traces;
  `migrate` now applies both `trace/migrations` and `memory/migrations` (or a unified dir).

## Tests
1. Append a turn, `persistSession`, reopen the DB, `loadSession('default')` → history + turnSeq
   identical (including a turn that carried a `thinking` + `tool_use` block — `raw_json` round-trips).
2. L2 timeline: 3 turns → 3 `l2_turns` rows in `t_ms` order with full `user_text`/`assistant_text`.
3. `LUNA_PERSIST=0` → no DB writes; `getSession` returns a fresh in-memory session each boot.
4. Migration idempotency (reuses `sql.test.ts` pattern; `migrate` twice = no-op).
5. Single-writer: two concurrent `chat.send` on one session serialize (existing `turn_in_progress`
   guard + WAL) — no interleaved L2 rows, no lost history.

## What this version explicitly does NOT include
- L1 windowing / compression (v0.4.1 — this version persists the *full* history; bounding comes next).
- L3 semantic facts, core memory, the `remember` upgrade (v0.4.2).
- Any recall / embedding (v0.4.3).
- Any dream / consolidation (v0.5.0).

## Risk
- **Swapping a shipped path** (`session.ts`): mitigate with `LUNA_PERSIST` default-on but
  test-overridable; keep the in-memory shape identical so `runTurn` is untouched except for the
  persist calls.
- **Two DBs vs one:** reuse the single trace DB connection (`getTraceStore`'s db) rather than open a
  second — avoids WAL-file contention. Confirm `migrate` handles multiple migration dirs or merge them.

## Notes for v0.4.1 (don't foreclose)
- Keep `history` the full thread here; v0.4.1 introduces a *separate* bounded "active window"
  derived from it + a `rolling_summary` column on `sessions`. Don't truncate `history` in v0.4.0 —
  L2 + full history are the ground truth the window re-derives from.
