# v0.4.2 — L3 semantic store + core memory + remember/forget upgrade

> **Status: PLANNED.** Initiative: memory-dream-substrate (Order 2, version 3/5). Risk:
> **Medium-High** (touches the per-turn prompt + reshapes the shipped `remember` tool contract).
> Depends: v0.4.0. Flag: `LUNA_MEMORY_INJECT` (default on; `=0` skips the per-turn L3 block).

## Goal

Give Luna durable facts she manages herself, and a prose **core memory** of who she is and where the
relationship stands — both injected into every turn. Upgrade the v0.2 `remember` stub (in-memory
`Map`, `kind: fact|preference|moment`) into the SQLite L3 store with Python's category structure and
a `forget` action, honoring the cut-list's "one `remember` tool with a discriminated input"
(final shape: `action` + `category`).

## What ships

### `packages/protocol/src/memory.ts` (extend)
- `L3Category = z.enum(['core_facts','preferences','key_moments','active_threads','project_context'])`
  (ported from `semantic_memory.py:42-46`; per-category caps confirmed at build).
- `L3Fact { id, category, text, created_ms, confidence?, expires_ms? }` (`active_threads` set
  `expires_ms`; TTL-expire on read, like Python's `ACTIVE_THREAD_TTL_DAYS`).
- `CoreMemory { self_state: string, relationship_status: string, updated_ms }` — **prose**, not
  Python's 5 structured fields (Alan decision E + 2026-06-12 confirmation).
- Remember tool input becomes a discriminated union on `action`:
  `{ action:'add', category, text, confidence? }` | `{ action:'forget', id }` |
  `{ action:'update_self', self_state?, relationship_status? }`.

### `packages/server/src/memory/` (extend)
- `migrations/0004_l3_core.sql` — `l3_facts` (+ index on `(category, created_ms)`; includes
  `deleted_ms INTEGER` nullable — **soft delete with time-validity**), `core_memory` (single row),
  `core_memory_audit` (append-only: `t_ms, prev_self_state, prev_relationship, source`) — the
  lightweight rollback (Alan decision E's prose + keep-the-undo).
- `l3Store.ts` — `addFact` (with `_dedupKey` punctuation-normalized dedup, ported from
  `semantic_memory.py:138-141`), `forgetFact(id)` (**soft delete: sets `deleted_ms`** — excluded
  from `listFacts`/injection/recall, retained for time-travel queries like "当时他还喜欢猫"; this
  deliberately corrects Python's hard-delete `ForgetTool` — TS keeps "this was once true"
  first-class, not buried in an audit log), `listFacts(category?)` (filters `deleted_ms` + TTL).
- `coreMemory.ts` — `getCore()`, `updateCore(patch, source)` (writes prior state to
  `core_memory_audit` first), `restore(n)` (roll back n steps from the audit table). Prose only;
  no consistency tripwire. The audit table is **core-memory-only** — facts use soft delete instead.
- `renderCoreBlock.ts` — builds the **stable** memory prefix: prose core memory + capped top facts
  per category. Goes into the system prompt **before a `cache_control` breakpoint** — it changes
  only when memory actually changes (a `remember` call or a dream update), so the prompt cache
  survives normal turns intact. **Never interpolate timestamps or per-turn values here.** This is
  the TS home of Python's `build_semantic_memory_block` — survives the cut-list's
  `core_memory_refiner` removal (the module name is cut; the rendering function is not) — but
  cache-aware placement is a TS-only requirement Python never had (Python re-sent full context
  every turn; the TS line's #1 goal is latency, and a byte-varying system prompt would invalidate
  the Anthropic prefix cache on every turn).

### Modified
- `packages/server/src/tools/builtin/remember.ts` — rewritten: SQLite-backed, discriminated `action`
  input, `forget` + `update_self` folded in. Stays `session-serial`. Removes the `// v0.4 will move
  this to SQLite` marker. (`getRememberStore`/`resetRememberStore` test helpers point at SQLite now.)
- `runTurn.build_request` — gains the `buildSystemPrompt(session)` seam v0.3 reserved: composes
  `[persona placeholder + renderCoreBlock()]` and marks a `cache_control: {type:'ephemeral'}`
  breakpoint after the stable prefix (gated by `LUNA_MEMORY_INJECT`).
- `packages/server/src/provider/types.ts` + `anthropic.ts` — `ProviderRequest.system` widens from
  `string` to system content blocks so the cache breakpoint can ride through the SDK.
- `ToolName` enum + registry — no new tool name (still `remember`); the surface stays at its current
  count (the discriminator does the work, per the cut-list — whose final shape is `action` +
  `category`, superseding the early `input.kind` sketch).

## Tests
1. `remember{action:'add', category:'core_facts', text}` → row in `l3_facts`; survives restart.
2. `remember{action:'forget', id}` → **`deleted_ms` set (row retained)**; excluded from `listFacts`
   and the injection block; a time-travel query (`listFacts({asOf})`) still sees it as valid before
   its `deleted_ms`.
3. Dedup: adding a punctuation-variant of an existing fact is skipped (not duplicated). Re-adding a
   soft-deleted fact's text creates a fresh row (the old row stays as history).
4. `active_threads` past TTL is filtered on read, not returned in the injection block.
5. `update_self` → `core_memory` updated, prior state in `core_memory_audit`; `restore(1)` rolls back.
6. `renderCoreBlock` injects prose core memory + capped top facts; `LUNA_MEMORY_INJECT=0` skips it.
7. End-to-end (mock provider): a turn where the model calls `remember` then the next turn's system
   block contains the new fact.
8. **Cache stability**: two consecutive turns with no memory change produce byte-identical system
   prompts (asserted via MockProvider request snapshots); a `remember` call changes it exactly once.

## What this version explicitly does NOT include
- Embedding / RAG recall — the injection block uses recency/category caps here; semantic retrieval
  is v0.4.3.
- Dream-driven reconciliation (`memory_audit` / `persona_update`) — v0.5.0. v0.4.2 ships the *tools*
  and *stores*; dream is what calls them in bulk offline.

## Risk
- **Prompt-cache discipline** — the system prompt must change **only** when memory actually changes,
  never per-turn. Test 8 guards it. Volatile, per-query content has no business in this version's
  block (that's v0.4.3's message-level recall block).
- **Contract reshape of a shipped tool** — `remember`'s input changes from flat to discriminated.
  v0.2's `remember` is only reachable via the LLM (v0.3) and `dev.dispatch_tool`; update both call
  paths + tests. No external consumer.

## Acceptance criteria
- [ ] L3 facts persist across restart; categories + TTL enforced.
- [ ] `remember`/`forget`/`update_self` all work via the discriminated tool input.
- [ ] Core memory is prose, audited, `restore(n)`-able — no 5-field structure, no consistency tripwire.
- [ ] Per-turn injection block renders; the renderer survives independent of the cut `core_memory_refiner` name.
- [ ] `bun test` green; `tsc` clean; no `as any` on the tool boundary.
- [ ] `DEVELOPMENT.md` v0.4.2 row + record.

- Dream's `memory_audit` (v0.5.0) calls `addFact`/`forgetFact` in bulk; `persona_update` calls
