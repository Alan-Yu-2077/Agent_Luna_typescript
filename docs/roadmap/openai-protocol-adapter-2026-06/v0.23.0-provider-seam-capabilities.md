# v0.23.0 — Provider seam: capabilities + factory

> **Status: PLANNED.** Initiative: OpenAI-protocol adapter (Order 16, version 1/4). Risk: **Low**.
> Depends: nothing. Flag: `LUNA_PROVIDER` (default `anthropic` → zero behavior change).

## Goal

Make the provider seam genuinely provider-agnostic *at the edges* before any OpenAI code lands:
a `ProviderCapabilities` descriptor + a `providerFor()` factory, with the existing Anthropic path
routed through both, byte-for-byte unchanged. This isolates the interface shape from the
new-provider risk so v0.23.1's translation work targets a stable seam. No network behavior changes.

## What ships

- **`provider/capabilities.ts` (NEW):** `type ProviderCapabilities = { thinking: boolean;
  promptCache: boolean; interleavedToolStreaming: boolean; toolUse: boolean; systemRole: boolean;
  maxOutputTokens: number }`. Add `capabilities: ProviderCapabilities` as a readonly field on the
  `Provider` interface ([`provider/types.ts:50`](../../../packages/server/src/provider/types.ts)).
- **`AnthropicProvider`:** declare `capabilities = { thinking:true, promptCache:true,
  interleavedToolStreaming:true, toolUse:true, systemRole:true, maxOutputTokens: MAX_TOKENS }`.
  No other change — `chatStream`/`complete` are untouched.
- **`provider/factory.ts` (NEW):** `providerFor(opts?: {apiKey?: string}): Provider` reads
  `LUNA_PROVIDER` (`anthropic` default; `openai` returns a `not-yet-implemented` throw until
  v0.23.1) and returns the right impl. `main.ts` ([`main.ts:71-77`](../../../packages/server/src/main.ts)) switches `new AnthropicProvider()` / `new AnthropicProvider({apiKey: summarizerKey})` to `providerFor()` / `providerFor({apiKey: summarizerKey})` — same result under the default.
- **`runTurn` cache-control gate:** the `cache_control:{type:'ephemeral'}` breakpoint
  ([`runTurn.ts:148`](../../../packages/server/src/turn/runTurn.ts)) becomes conditional on the active provider's `capabilities.promptCache` (Anthropic: still emitted; future OpenAI: skipped). Thread the capability in via the existing `TurnState` (the provider is already on `state.provider`). Anthropic path output identical.
- **`REWRITE_CONTEXT.md` amendment:** record that the provider LD is amended — chat is no longer
  Anthropic-SDK-only; a fresh `OpenAIProvider` (honoring per-instance `apiKey`) is allowed. Cite the
  rationale from the initiative README ("Why the amendment is justified").
- **`MockProvider`:** add a `capabilities` field (defaults mirroring Anthropic) so tests compile.

## Tests

- `provider/factory.test.ts`: `providerFor()` returns an `AnthropicProvider` by default; with
  `LUNA_PROVIDER=openai` it throws a clear not-implemented error (until v0.23.1); `apiKey` is
  threaded to the constructor.
- `runTurn.test.ts` (extend): with a `promptCache:false` provider the system block carries **no**
  `cache_control`; with `promptCache:true` it does (byte-identical to today).
- Full suite green: the Anthropic default path is unchanged (existing `anthropic.test.ts`,
  `runTurn.test.ts`, `ws.test.ts` pass without edits beyond the `capabilities` field).

## What this version explicitly does NOT include

- Any OpenAI translation or network code (`LUNA_PROVIDER=openai` throws) → v0.23.1.
- A model registry / per-model config → v0.23.3.
- UI exposure of provider/model choice → out of initiative unless Open Question #3 says otherwise.

## Risk

- **Low** — additive interface field + a factory + one conditional. The only behavior touch is the
  `cache_control` gate; mitigated by the byte-identical assertion test on the Anthropic path.
- Threading `capabilities` into `runTurn`'s system-block builder must not change the cached block's
  bytes for Anthropic (the prefix-cache invariant). Test asserts equality.

## Acceptance criteria

- [ ] `Provider` has a `capabilities` descriptor; both `AnthropicProvider` and `MockProvider` set it.
- [ ] `providerFor()` is the single construction point (used by `main.ts`); default = Anthropic.
- [ ] `cache_control` is gated on `capabilities.promptCache`; Anthropic system block byte-unchanged.
- [ ] `REWRITE_CONTEXT.md` records the provider-LD amendment.
- [ ] `bun test` green, `tsc` ×3 clean; default runtime behavior unchanged.

## Notes for v0.23.1 (don't foreclose)

- `ProviderCapabilities` is the branch point v0.23.1+ uses for thinking / tool / system-role /
  token-param differences — keep it small but extensible (add fields, don't reshape).
- `providerFor()` is where the OpenAI impl slots in — v0.23.1 only flips the `openai` branch from
  throw to `new OpenAIProvider(opts)`.
