# v0.23.2 — OpenAI streaming SSE → ProviderEvent (+ reasoning→thinking)

> **Status: PLANNED.** Initiative: OpenAI-protocol adapter (Order 16, version 3/4). Risk: **Medium-High**.
> Depends: v0.23.1. Flag: `LUNA_PROVIDER=openai` (streaming behind `LUNA_OPENAI_STREAM`, default
> on once verified; behind-flag at first).

## Goal

Restore the rewrite's #1 latency win on the OpenAI path: real **token streaming with interleaved
tool-use**, by reading the OpenAI chat-completions SSE stream and translating its deltas into the
existing `ProviderEvent` union as they arrive — no buffer-till-end. Plus reasoning-model support:
surface reasoning as `thinking_delta` so the existing thinking UI works.

## What ships

- **`OpenAIProvider.chatStream` real SSE path** (replaces the v0.23.1 non-streaming body; same
  emitted `ProviderEvent` sequence shape):
  - Request `stream: true` (+ `stream_options: {include_usage: true}` so the final usage chunk
    arrives). Read the SSE via the `fetch` body reader (the v0.23.1 test seam stays).
  - `choices[0].delta.content` → `text_delta`.
  - `choices[0].delta.tool_calls[]` (streamed): each carries an `index`; the first chunk for an
    index has `id`+`function.name` → emit `tool_use_start`; subsequent chunks carry
    `function.arguments` string fragments → emit `tool_input_delta` (partial_json), keyed by the
    `index→{id,name}` map (mirrors the Anthropic `openToolBlocks` map at [`anthropic.ts:85-101`](../../../packages/server/src/provider/anthropic.ts)).
  - `choices[0].delta.reasoning` / `reasoning_content` (o-series & compatible) → `thinking_delta`
    (capability-gated; absent on non-reasoning models — cleanly omitted).
  - On the final chunk: assemble the synthesized `assistantContent` (reuse the v0.23.1 builder from
    accumulated text + completed tool_calls), read `finish_reason`→`stopReason`, `usage`→
    `ProviderUsage`, emit `message_stop`.
  - Tolerate provider variance: `usage` may arrive on the last content chunk or a trailing
    usage-only chunk; tool-call argument fragments may be empty strings; `[DONE]` sentinel ends it.
- **`capabilities` flips:** `OpenAIProvider.capabilities.interleavedToolStreaming = true`;
  `thinking` becomes true when the configured model is a reasoning model (set in v0.23.3's registry,
  or a `LUNA_OPENAI_REASONING=1` interim flag here).
- **`LUNA_OPENAI_STREAM`** flag (default off this version, flip on after the live smoke) so a
  streaming bug can't regress the (already-working) v0.23.1 non-streaming path.

## Tests

- `provider/openai/stream.test.ts` (injected SSE chunk fixtures): a text-only stream yields ordered
  `text_delta`s then `message_stop`; a tool-call stream yields `tool_use_start` then ≥1
  `tool_input_delta` (fragmented arguments reassemble to valid JSON) then `message_stop` with the
  tool in `assistantContent`+`toolUses`; a reasoning stream yields `thinking_delta`; a trailing
  usage-only chunk maps into `message_stop.usage`; `[DONE]` terminates.
- Interleaving assertion: for a stream that emits text, then a tool call, then more text, the
  `ProviderEvent` order interleaves (not all text buffered to the end) — the latency contract.
- `runTurn.test.ts` (extend): a streamed OpenAI turn with a mid-stream tool call drives the dispatch
  loop the same way the Anthropic stream does (the `message`-tool streaming path at
  [`runTurn.ts:453-501`](../../../packages/server/src/turn/runTurn.ts) is provider-agnostic — assert it still fires `tool.progress`/bubbles).

## What this version explicitly does NOT include

- A model registry / per-model capability table → v0.23.3 (here, capability is a couple of interim
  flags).
- Non-reasoning provider quirks beyond streaming (system→developer role swap, `max_completion_tokens`
  naming) → consolidated in v0.23.3.

## Risk

- **Medium-High** — SSE shape + the streamed `tool_calls[].index` accumulation differ materially
  from Anthropic and vary across backends. Mitigated by: fixture-driven tests (no live dep), the
  `LUNA_OPENAI_STREAM` default-off flag (falls back to the proven v0.23.1 non-streaming path), and
  reusing the same synthesized-`assistantContent` builder so streamed and non-streamed turns persist
  identical history.
- Reasoning content placement is provider-specific (`delta.reasoning` vs `reasoning_content` vs a
  separate block) — handle the common shapes, omit cleanly when absent.

## Acceptance criteria

- [ ] A streamed OpenAI turn emits interleaved `text_delta`/`tool_use_start`/`tool_input_delta`,
      then a `message_stop` whose `assistantContent`+`usage` are correct (fixture-tested).
- [ ] Fragmented `tool_calls.arguments` reassemble into valid JSON tool input.
- [ ] A reasoning model's reasoning surfaces as `thinking_delta`; non-reasoning models omit it.
- [ ] `LUNA_OPENAI_STREAM=0` falls back to v0.23.1 non-streaming cleanly; Anthropic path unchanged.
- [ ] `bun test` green, `tsc` ×3 clean.

## Notes for v0.23.3 (don't foreclose)

- The reasoning-detection + token-param + role-name decisions made here as interim flags become
  per-model entries in v0.23.3's registry — keep them read through `capabilities`, not inline.
