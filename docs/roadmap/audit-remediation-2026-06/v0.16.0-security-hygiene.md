# v0.16.0 — Security hardening + hygiene

> **Status: PLANNED.** Initiative: Audit remediation (Order 9, version 1/3). Risk: **Low**.
> Depends: nothing. Flag: `LUNA_BIND_HOST` (host override) + `LUNA_DEV_TOOLS` (mutating routes).

## Goal

Close the audit's whole **P0/P1 security surface** with the one-line fix it identified, plus the cheap
high-value hygiene items, in a single low-risk batch. The server stops being reachable (and driveable)
off-host by default, the `/_workspace` write/reset routes stop being world-exposed, oversized input is
rejected, and the front-door docs + CI finally match reality. Nothing here changes the single-user
local experience; it removes the ways a shared network or a buggy client can reach the agent, its
memory, and the API key.

## What ships

### Security
- **S1 — loopback bind** (`packages/server/src/main.ts:72`): `Bun.serve({ port, hostname:
  Bun.env['LUNA_BIND_HOST'] ?? '127.0.0.1', ... })`. This single change pulls WS + `/_trace` +
  `/_chat` + `/_workspace` back to localhost → **closes S1, S2's exposure, and S3 at once**. LAN access
  becomes explicit opt-in via `LUNA_BIND_HOST=0.0.0.0`.
- **S2 — gate mutating dev routes** (`workspace/workspace.ts:113,132`): the read-only `/_workspace/api/all`
  stays under `LUNA_VIEWER`; the **mutating** `/reset` + `/edit` move behind an explicit
  `LUNA_DEV_TOOLS` flag (default **off**) so even on-host they're not a stray-click data-wipe. (No
  SQL-injection work — already allowlist-validated + bound; this is purely auth/gating.)
- **S5 — input caps**: add `.max(8000)` (or similar) to `ChatSendEvent.text`
  (`packages/protocol/src/events.ts:19`) and set `Bun.serve({ maxPayloadLength: ... })` in `main.ts`.
  Wire-contract change → the `web` consumer needs no change (it only sends), but the cap is enforced at
  both the schema and socket layers.

### Hygiene (cheap, independent)
- **C1 — CI**: add `.github/workflows/ci.yml` running `bun install` + `bun test` + `bun x tsc --noEmit`
  (per package) on push/PR. Protects every test-pinned invariant (byte-stable system prompt,
  soft-delete, dream-never-overlaps-turn, the new caps).
- **Doc1 — README**: rewrite the stale `README.md:12` ("scaffolding only … no runtime code yet") to
  reflect the shipped stack at v0.16.x.
- **Doc2 — orient skill**: refresh `.claude/skills/luna-ts-orient/SKILL.md` head from v0.12.0 → current,
  and the file map. (Skill content, not package code.)
- **C2 — WS resilience** (`packages/web/src/wsClient.ts:41,46`): buffer unsent frames and flush on
  reopen instead of silently dropping; add backoff+jitter to the fixed 1.5 s reconnect.
- **C3 — proactive clock** (`proactive/cadence.ts:39` vs `scheduler.ts:106`): use **one** clock for both
  the daily quota and quiet-hours (local, since quiet-hours are inherently local).
- **C4 — `fromBlob` safety** (`memory/recall/embed.ts:41`): copy into a fresh 4-byte-aligned buffer
  before constructing the `Float32Array` (defensive; cheap).

## Tests
- `main` bind: a test (or documented manual check) that the server is not reachable on a non-loopback
  interface by default, and that `LUNA_BIND_HOST=0.0.0.0` restores it.
- `workspace`: `/reset` + `/edit` are 403/absent unless `LUNA_DEV_TOOLS=1`; `/all` still served under
  `LUNA_VIEWER`.
- `events.test.ts`: `chat.send` over the cap → Zod reject; at/under → accept.
- `wsClient.test.ts`: a frame sent while CLOSED is buffered and flushed on reopen (inject a fake socket).
- `cadence`/`scheduler`: quota + quiet-hours now agree on the clock (existing tests updated).
- `tsc` clean (protocol + server + web); `bun test` green; CI workflow runs green on a PR.

## What this version explicitly does NOT include
**S4 (read_file sandbox)** — owned by Initiative 8 / v0.15.0 (`resolveInWorkspace`), not duplicated here.
No efficiency work (A1–A4 → v0.16.1). No persistence/vec0/dead-code removal (→ v0.16.2). No memory-depth
change (→ Initiative 10).

## Risk
**Low.** The bind change is the highest-leverage, lowest-LOC fix; the only behavior change is "not
reachable off-host by default," which is the *correct* default for a local companion (and reversible via
`LUNA_BIND_HOST`). The `LUNA_DEV_TOOLS` gate is additive. Input caps are generous (won't clip real
messages — well above the 280-char humanity cap). Watch: the WS payload cap must exceed the largest
legitimate `history`/event frame the server sends back, or reconnect rehydration could be clipped — size
it from the 300-turn `history` payload.
