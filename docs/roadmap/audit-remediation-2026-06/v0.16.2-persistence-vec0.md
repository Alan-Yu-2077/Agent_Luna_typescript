# v0.16.2 — Incremental persistence + dead-infra decisions

> **Status: PLANNED.** Initiative: Audit remediation (Order 9, version 3/3). Risk: **Medium**.
> Depends: v0.16.1 (`content_hash` column; capped recall). Flag: `LUNA_VEC0_KNN` (if vec0 is wired).

## Goal

Finish the audit's structural items: stop rewriting the whole `history_json` every turn (the last
O(N²)), and resolve the two pieces of dead infrastructure — `vec0` (write-only) and the text-mode path
(dead under message mode). These are decisions as much as code: each has a "wire it" and a "remove it"
branch, and this version is where the project commits.

## What ships

### A3/P2 — incremental `history_json`
`session.history` is never truncated and `persistSession` re-serializes + rewrites the whole array every
turn (`sessionStore.ts:74`). L2 is already the clean append-only per-turn source of truth, so:
- **persist only the active window** (`KEEP_MSGS` tail + the rolling summary) in `sessions.history_json`,
  and **rebuild the full history from L2 on load** when needed. The in-memory `session.history` can also
  be bounded to the window + summary (the model never sees more than the window anyway).
- this removes the O(N) per-turn serialize and the unbounded in-memory array in one move.
- **Interaction with Initiative 10:** the window grows to ~100 (clean) turns there — persisting "only the
  window" must store *that* window. Keep the rebuild-from-L2 path so the source of truth stays L2.

### D1/P3 — decide `vec0`
`vec_cache` is created + inserted every turn but never queried; retrieval is TS cosine. Pick one
(Open Q #2 in the README):
- **Wire it** (`LUNA_VEC0_KNN`): replace the brute-force cosine in `retrieve()` (`recall.ts:174`) with a
  `vec0` KNN `MATCH` query over `vec_cache`, fixing P3's O(candidates×dims) scan. Keep the `sqlite-vec`
  dep. Verify result parity with the TS cosine (golden) before flipping.
- **Remove it**: delete the `vec_cache` table + inserts + `initCustomSqlite`/`tryLoadVec` + the
  `sqlite-vec` dependency, and document that recall is TS cosine (fine at single-user scale).
- **Recommendation:** default toward *remove* for the current corpus; but Initiative 10 (~100-clean-turn
  window + diary candidates) enlarges the candidate set, so if Order 10 lands first, prefer *wire it*.
  Decide at build from the then-current corpus size.

### D2 — remove the dead text-mode path
`reply.token` / the `LUNA_MESSAGE_TOOL=0` branch / the controller's text-bubble path are dead under the
default message mode (`runTurn.ts:205`). If message mode is locked permanent (Open Q #3):
- remove the `reply.token` emission, the text-mode branch in `runTurn`, the `reply.token` `ServerEvent`
  variant (`packages/protocol`), and the consumer's text-bubble path (`web/src/controller.ts`) — a clean
  cross-package simplification leaving a single context-assembly path.
- if kept as an escape hatch, leave it but document it as legacy-only.

## Tests
- `sessionStore.test.ts`: a session with N turns persists a bounded `history_json` (window-sized, not
  O(N)); reload rebuilds the full timeline from L2; round-trip equality on the window.
- `recall.test.ts`: if vec0 wired — KNN results match the TS-cosine golden (same top-K, same order
  within tolerance); if removed — TS cosine path unchanged + no `vec_cache` references remain (grep
  test).
- if D2 removed: a wire test that no `reply.token` is emitted in either mode and the protocol union no
  longer carries it; controller test for the text-bubble path deleted; `LUNA_MESSAGE_TOOL=0` either
  errors clearly or is gone.
- `tsc` clean (all packages, since D2 touches the wire contract); `bun test` green.

## What this version explicitly does NOT include
No window-size increase (→ Initiative 10 v0.17.0). No diary injection (→ v0.17.1). No new retrieval
*quality* work beyond the vec0 mechanism swap.

## Risk
**Medium.** The `history_json` change touches persistence — the rebuild-from-L2 path must be exact
(golden round-trip test) or a reload could lose the tail; stage it behind a careful test. The vec0
"wire it" branch must prove parity with TS cosine before replacing retrieval (a silent ranking change
degrades recall quality). D2 is a wire-contract change → both `packages/protocol` consumers
(`server`, `web`) must build in lockstep (the rewrite's core caution).

## Acceptance criteria
- [ ] `history_json` is window-bounded; full history rebuilds from L2 on load; no per-turn full rewrite;
      golden round-trip green.
- [ ] `vec0` decision made + executed (wired-with-parity OR removed-with-dep-dropped); grep proves no
      orphaned write-only table.
- [ ] Text-mode path removed (single context-assembly path) or explicitly retained + documented; wire
      contract consistent across packages.
- [ ] `tsc` clean, `bun test` green, DEVELOPMENT.md v0.16.2 entry, **Initiative 9 closed**.

## Notes for Initiative 10 (don't foreclose)
The rebuild-from-L2 persistence here is what lets Initiative 10 grow the window to ~100 clean turns
without ballooning `history_json` (store the window, source-of-truth stays L2). If `vec0` is wired, diary
candidates (v0.17.1) get KNN for free; if removed, confirm TS cosine still holds at the larger corpus.
