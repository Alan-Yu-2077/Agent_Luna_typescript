# v0.16.3 — Clean durable history (strip thinking + collapse tool I/O)

> **Status: PLANNED.** Initiative: Audit remediation (Order 9, version 4/4). Risk: **Low–Medium**.
> Depends: v0.16.2 (incremental `history_json`). Flag: `LUNA_CLEAN_HISTORY` (default off → on after
> golden round-trip). *Discussion-derived (not an audit finding) — the efficiency foundation Initiative
> 10's deeper window builds on.*

## Goal

Make a stored "turn" cost what a *conversational* turn should cost, by keeping the **scratchpad** out of
durable history. Today `session.history` retains each turn's **thinking blocks** and full **tool-result
payloads** verbatim — so the 24-message window holds only ~4–9 real exchanges, and any larger window
would carry a lot of dead reasoning/tool spew. Stripping thinking (which the Anthropic API already drops
across turns anyway) and collapsing old tool results (keep the `tool_use` record, drop the re-fetchable
payload — the platform's own context-editing pattern) makes a window-turn ~200 tokens of *clean
conversation*. This is what lets Initiative 10 grow the verbatim window to ~100 turns affordably; it also
shrinks every turn's input on its own.

## Grounding (from the SOTA research, June 2026)
- **Thinking is ephemeral by Anthropic's own design**: *"previous thinking blocks are automatically
  stripped from the context window … not part of the conversation history the model sees for subsequent
  turns"* — the only case you *must* keep thinking is the current, still-open tool-use loop
  ([context windows](https://platform.claude.com/docs/en/docs/build-with-claude/context-windows)). So
  removing thinking from durable history changes nothing the model relied on.
- **Tool results are the universal #1 eviction target**: context editing *"keeps the `tool_use` record
  so the model still knows it made the call, but drops the bulky payload … re-fetchable"*
  ([context editing](https://platform.claude.com/docs/en/build-with-claude/context-editing)); Claude
  Code clears tool outputs first, LangChain offloads >20k-token results to a disk pointer, SWE-agent
  collapses observations beyond the last few to a one-liner.

## What ships

### Strip thinking from durable history
- When appending the assistant turn to `session.history` (`turn/runTurn.ts:239`), persist the
  **non-thinking** content blocks (the `message`/text blocks) and drop `thinking`/`redacted_thinking`
  blocks. (Within a live tool loop the current thinking is still passed back as required — this only
  affects what is *stored* for future turns.) Net: the window counts clean conversational turns.

### Collapse old tool results
- Keep tool-call results verbatim only for a small recent slice (e.g. the last ~3–5 turns, mirroring the
  platform default `keep` of 3 tool-use pairs); for older turns, replace the `tool_result` payload with
  a compact marker (`[tool_result elided: <tool> · <summary>]`) while keeping the `tool_use` record.
  The `message` tool's delivery envelope collapses to its `text` (the spoken line is the durable part;
  the segments/metadata are not). Done in the history-assembly path
  (`memory/l1Window.ts buildActiveContext` / the persist seam in `sessionStore.ts`).

### Plumbing
- A `cleanTurn()` helper that produces the durable form of a turn (strip thinking, collapse tool I/O),
  applied at persist time so both the in-memory window and `history_json` (v0.16.2) store the clean form.
- L2 (`l2_turns`) is unaffected — it already stores the user/assistant text per turn (the clean source of
  truth); this is specifically about the `sessions` window blob + the active context sent to the model.

## Tests
- `cleanTurn` strips `thinking`/`redacted_thinking`, keeps text/`message` blocks; collapses a >N-turns-old
  `tool_result` to a marker while preserving the `tool_use` id; the last ~3–5 turns keep full results.
- a turn that was clean already round-trips unchanged; an open tool-loop turn still carries its current
  thinking (the clean transform applies only to *persisted/older* turns, not the in-flight one).
- token-shape assertion: a representative 3-message-+-1-tool turn stores ~200 tokens clean vs the bloated
  baseline (golden, approximate).
- the model's context assembly (`buildActiveContext`) over a clean window is well-formed (valid
  `MessageParam[]`, no orphaned `tool_use` without a record).
- `tsc` clean; `bun test` green.

## What this version explicitly does NOT include
No window-size change — that's Initiative 10 / v0.17.0 (this just makes a turn cheap; the count stays at
the current default here). No diary work. No change to L2 storage or recall.

## Risk
**Low–Medium.** The model never relied on prior-turn thinking (the API strips it), so stripping it from
storage is safe. The collapse must (a) never touch the **in-flight** turn's thinking (signature-validated;
modifying it is a `400`), and (b) keep `tool_use`↔`tool_result` structurally valid in the assembled
context (an orphaned `tool_use` errors) — both pinned by tests. Behind `LUNA_CLEAN_HISTORY` with a golden
round-trip before flip.

## Acceptance criteria
- [ ] Durable history stores clean turns: thinking dropped, old tool results collapsed to markers, recent
      results kept; in-flight thinking untouched.
- [ ] A representative turn's stored size is ~200 tokens (golden); the assembled context is structurally
      valid.
- [ ] `tsc` clean, `bun test` green; flag flipped after golden round-trip; DEVELOPMENT.md v0.16.3 entry;
      **Initiative 9 closed**.

## Notes for Initiative 10 (don't foreclose)
This is the load-bearing dependency for v0.17.0: a ~100-turn verbatim window is ~20k tokens *because* each
stored turn is clean. Keep `cleanTurn` composable so the window resize is purely a count change on top of
it.
