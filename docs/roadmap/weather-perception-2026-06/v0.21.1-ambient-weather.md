# v0.21.1 — Passive ambient weather awareness

> **Status: PLANNED.** Initiative: Weather perception (Order 14, version 2/3). Risk: **Medium**.
> Depends: v0.21.0 (the `fetchWeather` client + `WeatherSnapshot` + `resolveLocation`). Flag:
> `LUNA_WEATHER_AMBIENT` (default-off).

## Goal
Make Luna **passively aware** of the weather — she *knows* it's a grey, cold morning without calling a
tool — by injecting a **TTL-cached, background-refreshed** weather snapshot into the **per-turn uncached
user tail**, exactly as Initiative 12 injects the time block. This is the
cache-sensitive heart of the initiative: the snapshot is volatile, so it must never touch the cached
system prefix, and no network call may be added to the reactive turn.

## What ships
**Background snapshot cache (`tools/web/weather/snapshot.ts`, new).**
- A module-level `WeatherCache`: `getSnapshot(): WeatherSnapshot | null` reads the last good snapshot
  **synchronously** (like `runTurn.ts:238` reads `lastInteractionMs`); returns `null` if cold or past a
  staleness threshold (`LUNA_WEATHER_TTL_MIN`, default ~30, omit past ~2h).
- Refresh is **off the hot path**: a `.unref()`'d `setInterval` (mirroring the proactive heartbeat /
  `scheduler.ts:44`) that calls `fetchWeather(resolveLocation(), resolveTz(), …)` every TTL and stores
  the result (or keeps the last good one + logs on failure — never throws). Started once at boot iff
  `weatherAmbientEnabled()`. Optionally also a refresh-on-stale-read that **fire-and-forgets**
  (`void refresh().catch()`, the `runTurn.ts:859` `maybeFold` pattern) and serves the next turn — but
  **never awaits** a fetch on the reply path.

**Pure formatter (`turn/weatherContext.ts`, new — sibling of `temporalContext.ts`).**
- `buildWeatherBlock(snapshot): string` — pure, synchronous, format-only (no fetch): e.g.
  `- Weather (Shanghai): overcast 18°C, feels 17°C · today high 21° / low 14° · rain likely later`.
  All arithmetic/labeling (WMO→text, °C, high/low) is done here in TS — hand Claude a finished fact.
- `weatherAmbientEnabled() = Bun.env['LUNA_WEATHER_AMBIENT'] === '1'` (the single flag governs both the
  injection gate and the L1 clause, the pattern time uses at `temporalContext.ts:11-14`).

**Uncached-tail injection (`turn/runTurn.ts`, the `build_request`/`parse_input` node).**
- Where `buildTimeBlock(...)` is pushed into the per-turn user-message `blocks` (`runTurn.ts:238-256`),
  add — when `weatherAmbientEnabled()` and `getSnapshot()` is non-null — `blocks.push({type:'text',
  text: buildWeatherBlock(snap)})`. **Wrap in try/catch and omit on failure** (the non-essential-layer
  pattern already there). **Never** into `buildSystemPrompt` (the `cache_control` block).

**Stable cached clause (`persona/l1Contract.ts`).**
- A data-free `WEATHER_CLAUSE` (e.g. *"If you know the weather, let it color your tone or open a cold
  morning warmly — never recite a forecast like a bulletin, and only when it's natural."*) appended in
  `renderL1Contract` like `TIME_CLAUSE` (`l1Contract.ts:98`), gated by `weatherAmbientEnabled()`, with
  the flag folded into the memo cache key (`l1Contract.ts:52`). Carries no interpolated data → byte-
  stable per process, preserving the cache invariant. Pass the gate into **both** `buildSystemPrompt`
  → `renderL1Contract` **and** the `parse_input` injection (single-flag-governs-both,
  `runTurn.ts:114,237`).

## Tests
- `weatherContext.test.ts`: `buildWeatherBlock` formats each snapshot shape (rain/clear/snow, day/night)
  into the expected labeled string; a `null` snapshot path is exercised by the caller test.
- `snapshot.test.ts`: `getSnapshot` returns `null` when cold and when stale (past TTL); a successful
  background refresh (injected `fetchWeather`) populates it; a failing refresh keeps the last good
  snapshot and never throws; the interval is `.unref()`'d.
- **Cache-invariant pin** (`recall.test.ts:193-238` analog): the cached system block is **byte-identical**
  across two turns with *different* weather snapshots — proves the snapshot rides the uncached tail only.
- `runTurn`: with `LUNA_WEATHER_AMBIENT=1` + a stubbed non-null snapshot, the per-turn user message
  contains the weather block and the reactive turn makes **no** weather network call (assert the
  fetcher is not invoked on the reply path); with the flag off, no block and no clause.

## What this version explicitly does NOT include
- **Proactive** weather in the opening (the after-a-night note) — **v0.21.2**.
- Any **subjective weather-mood** line — folded into v0.21.2's bounded suggestion if wanted (Open Q #5).
- The default-flip — stays **off** until v0.21.2 measures cache-hit-rate and flips the set.

## Risk
- **Medium** — this is the cache-discipline version. Two failure modes to prevent: (1) a weather string
  leaking into the cached prefix (busts the cache every turn) — the byte-identical pin test guards it;
  (2) a synchronous fetch sneaking onto the reactive path — the "fetcher not invoked on reply" test +
  the read-snapshot-synchronously design guard it. Mitigation: `getSnapshot` is pure-sync; all I/O is
  the `.unref()`'d background timer; omit-on-cold-or-stale so a dead network just means no weather block,
  never a slow or failed turn.

## Acceptance criteria
- [ ] A labeled weather snapshot rides the **uncached** per-turn tail under `LUNA_WEATHER_AMBIENT=1`; the
      cached system block is **byte-identical** across differing snapshots (pin test green).
- [ ] **Zero** synchronous network call on the reactive turn — the snapshot is read from cache; cold/stale
      → block omitted, turn unaffected.
- [ ] The `WEATHER_CLAUSE` is present (gated) and data-free; `tsc` clean ×3; suite green.
