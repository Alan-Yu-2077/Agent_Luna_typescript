# v0.21.0 — Weather tool + location config

> **Status: PLANNED.** Initiative: Weather perception (Order 14, version 1/3). Risk: **Low–Medium**.
> Depends: nothing. Flag: `LUNA_WEATHER` (default-off).

## Goal
Ship the **data spine** of weather perception: a validated location resolver, a no-key Open-Meteo client
(WMO-code mapping, SSRF-validated, soft-failing), and a model-callable `weather` **pull tool** Luna can
invoke when asked or when relevant. This is the standalone capability layer; v0.21.1 reuses the same
client for ambient injection, v0.21.2 for the proactive note. Nothing ambient or proactive ships here —
just "she can look it up."

## What ships
**Location resolver (`turn/temporalContext.ts`, co-located with `resolveTz`).**
- `resolveLocation(): { lat: number; lon: number; label?: string } | null` — reads `LUNA_LAT_LON`
  (`'lat,lon'` decimal), validates range (`-90..90` / `-180..180`), and on a bad/unset value returns
  `null` (degrade-not-throw, mirroring `resolveTz` `temporalContext.ts:82-91`). Optional
  `LUNA_WEATHER_LOCATION` supplies a human label for display only. **No** IP-geolocation (the fake-IP
  proxy `safeFetch.ts:77-81` makes it report the exit node); **no** silent tz-centroid guess in v1.

**Open-Meteo client (`tools/web/weather/openMeteo.ts`, new).**
- `fetchWeather(lat, lon, tz, { signal, units }): Promise<WeatherSnapshot>` — builds the verified
  endpoint (`api.open-meteo.com/v1/forecast?…current=temperature_2m,weather_code,precipitation,is_day,
  wind_speed_10m&daily=temperature_2m_max,temperature_2m_min,precipitation_probability_max,weather_code,
  sunrise,sunset&timezone=${tz}&temperature_unit=${units}`), **validates it through the exported
  `assertPublicUrl(url)`** (`safeFetch.ts:143-179`) for SSRF, then does a plain `fetch(url, {signal})` +
  `res.json()`. **Do NOT route through `safeFetch`** — its `text/html|text/plain` content-type gate
  (`safeFetch.ts:360-366`) rejects the API's `application/json`.
- `WeatherSnapshot = { label, tempC, feelsLike?, condition, code, isDay, precipMm, highC, lowC,
  precipChance, sunrise, sunset, observedMs }`. A pure `wmoToCondition(code)` table (0 clear … 95
  thunderstorm; unknown → fall back to `precip>0` wet/dry) lives here and is unit-tested.
- A **test seam**: `setWeatherFetcher(fn)` (mirroring `web_fetch`'s `setWebFetcher`) so tests inject a
  fake JSON response and never touch the network.

**`weather` pull tool (`tools/builtin/weather.ts`, new).**
- `defineTool` (`tools/defineTool.ts:29-57`): `name:'weather'`, `concurrency:'safe-parallel'`,
  `proactiveRisk:'safe'` (read-only, silent-OK in proactive turns), `timeoutMs` from
  `LUNA_WEATHER_TIMEOUT_MS` (default ~10000, read once at module load, `web_search.ts:59-75`).
- Input `z.object({})` (zero-arg, uses the configured location like `time_now`
  `tools/builtin/time_now.ts:4-31`). Output a typed `z.object` of the snapshot fields; `summarize`
  returns one line (`Shanghai · overcast 18°C, high 21°/low 14°`).
- `execute`: `yield {kind:'progress', payload:{note:'正在查天气…'}}`; early-`yield {kind:'err',
  code:'aborted'}` if `ctx.abortSignal.aborted`; resolve location (→ if `null`,
  `yield {kind:'err', code:'execution_exception', message:'location not configured (set LUNA_LAT_LON)',
  recoverable:true}`); else `fetchWeather(...)` in try/catch and `yield {kind:'ok', data}` or a
  recoverable `'err'` — **never throw past the generator** (`web_fetch.ts:102-122`).

**Registration (3 places).**
- Add `'weather'` to the `ToolName` `z.enum` (`protocol/src/tools.ts:3-33`).
- `weatherTools` sub-registry + `weatherEnabled()` (`Bun.env['LUNA_WEATHER'] === '1'`, **default-off**;
  no key needed, so no key-degrade) + `withWeather(base)` in `tools/registry.ts:182-199`.
- Wire `withWeather(...)` into the `main.ts:94-105` boot nest + a `[weather]` boot-log tag.

## Tests
- `weather.test.ts` (mirrors `time_now.test.ts`): with `setWeatherFetcher` returning a canned snapshot,
  `execute` yields exactly one `{kind:'ok'}` with the typed fields; `summarize` contains the label;
  an unset `LUNA_LAT_LON` yields a recoverable `'err'` (not a throw); an aborted ctx yields `'aborted'`.
- `openMeteo.test.ts`: `wmoToCondition` covers each documented WMO bucket + the unknown→precip fallback;
  the URL builder emits the exact verified param set; `assertPublicUrl` is invoked on the endpoint.
- `resolveLocation`: valid `'31.23,121.47'` parses; out-of-range / garbage / unset → `null` (no throw).
- `registry`/`tsc`: `weather` mounts only when `LUNA_WEATHER=1`; the `ToolName` enum addition compiles in
  both `protocol` consumers (`server` + `web`).

## What this version explicitly does NOT include
- **Ambient injection** (the per-turn uncached snapshot + L1 clause) — **v0.21.1**.
- **Proactive weather** in the opening — **v0.21.2**.
- Free-text **geocoding** (Open-Meteo needs lat/lon; a name→coord pre-call) — deferred (Open Q #4).
- Any cached/background snapshot — the tool fetches live per call; the TTL cache is v0.21.1's concern.

## Risk
- **Low–Medium.** A new outbound network tool, but a single fixed trusted host, no key, read-only,
  default-off. Mitigation: `assertPublicUrl` keeps the SSRF posture uniform; soft-fail means a network
  blip is a describable `err`, not a crash; the test seam keeps the suite network-free. Verify a real
  call once manually (behind the fake-IP proxy) before flipping the flag in a later version.

## Acceptance criteria
- [ ] `LUNA_WEATHER=1` mounts a `weather` tool that returns a structured snapshot for the configured
      `LUNA_LAT_LON`; unset location → recoverable `err`, never a throw or a brick.
- [ ] The weather GET is `assertPublicUrl`-validated and does **not** go through `safeFetch`'s
      text-only gate; WMO codes map to human conditions; `tsc` clean ×3, suite green (network-free).

## Notes for v0.21.1 (don't foreclose)
- Keep `fetchWeather` + `WeatherSnapshot` + `setWeatherFetcher` **decoupled from the tool** (in
  `tools/web/weather/`) so v0.21.1's background refresher and `buildWeatherBlock` reuse them directly
  without importing the tool. Keep `resolveLocation` pure + env-read-per-call so the ambient path and
  the tool share one truth.
