# v0.22.3 — Fuzzy detectors + delete the wake-gate (initiative close)

> **Status: PLANNED.** Initiative: Proactive pipeline redesign (Order 15, version 4/4). Risk: **Low**.
> Depends: v0.22.2. Flags: `LUNA_PROACTIVE_OPEN_THREADS`, `LUNA_PROACTIVE_FOLLOW_THROUGH` (default `0`).

## Goal

Add the two "soft" detectors that need heuristics (so they ship **default-off** and seed *gently*),
then **remove the dead `wakeGate`** now that nothing reaches for it. Close the initiative.

## What ships

**`proactive/detectors.ts`** — two flagged-off detectors:
- `openThreadAged` — reads `listFacts({ category: 'active_threads' })` (L3) and/or a conservative
  question-shape + age heuristic over `listRecentL2`; fires when a thread has been open > N hours.
  Seeds **softly**: *"there may be an open thread about X — only raise it if it still feels alive,"*
  so a stale/false positive yields silence, not an awkward nudge. `debounceKey = "thread:<id>"`.
- `promisedFollowThrough` — detects a prior "I'll do/check X" beat that hasn't been followed up; soft
  seed; `debounceKey = "promise:<hash>"`.
- Both default **OFF** (`LUNA_PROACTIVE_OPEN_THREADS` / `LUNA_PROACTIVE_FOLLOW_THROUGH`), enabled after
  observation, because their precision is heuristic.

**Delete the wake-gate** — remove `proactive/wakeGate.ts` (`wakeGate`, `WakeVerdict`, `WakeIntent`,
`buildWakeContext`, `parseVerdict`, `WAKE_SYSTEM`), the `LUNA_PROACTIVE_LLM_GATE` branch in
`scheduler.ts`, and the now-unused `dreamLlm` reference in the scheduler hot path (`dreamLlm` stays
only for the real dream-cycle handoff). Drop `wakeGate.test.ts`. With it go the `judgment_unparseable`
/ `judgment_unavailable:exception` failure classes entirely.

**Dev trigger** — a gated `dev.*` path (e.g. extend the existing `dev.dispatch_tool` gating, or a
`LUNA_DEV_TOOLS`-only `dev.proactive_fire`) to **force one proactive turn** bypassing detectors, for
live "is the whole chain wired?" verification without waiting on a real trigger.

**Docs** — mark Initiative 15 ✅ in the master roadmap; `DEVELOPMENT.md` entries across v0.22.0–v0.22.3.

## Tests

- `detectors.test.ts`: `openThreadAged` fires only past the age threshold and only with the flag on;
  a thread with no aged candidate → null; soft seed present. `promisedFollowThrough` likewise.
- Repo-wide: no remaining import of `wakeGate`/`WakeVerdict` (a grep guard or a compile check); the
  scheduler no longer references `dreamLlm` except via the dream handoff.
- `scheduler.test.ts`: the dev force-trigger runs exactly one proactive turn under the single-turn lock.

## What this version explicitly does NOT include

- Any return of an LLM "ambient reach-out" gate (Open Question #1 — deferred unless she feels too
  scheduled in practice).
- Multi-user proactive fan-out.

## Risk

- Fuzzy detectors are the false-positive risk — mitigated by **default-off** + **soft seeding** (the
  turn can still choose silence) + per-detector debounce + the daily quota.
- Deleting `wakeGate` is safe only once v0.22.0–v0.22.2 fully cover the openings and the flag has had
  one release as a fallback — confirm no caller before removing.

## Acceptance criteria

- [ ] `openThreadAged` + `promisedFollowThrough` ship default-off, pure, clock-injectable, soft-seeded.
- [ ] `wakeGate.ts` + its symbols + tests are **gone**; no import remains; scheduler hot path is
      LLM-free.
- [ ] The dev force-trigger fires one proactive turn on demand (live-verify the chain).
- [ ] **Initiative acceptance** (from the README) all checked: she actually speaks; zero idle LLM
      polling; after-a-night reliable; silent draft doesn't burn quota; rail still caps; tests green.

## Notes (initiative done)

- The proactive path is now: cheap deterministic detectors → the same turn graph → `{spoke}` is the
  decision. No speculative LLM. If a future "spontaneous, reason-less" reach-out is wanted, add it as
  **one more detector** (e.g. a rare randomized "long-quiet musing" with its own debounce), not a
  per-tick gate — the architecture makes that a small additive change.
