# v0.22.0 — Detector-MVP: she actually starts speaking

> **Status: PLANNED.** Initiative: Proactive pipeline redesign (Order 15, version 1/4). Risk: **Low**.
> Depends: v0.21.6 (`preloadSessions`) deployed. Flag: `LUNA_PROACTIVE_LLM_GATE` (default `0`).

## Goal

The **smallest change that makes Luna actually reach out**. Replace the per-tick LLM wake-gate with a
single inline **deterministic** detector — *first connected tick after a night* — that runs the
existing silence-capable proactive turn. Let the turn's own `{spoke}` be the decision (drafting-as-
decision). Add the **spoke/silent quota split** so a silent draft doesn't burn the daily budget. Keep
`wakeGate` in-tree behind a default-off flag as a one-release fallback. After this version she
reliably greets the morning after an overnight gap, and the cadence rail still hard-caps spam.

## What ships

**`proactive/scheduler.ts` (`tickOnce`)** — in the per-session loop:
- **Default path** (`LUNA_PROACTIVE_LLM_GATE !== '1'`): skip `wakeGate` entirely. Gate the detector by
  the **anti-spam subset only** — add a small `passesAntiSpam(cadence, ctx)` = **quiet-hours +
  cooldown + quota** (a preview of v0.22.1's `safetyRail`). **Deliberately NOT `deep_absence` and NOT
  the 10 m `too_soon` floor**, because a gap *> 18 h* (overnight into the afternoon, a weekend, a
  2-day absence) is exactly when an after-a-night / after-absence greeting *should* fire — reusing the
  full `shouldConsiderWake` would swallow it (the never-fires hole this initiative exists to kill). If
  it passes, compute `afterANightOpening(now, lastInteractionMs(session))`; if true, run
  `runProactiveTurn({intent: 'spontaneous', seed: <after-a-night seed>})`. Else `continue` (no LLM
  call, microseconds).
- **Fallback path** (`=== '1'`): the current `shouldConsiderWake` → `wakeGate` flow, unchanged.
- Keep the existing `ticking` reentrancy guard, `.unref()`, `activeTurn`/`isDreaming` re-check, and
  `pendingDream` handoff verbatim.

**`proactive/cadence.ts`** — split the post-fire commit:
- `commitProactiveSpoke(c, now)` = current `commitProactive` (stamp `lastProactiveMs` **and** bump the
  daily quota).
- `commitProactiveSilent(c, now)` = stamp `lastProactiveMs` (→ cooldown) **without** bumping the quota.
- `tickOnce` picks by the turn's `{spoke}`: `spoke ? commitProactiveSpoke : commitProactiveSilent`.
  Either way the cooldown stamps, so a silent consideration can't re-fire next tick.

**`proactive/proactiveTurn.ts`** — `runProactiveTurn` accepts an optional `seed: string` appended to
the USER-role framing (rides the **uncached** tail — cache invariant preserved). Factor the private
`lastInteractionMs(session)` helper out to a shared export (used by both the turn and the scheduler).

**`wakeGate.ts`** — untouched code, now only reachable via `LUNA_PROACTIVE_LLM_GATE=1`.

No schema change, no migration, no wire-contract change.

## Tests

- `cadence.test.ts`: `commitProactiveSilent` stamps `lastProactiveMs` but leaves `quotaUsed`
  unchanged; `commitProactiveSpoke` bumps both. A silent turn keeps the cooldown active next tick.
- `scheduler.test.ts`: with `LUNA_PROACTIVE_LLM_GATE` unset and a stubbed `afterANightOpening`→true +
  a passing cadence, a tick runs **one** `runProactiveTurn` and **no `wakeGate` call**; →false runs
  neither. A returned `{spoke:false}` commits silent (quota unchanged); `{spoke:true}` commits spoke.
- `scheduler.test.ts` (the deep_absence regression): a **> 18 h** gap (e.g. 20 h overnight, or a
  weekend) with `afterANightOpening`→true **still fires** — the detector is gated by `passesAntiSpam`,
  not swallowed by `deep_absence`. (Asserts the fix for the review's HIGH finding.)
- `scheduler.test.ts`: with `LUNA_PROACTIVE_LLM_GATE=1`, the legacy `wakeGate` path still runs (the
  fallback is intact).
- `proactiveTurn.test.ts`: a `seed` appears in the framing (the uncached user tail), not the system
  block.

## What this version explicitly does NOT include

- The detector **registry** + scheduled-window/weather detectors → v0.22.1/v0.22.2.
- The per-day **slot** persistence + migration → v0.22.1.
- **Event-hook** firing (ws reconnect / weather refresher) → v0.22.2.
- **Deleting** `wakeGate` → v0.22.3 (kept as the flagged fallback for exactly one release).

## Risk

- After-a-night may fire only once per overnight gap — acceptable for an MVP (the scheduled-floor in
  v0.22.1 adds daily openings). Mitigation: the seed makes the turn likely to actually speak (it has a
  concrete reason), unlike the old "no natural reason → hold" path.
- A flaky proactive turn throws → it emits nothing and retries next eligible tick (no fail-to-
  permanent-silence; relies on `runTurn`'s own error handling, no suppressing backoff added).

## Acceptance criteria

- [ ] With the flag unset, a tick after an overnight gap (cadence-eligible, non-quiet-hours) runs a
      proactive turn and **no `wakeGate` LLM call** (verified in traces: a proactive turn with
      `spoke=true` appears).
- [ ] A truly idle non-overnight tick makes **zero LLM calls**.
- [ ] A silent proactive draft leaves `proactive_quota_used` unchanged but stamps the cooldown.
- [ ] `LUNA_PROACTIVE_LLM_GATE=1` restores the exact legacy behavior.
- [ ] `bun test` green, `tsc` ×3 clean.

## Notes for v0.22.1 (don't foreclose)

  {intent, seed} | null`, first-match-wins. v0.22.1 lifts the inline after-a-night into the registry
  unchanged.
- The spoke/silent commit split is the contract the whole initiative leans on — keep it stable.
