# v0.9.0 — Measurement sweep + contract tuning + initiative close

> **Status: PLANNED.** Initiative: Action Integrity Rails (Order 4, version 5/5, capstone). Risk:
> **Low**. Depends: all of v0.8.x. Flag: flips `LUNA_DECISION_AUDIT`, `LUNA_L1_CONTRACT`,
> `LUNA_INTEGRITY_GUARD` defaults **on**.

## Goal

Prove the initiative did what it set out to — convert the recorded baseline into a before/after,
tune the fuzzy edges (intent dictionary, recall wording) against real numbers, flip the defaults
on, and close. This is the Initiative-3 A/B discipline applied to action integrity.

## What ships

- **Extend `scripts/ab-message-mode.ts`**
  ([packages/server/scripts](../../../packages/server/scripts/ab-message-mode.ts)) into an
  integrity sweep: same scripted conversation (plus a few turns engineered to bait defection —
  "你能查一下今天天气吗", "记住我下周一要交报告", a paraphrased callback to an earlier fact) run
  **four ways**: contract+guards off (baseline), L1 only, L1+guards, L1+guards+recall. Per-turn
  columns: intent-without-act defections (from `decision` traces), corrective retries used,
  tool-fire rate on tool-warranted turns, recall calls + hit usefulness, top-level leak chars,
  median first-visible latency, humanity violations (regression guard — must stay 0).
- **Recorded results table** in the DEVELOPMENT entry (like the v0.7.0 A/B): the defection
  before/after is the headline acceptance number for the whole initiative.
- **Tuning, against the numbers**: trim/extend the intent dictionary to minimize false-positive
  retries; adjust L1 recall wording if recall over-/under-fires. Text changes only — no
  architecture.
- **Default flip**: `LUNA_DECISION_AUDIT`, `LUNA_L1_CONTRACT`, `LUNA_INTEGRITY_GUARD` default on
  (`!== '0'`), each keeping a `=0` escape hatch (the Initiative-3 pattern). `.env.example`
  documents all three.
- **Docs close**: REWRITE_CONTEXT LD #14 marked **LANDED** with as-shipped notes; Open Q #9 marked
  resolved-and-shipped; roadmap master README → Initiative 4 ✅ shipped, head v0.9.0; orient skill
  gains the action-integrity file map; the `reasoning-spec` misreading note in
  [[luna-reasoning-intent]] confirmed honored.

## Tests

- The sweep runs green and writes the four-way table (script-level, not unit — but it must execute
  without error in CI-less manual run, like the v0.7.0 A/B).
- Default-on: full suite passes with no env set; tests that pin the text path or flag-off behavior
  set the relevant `=0` explicitly (audit, don't blanket-flip — the Initiative-3 lesson).
- Humanity violations stay 0 in message mode (no regression from the new contract text).
- All three escape hatches verified: each `=0` restores the prior version's behavior.

## What this version explicitly does NOT include

No proactive / self-continuation (Initiative 5 — those are the gate consumers, designed fresh
there). No new capabilities — this version is measurement, tuning, and closing. If the sweep
reveals a *structural* gap (not a tuning one), it's logged as an Open Question for Initiative 5
planning, not patched here.

## Risk

Low — flips and tuning over already-shipped, already-tested behavior. The one honest risk: the
before/after might show the contract helps less than hoped. That's an acceptable outcome — it's
*measured*, the escape hatches exist, and a weak result redirects effort (maybe more structural
enforcement, less prompt) rather than shipping a vibe. Recording a null result honestly is part of
the acceptance criteria.

## Acceptance criteria

- [ ] Four-way sweep recorded; intent-without-act defection before/after is the headline number.
- [ ] Tool-fire rate on tool-warranted turns improved or explained.
- [ ] Humanity violations still 0; latency parity held; leak not worsened.
- [ ] Defaults flipped on with `=0` escape hatches, each verified.
- [ ] LD #14 landed in REWRITE_CONTEXT; Open Q #9 resolved; roadmap + orient skill updated.
- [ ] Initiative 4 acceptance criteria (README) all checked.

## Notes for Initiative 5 (don't foreclose)

The `decision` trace surface taxonomy (this initiative, zero-LLM) plus the off-hot-path
`complete()` summarizer lane (already shipped for dream, unused by I4) are exactly what a
proactive/self-continuation **gate** would reuse — Initiative 5 inherits a measurement substrate
and a logging shape, and adds the one thing I4 deliberately didn't build: the *trigger* judgment
(the decision with no turn to ride). That is the legitimate gate, built then, with its first real
consumer — as Python's own spec said it should have been.
