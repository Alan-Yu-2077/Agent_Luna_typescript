# v0.8.3 — `recall` tool + tool-trigger checklist

> **Status: PLANNED.** Initiative: Action Integrity Rails (Order 4, version 4/5). Risk: **Low**.
> Depends: v0.8.1 (L1 contract to host the checklist). Flag: none (always-mounted per LD #10);
> behavior measured via v0.8.0 traces.

## Goal

Resolve Open Q #9 — give Luna an **agentic** memory search to complement the automatic injection
shipped in v0.4.x. She decides "让我想想" and queries her own memory. Pairs with a tool-trigger
checklist line in the L1 contract so "该调不调" has both a capability to reach for and a reasoning
prompt to reach for it.

## What ships

- **Protocol** — `ToolName` += `'recall'`
  ([`tools.ts`](../../../packages/protocol/src/tools.ts)). Flat root-object input (v0.5.2 gateway
  rule): `{ query: z.string().min(1), scope?: z.enum(['facts','timeline','both']),
  limit?: z.number().int().min(1).max(10) }`. Output: ranked hits
  `{ id, text, kind, score, when_ms? }[]` — the same shape `retrieve()` already produces.
- **Tool** — `packages/server/src/tools/builtin/recall.ts` via `defineTool`. `execute` calls the
  existing `retrieve(sessionId, query)`
  ([`memory/recall/recall.ts`](../../../packages/server/src/memory/recall/recall.ts)) — **reuses
  the shipped hybrid recall** (sqlite-vec + CJK lexical + recency), no new retrieval code.
  `concurrency: 'safe-parallel'` (read-only), small `timeoutMs`, `summarize` = hit count + top
  snippet. Soft-deleted facts excluded (the store already does this).
- **Registry** — add to **both** `builtinRegistry` and `messageRegistry`
  ([`registry.ts`](../../../packages/server/src/tools/registry.ts)); always mounted (LD #10), no
  flag. The wire-schema regression test (already iterating the registry) covers it for free.
- **L1 contract checklist line** (v0.8.1 block gains one item, behind the same
  `LUNA_L1_CONTRACT`): "Before you answer, a quick pass in thinking: does she reference something
  you feel you should already know but don't have in front of you? **Recall it first.** Did a
  durable fact just appear? Save it. Are you asserting from a hazy impression? Check." The recall
  clause only appears now that the tool exists (don't point at a missing tool — the v0.8.1 note).
- **Decision visibility**: because recall is a normal tool, its call/no-call is already in the
  trace tree; the v0.8.0 detector taxonomy can later add a `'no_recall'` surface if measurement
  shows under-recall (not built speculatively here).

## Tests

- `recall` input schema: flat root object (wire-schema regression test passes it); query required;
  scope/limit optional with bounds.
- `execute` returns ranked hits from `retrieve()`; respects `limit`; excludes soft-deleted; empty
  query rejected; no memory db configured → structured err (not a throw), mirroring `remember`.
- Registry: present in both registries; `toolsToAnthropicFormat` emits a flat object schema for it.
- L1 checklist line present only when `LUNA_L1_CONTRACT=1` and references `recall`; system prompt
  still byte-stable across no-change turns (snapshot test extended).
- Real-LLM smoke: a question about a previously-stored fact, asked with zero-shared-keywords
  paraphrase, prompts a `recall` call that surfaces the row (reuses the v0.4.3 paraphrase-hit
  evidence path).

## What this version explicitly does NOT include

No change to automatic injection — recall is the **complement**, not a replacement; both coexist
(injection is the floor, recall is the reach). No `no_recall` defection surface yet (measure
first, v0.9.0). No multi-hop / iterative recall (a single query per call; the model can call again
if needed — interleaved tool-use already allows it).

## Risk

Low — additive tool over already-shipped retrieval. Only watch item: recall-spam (calling it every
turn). That's a *measured* concern, not a guessed one — the L1 wording stays gentle and v0.9.0
checks call frequency against value; tune wording if it over-fires.

## Acceptance criteria

- [ ] `recall` mounts always, callable by the model, returns hybrid-recall hits.
- [ ] Its schema is a flat root object (regression test green).
- [ ] L1 checklist references recall only when the contract flag is on; cache invariant holds.
- [ ] Real-LLM smoke: paraphrased question → `recall` call → correct row surfaced.
- [ ] Automatic injection unchanged (coexistence verified).

## Notes for v0.9.0 (don't foreclose)

Keep recall's trace shape uniform with other tools so the v0.9.0 sweep can count call frequency and
hit-usefulness without special-casing. If under-/over-recall shows up, the fix is L1 wording +
maybe a `'no_recall'` audit surface — both cheap, both deferred to the measured decision.
