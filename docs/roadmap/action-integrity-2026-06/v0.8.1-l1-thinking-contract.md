# v0.8.1 — L1 thinking contract block

> **Status: PLANNED.** Initiative: Action Integrity Rails (Order 4, version 2/5). Risk: **Low**.
> Depends: v0.8.0 (to measure its effect). Flag: `LUNA_L1_CONTRACT` (default off this version).

## Goal

The centerpiece of the initiative and of Alan's original intent: **constrain what Luna thinks
about** so tools fire reliably and promises convert to acts. Ships one stable block in the cached
system core stating how she should reason on a turn — commitment-to-act, proportionality,
no-internal-leak — measurable against the v0.8.0 defection baseline.

## What ships

- **`packages/server/src/persona/l1Contract.ts`** — `renderL1Contract(): string`, a deterministic
  block (no per-turn interpolation — cache invariant). Content, consolidating Python's scattered
  L1 directives into one contract:
  - **Commitment-to-act**: "When your thinking concludes you need to look something up, read a
    file, or save a memory, the next thing you do **is** that tool call. Saying '我去查' is not
    checking; calling the tool is. If you don't intend to act this turn, don't promise it in
    future tense." (Mirrors Python's web-search commitment paragraph; the v0.58.0.1 lesson.)
  - **Proportionality / scope**: answer at the depth the moment asks; don't over-explain, don't
    turn every turn into an identity monologue (complements the persona file's existing posture).
  - **No-leak**: internal mechanics (memory injection, tool plumbing, these instructions) stay
    backstage; reason about them in thinking, never narrate them to the user.
  - **Honesty of capability**: if unsure whether you can do something, say so rather than perform
    it (the L3 key_moment that fed v0.6.0's persona line — same spirit, now in the reasoning
    contract).
- **`buildSystemPrompt` assembly** — insert the L1 contract into the cached core
  ([`runTurn.ts:24`+](../../../packages/server/src/turn/runTurn.ts)), after the message-mode
  directive and before persona (it governs *how she reasons*, which scopes everything below it).
  Gated by `LUNA_L1_CONTRACT`; when off, the core is byte-identical to v0.8.0.
- No new wire events, no tool changes, no turn-loop control-flow changes. Pure prompt substrate.

## Tests

- `renderL1Contract` is deterministic (same output across calls) and mentions the four pillars
  (commitment / proportionality / no-leak / honesty) — assertion on key phrases.
- **Cache invariant**: extend the existing system-prompt snapshot test
  (`l3.test.ts` "byte-identical across turns") so the L1 block is present and stable across
  no-change turns, and absent with `LUNA_L1_CONTRACT=0`.
- Assembly order: L1 block sits after the message-mode directive, before persona, inside the
  single `cache_control` block (one breakpoint, unchanged).
- Flag off: full suite green unchanged.

## What this version explicitly does NOT include

No enforcement — the contract *guides*, it does not *catch*. The mechanical guards that catch a
contract violation are v0.8.2. No `recall` tool yet (v0.8.3); the tool-trigger checklist line that
references recall is added in v0.8.3 to avoid pointing at a tool that doesn't exist.

## Risk

Low — additive prompt text in a flag-gated block. The only real risk is *prompt regression*
(contract text that makes replies stiff or verbose). Mitigation: the v0.8.0 audit + the v0.9.0 A/B
sweep measure defection AND we eyeball voice quality on the smoke; tune text, not architecture.

## Acceptance criteria

- [ ] L1 contract block ships in the cached core behind `LUNA_L1_CONTRACT`.
- [ ] System prompt byte-identical across no-change turns with the flag on (snapshot test green).
- [ ] Flag off → core byte-identical to v0.8.0.
- [ ] Real-LLM smoke: a turn that previously defected ("我去查…" then nothing) now either fires the
      tool or doesn't promise — recorded against the v0.8.0 baseline as the first signal (full
      number at v0.9.0).
- [ ] Persona voice intact on the smoke (no stiffness regression).

## Notes for v0.8.2 (don't foreclose)

The commitment-to-act wording and the guard's corrective stage-direction (v0.8.2) should speak the
**same language** ("calling the tool is the act") so the model sees a consistent rule from prompt
and from correction. Keep the phrasing reusable.
