# v0.8.0 — Decision traces + async defection audit (the instrument, first)

> **Status: PLANNED.** Initiative: Action Integrity Rails (Order 4, version 1/5). Risk: **Low**.
> Depends: nothing. Flag: `LUNA_DECISION_AUDIT` (default off this version, on at v0.9.0).

## Goal

Build the measurement instrument **before** the behavior changes, so every later version's effect
is a recorded before/after number rather than a guess. Ships a typed `decision` trace event, an
**off-hot-path** audit that detects "thinking promised an act, no act followed," and the
`/_trace` rendering that interleaves decision rows into the per-turn tree. With the flag off this
is pure addition: nothing about the turn changes, but turning it on records a defection baseline
on current v0.7.0 behavior.

## What ships

- **Protocol** — extend `TraceEvent`
  ([`trace.ts:44`](../../../packages/protocol/src/trace.ts), a
  `z.discriminatedUnion('kind', …)`) with a `DecisionTraceEvent`:
  `{ schema_v, kind: 'decision', trace_id, turn_id, session_id, t_ms, surface: string,
  decision: string, reason: string, evidence?: Record<string, unknown> }`. Mirrors the existing
  variants' field discipline; `surface` is the Python audit's surface tag (e.g.
  `'intent_no_act'`), `decision`/`reason` are the override-not-depend record.
- **Audit module** — `packages/server/src/turn/integrity/defectionAudit.ts`. **Zero-LLM by
  design** (2026-06-13 audit amendment: an earlier draft had a summarizer-lane `complete()` call
  to narrate the `reason` — cut; Python's own audit just logs the evidence tail, and ours does
  too. The initiative costs zero API until the v0.9.0 sweep):
  - `detectDefection(messageTexts: string[], lastIsFinal: boolean | null, thinking: string,
    calledToolNames: string[]): { defected: boolean; matched?: string;
    kind?: 'is_final_promise' | 'message_intent' | 'thinking_intent' }` — pure, synchronous.
    Three detectors, in confidence order:
    1. **Structural (`is_final_promise`)**: the turn's last delivered `message` had
       `is_final:false` yet the turn ended — mechanically certain (no dictionary).
    2. **Message-text heuristic (`message_intent`)**: a **delivered message text** matches a
       promise pattern (CJK + English: 「我去查 / 让我看看 / 我先读 / 马上查 / 我来记一下」,
       `I'll search / let me check / I'll look up`) AND no non-`message` tool fired this turn.
       This is the user-facing lie itself, matched on **exact** text (`messageTexts` is
       verbatim). **Primary heuristic signal — this is a deliberate divergence from Python**,
       which matched on raw thinking: our thinking is `display:'summarized'` and may paraphrase
       or drop intent phrasing, so it cannot be the load-bearing signal on this stack.
    3. **Thinking heuristic (`thinking_intent`), audit-only tier**: same dictionary over the
       summarized thinking. Counted in traces for completeness, but **low-confidence by
       definition** — v0.8.2 must never drive a corrective retry from this kind.
  - `runDefectionAudit(state)` — **synchronous pure detection + one trace write**, scheduled
    fire-and-forget after `turn.result` (same discipline as `maybeFold`). `reason` = the matched
    text ± surrounding evidence tail; `evidence` carries `{ matched, kind, called_tools }`.
    Guarded by `LUNA_DECISION_AUDIT`.
- **runTurn wiring** — in the `finalize` tail, after the existing
  `flushTrace`/`maybeFold` ([`runTurn.ts`](../../../packages/server/src/turn/runTurn.ts) finalize
  block), schedule `void runDefectionAudit(...)` when the flag is on. Never awaited; never affects
  the turn's `turn.result` (same fire-and-forget discipline as `maybeFold`).
- **Trace viewer** — `/_trace` renders `decision` rows inline in the turn tree (distinct glyph,
  `surface · decision`, reason on expand). The "decision replay tree" is the existing per-turn
  view gaining these rows.

## Tests

- `detectDefection`: `is_final:false`-then-end → `kind:'is_final_promise'`; delivered message
  "我马上去查一下" + no tool call → `kind:'message_intent', matched`; message promises + tool
  DID fire → not defected; clean turn → not defected; promise only in summarized thinking →
  `kind:'thinking_intent'` (audit tier, distinguishable from the retry-eligible kinds);
  promise phrase split across two bubbles → each bubble matched independently (no cross-bubble
  concatenation surprises).
- `DecisionTraceEvent` parses/rejects at the protocol boundary; round-trips through the trace
  store; appears in a `/_trace` turn render.
- Flag off: full existing suite green unchanged; no `decision` rows written.
- Fire-and-forget discipline: a throwing detector (forced) does not affect `turn.result`
  (override-not-depend); no LLM call exists anywhere in the audit path (assert no
  `complete()` invocation).

## What this version explicitly does NOT include

No behavior change to the turn (no L1 contract yet — v0.8.1; no guards/retries — v0.8.2). The
audit **observes**, it does not correct. No tuning of the intent dictionary beyond a starter set
(that's v0.8.2/v0.9.0 against the baseline this version records).

## Risk

Low — additive, flag-gated, off the hot path, zero LLM. Only subtlety: the dictionary will be
imperfect on day one; that is fine *here* because this version only counts (no behavior change) —
the recorded false-positive/negative texture is exactly the v0.8.2 tuning input.

## Acceptance criteria

- [ ] `decision` trace variant lands in the protocol union and renders at `/_trace`.
- [ ] `detectDefection` is pure, synchronous, zero-LLM, and covers all three detector kinds with
      tests (message_intent primary; thinking_intent marked audit-only).
- [ ] With `LUNA_DECISION_AUDIT=1`, a scripted delivered-message "我去查" + no-call turn produces
      exactly one `decision` trace; the turn result is byte-identical to flag-off.
- [ ] Audit runs off the hot path (not awaited) and cannot fail a turn.
- [ ] Full suite green; real-LLM smoke records at least one genuine defection sample for the
      v0.8.1 baseline.

## Notes for v0.8.1 (don't foreclose)

The detector's `kind`/`surface` taxonomy should be open enough that v0.8.3's recall-no-recall and
any Initiative 5 gate can add surfaces without a schema change. Keep `evidence` a free
`Record<string, unknown>` for exactly this.
