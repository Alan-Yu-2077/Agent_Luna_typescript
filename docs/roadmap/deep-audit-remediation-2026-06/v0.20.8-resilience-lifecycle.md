# v0.20.8 — Resilience & lifecycle

> **Status: PLANNED.** Initiative: Deep-Audit Remediation (Order 13, version 9/10). Risk: **Low–Medium**.
> Depends: nothing (reuses v0.20.2's abort-threading pattern + v0.20.3's `AbortController` seam if
> present). Flag: none.

## Goal
Close the robustness/lifecycle gaps that bite under disconnects, idle, transient backend failures, and
trace hiccups — none are crashes or data loss, but each is a real degraded-resilience defect. Server
side: off-path trace flush can abort a whole dream/proactive pass; a disconnected client's turn burns
tokens; the continuation timer blocks shutdown and wastes an LLM call; the wakeGate anti-repeat context
is dead. Client side: no keepalive ping (idle drop + flicker), reconnect backoff resets on flap, TTS
warm-up can hang the boot gate, and the TTS failure latch never recovers.

## What ships
**Server resilience.**
- Guard off-path trace flush: wrap `flushTrace` in `dream/cycle.ts:123` and `proactive/scheduler.ts:81`
  in try/catch (mirror `runTurn.ts:840-844`) — better, push the guard **inside** `store.flush()` /
  `flushTrace()` (`trace/instrument.ts:25-28`) so all current and future callers inherit never-throw.
  (A transient SQLite write must not kill a dream consolidation pass.)
- Turn abort on disconnect: add `signal?: AbortSignal` to `ProviderRequest` (`provider/types.ts:29-33`),
  pass it into `AnthropicProvider.chatStream` → `messages.stream({signal})` (`anthropic.ts:70-77`). Use a
  **per-reactive-turn `AbortController`** (created when a `chat.send` turn starts — **not** a long-lived
  per-`Session` one), and abort it in `handleClose` (`ws.ts:98-105`) so a disconnected client's turn stops
  between/within rounds instead of running to `MAX_TOOL_ITERATIONS=8`. **Explicitly exclude socket-less
  turns**: proactive (`proactiveTurn.ts`) and continuation turns have no client socket and are
  fire-and-forget by design (LD #15) — they must run to completion and must **never** be wired to this
  signal, or a disconnect would regress proactive agency. (The continuation-on-disconnect cancellation
  below is separate and intentional: it cancels a *not-yet-fired* timer, not an in-flight LLM call.)
- Continuation lifecycle (`proactive/continuation.ts:49-57`): `.unref()` the `setTimeout` (mirror
  `scheduler.ts:44`) so it never blocks shutdown, and either gate `fireContinuation` on
  `activeSockets` being non-empty or track + clear the timer in `handleClose` — so a disconnect doesn't
  burn a proactive LLM call no one sees.
- wakeGate anti-repeat (`proactive/scheduler.ts:121`): populate `recentProactive` from the last N
  `proactive:` L2 turns' `assistant_text` (`sessionStore.ts` `listL2`/`listRecentL2`, filter by the
  `proactive:` turn_id prefix) before `buildWakeContext`, so the "do not repeat these" block
  (`wakeGate.ts:58-61`) is no longer dead and Luna stops repeating openers.

**Client resilience (`packages/web/src`).**
- Keepalive ping (`wsClient.ts`): `setInterval(~30s)` while OPEN emitting `{type:'ping', seq}` (the
  server already replies `pong`, `ws.ts:137`; the controller already consumes it, `controller.ts:29-30`);
  start on open, clear on close. Stops the ~120s idle drop + history-reflicker (and the silent
  proactive-loss in the dead window).
- Reconnect stability window (`wsClient.ts:35-39,51-60`): only reset `attempt=0` after the socket has
  stayed open for a window (e.g. a 5s timer set on `open`, cleared on `close`) so a flapping server
  actually backs off instead of hammering at the ~1.5s base.
- `warmUpTts` deadline (`ui/bootGate.ts:64-121`): add an overall `setTimeout`-driven `finish('failed')`
  (e.g. 60–120s) and/or AbortController-per-fetch timeouts on the `/health` + `/speak` calls, so a
  stalled sidecar can't hang the boot gate; makes the documented "failed on timeout" true.
- TTS failure-latch self-heal (`audio/webAudioSink.ts:68-71`): replace the permanent `disabled=true`
  with a `mutedUntil` timestamp (re-attempt after ~60s, reset `fails` on success), and/or treat
  502/504 like 503 (retryable). A brief sidecar restart no longer mutes the whole session.

## Tests
- `dream.test.ts` / `scheduler.test.ts`: a throwing `flushTrace` does **not** abort the dream cycle /
  suppress the proactive turn (inject a flush that throws; assert subsequent steps still run).
- `ws.test.ts` / `session`: a disconnect mid-turn aborts the in-flight turn (assert the provider stream
  is signalled / the turn stops); a reconnect rehydrates cleanly.
- `continuation.test.ts`: the timer is `.unref()`'d; a disconnect in the window cancels the continuation
  (no `runProactiveTurn` fired).
- `scheduler.test.ts` / `wakeGate.test.ts`: `recentProactive` is populated from prior proactive turns
  (the de-dup block renders in production context).
- `wsClient.test.ts`: keepalive ping emitted while OPEN; backoff escalates against a flap (no reset
  without the stability window).
- `webAudioSink`/`bootGate` unit: latch self-heals after the window; `warmUpTts` resolves `'failed'`
  on a stalled fetch within the deadline.

## What this version explicitly does NOT include
- The tool-spawner process-tree kill / grep abort — **v0.20.2** (different layer).
- Any change to the wake *decision* logic or cadence governor (LD #15 semantics unchanged) — only the
  anti-repeat context is wired and the timer lifecycle fixed.

## Risk
- **Low–Medium**, but this is the **largest bucket** (8 small fixes across server + client). Mitigation:
  each fix is independently testable and revertible; land them as separate commits within the version;
  the turn-abort change touches the provider interface — keep `signal` optional so non-aborting callers
  (tests, dream `complete()`) are unaffected. Push the trace-flush guard into `store.flush()` once so it
  can't be forgotten at a future call site.

## Acceptance criteria
- [ ] A throwing trace flush never aborts a dream/proactive pass.
- [ ] A client disconnect aborts the in-flight turn (no run-to-completion token burn).
- [ ] The continuation timer is `.unref()`'d and cancelled on disconnect; wakeGate sees recent openers.
- [ ] Client: keepalive ping prevents idle drop; backoff escalates on flap; `warmUpTts` can't hang;
      TTS latch recovers within a session.
- [ ] Full suite green; `tsc` clean ×3.

## Notes for v0.20.9 (don't foreclose)
- The optional `ProviderRequest.signal` added here is a wire-internal interface change (not a protocol
  event); v0.20.9's contract tightening is in `@luna/protocol` and is independent — don't conflate them.
