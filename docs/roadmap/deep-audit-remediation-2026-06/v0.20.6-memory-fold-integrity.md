# v0.20.6 — Memory fold & summarization integrity

> **Status: PLANNED.** Initiative: Deep-Audit Remediation (Order 13, version 7/10). Risk: **Medium**.
> Depends: nothing. Flag: none (fixes within shipped memory/dream behavior).

## Goal
Close three confirmed integrity gaps on the memory-truth path: `loadSession` silently **drops the
newest turns** past 10k rows (ASC `LIMIT`), an empty/truncated `complete()` digest **overwrites the
rolling summary with `''`** (silently shrinking active context), and the dream salience step maps scores
to turns **positionally with no length check**, permanently mis-rating turns on a shifted LLM response.

## What ships
**Don't drop the newest turns (`memory/sessionStore.ts:34,122-127`).**
- `loadSession` rebuilds history from `listL2(id)`, which is `ORDER BY t_ms ASC … LIMIT 10000` — past
  10k turns this returns the OLDEST 10k and discards the most recent. Fix: **remove the cap** for this
  once-per-boot load (ASC order, oldest→newest, is exactly the frame `buildActiveContext`/`planFold`
  index against). **Do NOT** use the `ORDER BY … DESC … LIMIT`+reverse variant — a `LIMIT` truncation
  that lops the oldest rows shifts the **absolute `window_low_water` offset** that `planFold`
  (`l1Window.ts:77`) and `buildActiveContext` (`l1Window.ts:49`) index against, corrupting the fold
  boundary. If a row cap ever becomes necessary it must preserve that absolute offset, not reorder.
  Log when (if ever) truncation occurs.

**Guard empty/truncated digests (`memory/l1Window.ts:144` + `provider/anthropic.ts:48-59`).**
- In `maybeFold`, reject an empty digest before `commitFold`: `if (!digest) return false` (mirror the
  existing guard in `dream/llm.ts:51`), so an empty `result.text` never overwrites `rolling_summary`
  and never advances `window_low_water`.
- Drop `thinking:{type:'adaptive'}` from `complete()` — extended thinking is pure overhead/risk for a
  summarization call and competes with the 1024-token output budget (it counts toward `max_tokens`).
  Optionally surface `stop_reason` in `CompleteResult` (`provider/types.ts:41-44`) so callers can detect
  truncation; the empty-guard above is the load-bearing fix either way.

**Dream salience length check (`dream/cycle.ts:142-148` + `dream/llm.ts:91-93`).**
- Reject the whole patch when `patch.scores.length !== unrated.length` (return `['failed','score/turn
  count mismatch']`) so a misaligned LLM response is retried next cycle rather than written. (A mid-list
  drop/insert otherwise shifts every later turn onto the wrong neighbour's score, written permanently by
  `setImportance` `sessionStore.ts:152-155`.) Optionally have the prompt echo each turn's ordinal and
  match on index for robustness.

## Tests
- `sessionStore.test.ts` / `l1Window.test.ts`: seed `> cap` L2 turns, assert `loadSession` keeps the
  **newest** turns (the verbatim tail), not the oldest; assert a truncation log fires.
- `l1Window.test.ts`: a `complete()` that returns empty text (drive via `MockProvider` returning `''`)
  does **not** overwrite `rolling_summary` and does **not** advance `window_low_water`; the prior summary
  survives. (Today `MockProvider`'s default returns `'[mock summary]'`, so add an empty-responder case.)
- `dream.test.ts`: a `rate_salience` patch with `scores.length !== unrated.length` is rejected (no
  `setImportance` writes); the aligned case still rates correctly.

## What this version explicitly does NOT include
- The fired-and-forgotten `maybeFold`-overlaps-dream race (`runTurn.ts:845`) — CAS already makes it
  safe; only a clarifying comment is warranted, folded into **v0.20.8** (resilience).
- `restore(n)` undo semantics — **v0.20.9** / Open Question #1.
- Any change to the fold *trigger* timing or the GA recall ranking.

## Risk
- **Medium** (the empty-digest guard is the high-value fix; the 10k-turn drop is a *remote* scenario for
  a single user, so its practical severity is closer to low–medium). Touches `commitFold`/`maybeFold`
  (the CAS fold path) and `loadSession` (the boot truth path). Mitigation: the empty-guard is a strict
  superset of current behavior (only blocks a bad write); the `loadSession` **un-cap** must preserve the
  in-memory `history` shape `buildActiveContext` slices — pin with the `>cap` test. Keep the existing
  CAS-overlap test (`l1Window.test.ts:135-155`) green.

## Acceptance criteria
- [ ] Past 10k turns, reload preserves the newest turns (regression red→green).
- [ ] An empty/truncated digest never overwrites the rolling summary (regression red→green).
- [ ] A length-mismatched salience patch is rejected, not written.
- [ ] `complete()` no longer forces adaptive thinking for summarization; full suite green; `tsc` clean.

## Notes for v0.20.7 (don't foreclose)
- This version establishes the "reject-bad-write rather than commit-degraded" discipline; v0.20.7's
  atomic-write + fuzzy-uniqueness fixes follow the same "fail safe rather than corrupt" principle on the
  edit path.
