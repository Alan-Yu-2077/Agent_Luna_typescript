# v0.20.7 — Edit & code-map correctness

> **Status: PLANNED.** Initiative: Deep-Audit Remediation (Order 13, version 8/10). Risk: **Low–Medium**.
> Depends: nothing. Flag: none (correctness fixes to existing code tools).

## Goal
Three confirmed code-tool defects: file writes are **not crash-atomic** (truncate-in-place can corrupt
the user's source on a mid-write crash, despite `multi_edit` advertising "all-or-nothing"); the fuzzy
matcher **undercounts to `count:1`**, defeating the self-edit (and edit/multi_edit) uniqueness guard so
an ambiguous match silently edits the first region; and `isExported` mislabels **every method of an
exported class** as exported, skewing the repo map's ranking and output.

## What ships
**Atomic writes (`tools/builtin/edit.ts:139`, `multi_edit.ts:157`, `write_file.ts:138`).**
- Replace the single `await Bun.write(resolved, toWrite)` with write-to-sibling-temp-then-rename:
  `Bun.write(`${resolved}.luna-tmp-${pid}`, toWrite)` then `rename(tmp, resolved)` (atomic within a
  filesystem; the same-directory temp is required so the rename stays intra-fs). Clean up the temp on
  failure. Keep the existing in-memory all-or-nothing guard; this only makes the **disk landing** atomic.
  These are the only three `Bun.write` call sites in the server.

**Fuzzy multi-window uniqueness (`tools/editCore.ts:55-58` `findEditMatch`).**
- When the fuzzy path produces more than one **distinct** candidate window, report `count = number of
  distinct matching windows` (dedup-then-count), not `countOccurrences(content, candidates[0])`. That
  makes the `> 1` uniqueness guard fire for ambiguous fuzzy matches in `selfEdit.ts:53` **and** the live
  `edit.ts:125`/`multi_edit` guards (broader blast radius — a real silent partial write there). The
  author already deduped identical windows (`editCore.ts:56` comment); this extends it to distinct
  whitespace windows. `fuzzed:true` surfacing stays.

**`isExported` class methods (`code/symbols.ts:63-73`).**
- Add `class_body` (and `class_declaration`) to the stop-set, or special-case `method_definition` as
  never-class-exported, so a method of an exported class is `exported=false`. Restores the correct
  `repoMap.ts:188` ×1.5 ranking boost and the `exported` flag in `repo_map`/`find_symbol` output.

## Tests
- New `editCore.test.ts` (also chips at v0.20.9 test-debt): a fuzzy `old_string` that matches two
  distinct-indentation regions reports `count >= 2` (uniqueness guard fires) — `{found, count, fuzzed}`
  asserted; the identical-window dedup still collapses to the true count.
- `selfEdit.test.ts`: an ambiguous fuzzy match is **rejected** with the "not unique — add context" error
  rather than silently splicing the first region.
- `edit.test.ts` / `multi_edit.test.ts`: a crash-atomicity-shaped test where possible (assert a temp file
  is used + cleaned, and a write failure leaves the original intact); at minimum assert the temp+rename
  path is taken.
- `repoMap.test.ts`: a method inside an exported class asserts `exported=false`; a top-level export still
  `exported=true`.

## What this version explicitly does NOT include
- The fuzzy `replace_all` + multi-indentation narrow behavioral limitation beyond the count fix — the
  count is honest and `fuzzed:true` is surfaced (this was the refuted "count divergence" finding; only
  the uniqueness-count is a real bug, fixed here).
- Any change to the str_replace edit semantics / diff rendering.

## Risk
- **Low–Medium.** The temp+rename change alters the write mechanism on the user's source files —
  mitigation: same-directory temp (intra-fs rename), cleanup-on-failure, and keep the in-memory guard;
  test the failure path. The `isExported` stop-set change is narrow and well-pinned by the probe in the
  audit.

## Acceptance criteria
- [ ] `edit`/`multi_edit`/`write_file` land via temp+rename; a simulated mid-write failure leaves the

- [ ] An ambiguous fuzzy match is rejected by the uniqueness guard instead of editing the first region.
- [ ] Methods of an exported class report `exported=false`; repo-map ranking/output corrected.
- [ ] Full suite green; `tsc` clean ×3.
