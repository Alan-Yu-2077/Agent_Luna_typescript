# v0.20.2 — Subprocess & resource cleanup

> **Status: PLANNED.** Initiative: Deep-Audit Remediation (Order 13, version 3/10). Risk: **Medium**.
> Depends: nothing (v0.20.0 may have added the argv spawner seam; reuse it if present). Flag: none.

## Goal
Stop the process/handle leaks the audit confirmed on the code-agent path: the documented "kills the
process TREE" guarantee is a no-op (grandchildren survive every timeout/abort), `grep`/`find_symbol`/
`repo_map` never honor the abort signal (orphaned `rg` on timeout), tree-sitter `Parser` objects leak
WASM heap on every parsed file, and the SIGKILL-escalation timers are never cleared on clean exit.

## What ships
**Make the process-tree kill real (`tools/shellCore.ts:49-87`).**
- Spawn the child in its **own process group** so `process.kill(-pgid, sig)` actually reaches the tree
  — either run the command under `setsid`/a `setpgid` wrapper inside the spawner (not in the user's
  command text, which would collide with the deny-regex), or, if Bun cannot start a new group, enumerate
  descendants (`pgrep -P` walk) and signal each. Verify a backgrounded grandchild (`sleep 30 & wait`)
  is actually killed.
- Clear the SIGKILL-escalation `setTimeout`s (`:74,86`) in the `finally` (`:101-104`) alongside `timer`
  so a clean early exit leaves no dangling escalation (they are `.unref()`'d so harmless, but tidy).

**Thread abort into the rg-backed tools.**
- `grep.execute` (`tools/builtin/grep.ts:180`): destructure `ctx`, pass `ctx.abortSignal` into
  `runGrep`/`ripgrepRunner` (`:51-95`) → `Bun.spawn(..., {signal})` (or kill `proc` on abort), and
  check `signal.aborted` between files in the jsRunner loop (`:119`). Mirror `shell.ts:145`.
- `find_symbol.ts:54` and `repo_map.ts:57`: accept `ctx` and pass the signal into `locateSymbol`/
  `buildRepoMap` so their rg-backed work cancels too.

**Free the tree-sitter Parser (`code/symbols.ts:148-164`, `code/treeSitter.ts:108`).**
- Pool one `Parser` per grammar alongside `langCache` (`treeSitter.ts:68-90`), calling `setLanguage`
  once and reusing it across the synchronous parse loop — removes both the leak and the per-file
  constructor cost. Add the pooled parser to `resetTreeSitterForTests()` (`:118-122`) teardown.
- Failing that, the minimal fix: `parser.delete()` in `parseWithLoaded`'s `finally` after `tree.delete()`.

## Tests
- `shellCore.test.ts` (currently only covers `capOutput`/`clampTimeout`): a spawned command that
  backgrounds a grandchild and exceeds its timeout — assert the grandchild PID is dead after the kill
  (poll `process.kill(pid,0)` → throws ESRCH). This is the regression the false invariant hid.
- `grep.test.ts`: an aborted grep (pre-aborted `AbortSignal` or a timeout) returns promptly and leaves
  no live `rg`; assert the generator stops and (where observable) the subprocess is gone.
- `code/symbols` or `repoMap.test.ts`: parsing N files reuses a pooled parser (assert a single parser
  instance / no growth) — or at minimum that `resetTreeSitterForTests` clears it.

## What this version explicitly does NOT include
- Turn-level abort on client disconnect (the `provider.chatStream` AbortSignal) — that is **v0.20.8**
  (different layer: session/WS lifecycle, not the tool spawner).
- Any deny-regex / firewall change — v0.20.0.

## Risk
- **Medium.** Process-group handling is platform-sensitive (macOS/BSD vs Linux). Mitigation: keep the
  spawner's external contract identical; gate the new group/kill logic behind the spawner so callers
  are unaffected; test the grandchild-kill on the dev macOS runtime explicitly. The pooled-parser change
  must stay correct for the *sequential* parse loop only (no concurrent parsing today — verified).

## Acceptance criteria
- [ ] A timed-out/aborted subprocess-spawning command leaves **no** surviving grandchild (regression).
- [ ] `grep`/`find_symbol`/`repo_map` cancel their rg work on abort; no orphaned `rg` after a timeout.
- [ ] tree-sitter parsing no longer allocates one `Parser` per file (pooled or deleted); reset teardown
      covers it; `repoMap`/`find_symbol` output unchanged.
- [ ] Full suite green; `tsc` clean ×3.

## Notes for v0.20.8 (don't foreclose)
- The abort-threading pattern established here (ctx.abortSignal → spawn/{signal}) is the same one
  v0.20.8 uses to thread an AbortSignal from the WS/session lifecycle into `provider.chatStream` — keep
  the `ProviderRequest`/spawner signal plumbing consistent.
