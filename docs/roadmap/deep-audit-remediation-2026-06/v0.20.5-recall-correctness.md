# v0.20.5 — Recall correctness

> **Status: PLANNED.** Initiative: Deep-Audit Remediation (Order 13, version 6/10). Risk: **Low–Medium**.
> Depends: nothing (v0.20.4 hardens `resolveTz` that `relativeLabel` uses; not required). Flag: none.

## Goal
Fix three confirmed recall defects: the agentic `recall` tool's `scope='timeline'` **silently drops
every diary hit** (diaries became first-class candidates in v0.17.1 but the filter still hard-codes
`l2`); the over-fetch-then-filter design **starves** facts/timeline when the global top-k×2 is dominated
by the other source; and a `LUNA_EMBEDDING_MODEL` dimension change silently turns cosine into **NaN**,
degrading recall to lexical-only with no error.

## What ships
**Diary belongs to the timeline (`tools/builtin/recall.ts:59-61`).**
- Change the scope filter to `scope === 'facts' ? h.source === 'l3' : h.source !== 'l3'` (so timeline =
  `l2` + `diary`; facts = `l3`). Update the `RecallHit.source` enum description and the `Input` scope
  doc (`recall.ts:15,21`).

**Stop the scope starvation (`tools/builtin/recall.ts:55-69`).**
- Push the scope into `retrieve()` so the `k` limit applies per-scope: add an optional `source` filter
  to `retrieve()` (`memory/recall/recall.ts:140-233`) that filters `collectCandidates` (`:96-132`)
  **before** ranking; OR, if keeping `retrieve()` source-agnostic, loop/raise the over-fetch factor in
  `recall.ts` until `filtered.length >= limit` or the candidate pool is exhausted. Prefer pushing scope
  into `retrieve()` (cleaner, bounds the work). The hot-path auto-injection (non-scoped) is unaffected.

**Embedding dimension guard (`memory/recall/embed.ts:57-67` + the cache key).**
- In `cosine`, assert `a.length === b.length`; on mismatch treat it as a cache miss (return `null`/skip),
  never produce `NaN`. (Also fixes the dim-*decrease* silent-partial-cosine direction, where the loop
  returns a finite-but-wrong value.)
- Fold model identity into the **embedding** cache key — but **NOT** by mutating the shared `contentHash`
  (`embed.ts:53-54`), which also keys L2/L3 row hashes: rewriting it to `contentHash(`${model}\n${text}`)`
  would force a permanent cache miss on **every existing row**, not just embeddings. Use a **separate
  embedding-cache keying fn** (hash `model\ntext` for the embedding lookup only), or validate via the
  stored `dim` column (`migrations/0005_embeddings.sql:4`, written but never read today). **Per Open
  Question #2**: default to **lazy re-embed on miss** (no migration); the `cosine` length-guard is the
  safety net for any pre-existing stale rows.

## Tests
- `recall.test.ts`: seed a diary candidate + assert it **survives** `scope='timeline'` (the current
  `:106-107` `every(h.source === 'l2')` assertion must be updated to allow `diary`); add a fact-survival
  test under source skew — seed many recent `l2` turns + a few highly-relevant `l3` facts, assert
  `scope='facts'` still returns those facts (the starvation regression).
- `embed`/`recall` unit: a query vector and a candidate vector of different lengths produce a **miss**
  (hit dropped cleanly), not a NaN-poisoned score; a same-content query under a changed model name does
  not reuse the old-dim cached vector.
- The prompt-cache pin (`recall.test.ts:193-238`) stays green (system block byte-identical across
  queries — recall content rides the uncached tail; unchanged here).

## What this version explicitly does NOT include
- Recall **ranking** changes (the GA `α·recency+β·importance+γ·relevance` blend stays) — only the scope
  filter, the over-fetch starvation, and the dimension safety.
- A cache-rebuild migration — deferred per Open Question #2 (lazy re-embed chosen).

## Risk
- **Low–Medium.** Pushing scope into `retrieve()` touches the shared retrieval path used by both the
  tool and (a different entry of) auto-injection — keep the default (no source filter) byte-identical so
  the hot path is unchanged; the source filter is opt-in from the tool only. The cache-key change
  invalidates content-only rows lazily — acceptable (one re-embed per row after a model swap, which is
  itself a rare operator action).

## Acceptance criteria
- [ ] `recall(scope='timeline')` returns diary hits; `scope='facts'` returns facts even under heavy
      recent-l2 skew (both regression-tested).
- [ ] A dim mismatch yields a clean miss, never `NaN`; recall does not silently fall back to lexical-only
      after a model swap.
- [ ] Cache invariant test green; hot-path auto-injection unchanged; full suite green; `tsc` clean.

## Notes for v0.20.6 (don't foreclose)
- The `retrieve()` source-filter parameter added here is orthogonal to v0.20.6's `loadSession` ordering
  fix; both touch `memory/`, keep their changes independent (recall ranking vs persistence ordering).
