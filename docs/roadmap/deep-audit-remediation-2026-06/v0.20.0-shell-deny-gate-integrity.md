## Goal
Close the worst safety-gate cluster from the deep audit: the verify tools (`typecheck`/`run_tests`/
`lint`) execute model-supplied paths through `/bin/zsh -lc` with `JSON.stringify`, which is both an
**arbitrary-command-injection** vector and a **clean bypass of the `shell` deny-regex** (these tools
never call `classifyShellCommand`). Plus the deny-regex itself is evadable, and the evaluator firewall
protects the deny-regex *data* but not the *enforcer* files. This version makes LD #10's deny-regex
the robust gate it is documented to be, and makes LD #14's firewall cover the code that enforces it.

## What ships
**Argv-spawn the verify tools (kills the injection class structurally).**
- Give `realSpawner`/`activeSpawner` (`tools/shellCore.ts:47,114-116`) an **argv path** that calls
  `Bun.spawn([...argv])` directly (no `/bin/zsh -lc`), and route `typecheck.ts:87-89`,
  `run_tests.ts:88`, `lint.ts:75-76` through it: `['bun','x','tsc','--noEmit','-p',input.path]`,
  `['bun','test',input.path]`, `['bun','x','prettier','--check',input.path]`. No string interpolation
  of `input.path` anywhere. **Also** gate the path itself with `resolveInWorkspace(input.path, 'execute')`
  before spawn â€” today **only `input.cwd` is gated, not `input.path`** (correcting the audit's own
  parenthetical), so the argv change alone would still leave `path` un-blocklisted (e.g.
  `run_tests({path:'~/.aws'})` â†’ `bun test ~/.aws`). Low-stakes for these read/check tools (no write, no
  secret-byte exfil on a normal path), but close it here.

**Broaden + normalize the deny-regex (`tools/shellDeny.ts`).**
- Add a `find â€¦ -delete` / `find â€¦ -exec rm` rule (the `-delete` form is currently un-gated and
  live-verified to delete).
- Broaden the curl|sh rule (`:34`): interpreter alternation `sh|bash|zsh|dash|python|python3|perl|
  ruby|node|php`, and drop the `[^|]*` anchoring so an intermediate pipe (`curl â€¦ | tee x | sh`)
  still matches.
- Normalize the command before matching: collapse empty-quote splices (`r""m`/`r''m` â†’ `rm`) so the
  `rm -rf` rule fires. (Do **not** bother with `$IFS` â€” verified inert under zsh; note it in a
  comment instead of adding a fragile rule.)
- Correct the overstated `:9-13` comment: "best-effort hard block over the known destructive forms",
  not "ALWAYS hard-blocks".

**Firewall the enforcer files (`tools/workspace.ts:108-125` `evaluatorFiles()`).**
- Add `tools/builtin/shell.ts`, `tools/shellCore.ts`, and `tools/builtin/run_tests.ts` (its
  `parseBunTestOutput` is what `save_skill` trusts for green/red) to the firewall set, so a
  propose-self-edit / edit cannot neuter the deny-regex caller, the spawner, or the verify parser.
