## Goal
The cleanup capstone: prune dead/stale wire-contract schemas and tighten two loose ones, fix the
developer-facing config drift (`.env.example`, `.prettierignore`), pay down the highest-value test debt
the audit named (the **untested SSRF DNS-pin transport** above all), and clear the cosmetic frontend
nits. None are runtime defects; all reduce future-drift and refactor risk. Initiative 13 ships ✅ here.

## What ships
**Protocol contract (`packages/protocol/src`).**
- Delete the dead, stale `SessionRow` (`memory.ts:13-19`) and `L2Turn` (`memory.ts:3-11`) — zero refs,
  drifted from the live `sessions` shape (which now carries `rolling_summary`/`window_low_water`/
  `proactive_*`). The server defines its own row types; remove the misleading parallel definitions.
- Tighten `Citation.url` (`events.ts:109-113`) with an **http/https `.refine()` only — NOT
  `z.string().url()`**. `outbound.ts` hard-`.parse()`s every emitted frame, and zod's `.url()` is
  *stricter* than the WHATWG `URL` constructor that `safeHttpHref` (`bubbles.ts`) uses to admit citations —
  so a citation URL the renderer accepts but `.url()` rejects would **throw inside `outbound` and fail
  `turn.result` delivery** for the whole turn. A scheme `.refine()` (or a drop-bad-url-at-emit fallback)
  encodes the untrusted-origin invariant without that outbound-throw failure mode.
- Make `ToolEvent.tool_name` (`tools.ts:62-79`) the `ToolName` enum (matching `ToolCall` + the real
  invariant), OR switch the consumers (`ws.ts:320,328`, `runTurn.ts:431,439`) from `.parse()` to
  `.safeParse()` with a graceful fallback. Prefer the former.

**Config / build.**
- Add the ~37 missing flags to `.env.example` (at least the behavior/security gates: `LUNA_BIND_HOST`,
  `LUNA_MEMORY_EMBEDDING`, `LUNA_MEMORY_INJECT`, `LUNA_RECALL_ASYNC`, `LUNA_CODE_WRITE`/`LUNA_SHELL`/
  `LUNA_SELF_EDIT`/`LUNA_SKILLS`/`LUNA_REPO_MAP`, `LUNA_PERSIST`, `LUNA_TRACE_RETENTION_TURNS`, the
  `LUNA_L1_*`/diary/dream tuning, and the `LUNA_TTS_*` dev vars) with defaults + one-line descriptions.
- Add `packages/web/public/` to `.prettierignore` so `prettier --write .` (`package.json:12`) stops
  reformatting the 206KB minified Cubism core + the 23 vendored model JSONs.

**Test debt (highest-value first).**
- **`safeFetch` DNS-pin** (`web/safeFetch.ts:252-304`): a real-socket test — stand up an `http.Server`
  on a local IP, drive `safeFetch` with the real `pinnedFetch` (no injected `fetchImpl`), assert it
  connects to the pinned IP and that a resolver returning a different IP than the pin cannot be reached;
  unit-test `pinnedLookup` for both `all:true`/`all:false` callback shapes. Then make the source comment
  + DEVELOPMENT.md "real-HTTPS smoke" claim true (or reword it). **This is the most security-critical
  untested path in the codebase** — a silent `pinnedLookup` regression reopens SSRF/rebinding.
- provider `chatStream` SSE→ProviderEvent mapping (`anthropic.ts`): a synthetic-stream test asserting
  the ProviderEvent sequence for text-only, interleaved thinking+tool_use, multiple concurrent tool
  blocks (index attribution), and multi-chunk `input_json_delta`; a `complete()` text-join + only-thinking
  (empty-text) case.
- `fsScan`/`readTracking`/`defineTool` sibling tests (some started in v0.20.1/v0.20.7) — `.gitignore`
  literal/glob/negation parsing, exact `maxEntries` boundary, symlink emit/descent, readTracking session
  isolation + canonical-key matching.

**Cosmetic frontend nits.**
- faceVm gaze gate (`faceVm.ts:200-210`): apply the `gazeActive` + `GAZE_KEYS` skip (mirroring `:608`)
  in the tick flush loop so the focusController owns the eyeballs whenever gaze-follow is on, regardless
  of emotion layer.
- toolLabels (`ui/toolLabels.ts:28-33`): exact-match the stripped chip text against `ToolName` /`CUTE`
  instead of `includes()` (fixes `recall_skill`→"recall", `propose_self_edit`/`multi_edit`→"edit"
  mislabels and the finish-summary substring splice), and add CUTE entries (or a passthrough) for the 9
  unlabeled tools.

**Decisions to settle here (Open Questions).**
- `restore(n)` (`coreMemory.ts:40-55`, OQ#1): has **no production caller** — default to **deleting** the
  unreachable method + its misleading comment, unless the owner wants a real undo surface (then implement
  the multi-step walk). Decide with the owner.
- `physicsPassthrough` dead data (`faceData.ts:18,80`): either reimplement the unconditional passthrough
  skip in tick/flush (Python `face-vm.js:814` parity) or delete the inert field from `EmotionDef` + all
