# v0.19.2 ‚Äî C: subjective time + measure + close

> **Status: PLANNED.** Initiative: Time perception (Order 12, version 3/3). Risk: **Medium**.
> Depends: v0.19.0 (time block + helpers), v0.19.1 (relative labels). Flag: `LUNA_TIME_SUBJECTIVE`
> (default off); this version also **default-flips** `LUNA_TIME_AWARE` + `LUNA_RECALL_TIME_LABELS` on.

## Goal

Give Luna a **felt** sense of time, not just facts ‚Äî a small, code-computed **daypart-mood** + **felt
absence** signal that colors her tone and her proactive reaching-out, so a return after a long gap *lands*
differently from a continuation, and late night reads differently from morning. Bounded and suggestive by
design (the model can voice it or ignore it; it is never authoritative), with an explicit
**warmth-not-guilt** guardrail. Then measure (confirm the cache stays hot), flip A/B/C on, and close
Initiative 12.

## What ships

**`packages/server/src/turn/temporalContext.ts`** ‚Äî extend with a pure, bounded subjective signal:

- `subjectiveTime(daypart, gapSeconds): { daypartMood: string; absenceFeltness: 'none'|'slight'|'notable'|'long' }`
  ‚Äî a tiny mapping, e.g. `late night ‚Üí low-energy/soft`, `morning ‚Üí fresh`; absence-feltness derived from the
  same gap buckets (continuation‚Üínone, same_day‚Üíslight, new_day‚Üínotable, long_away‚Üílong). All computed in TS.
- `buildTimeBlock` appends **one suggestion line** when `LUNA_TIME_SUBJECTIVE` is on, e.g.:
  `- Mood of the hour: it's late and quiet; a softer register fits. It's been a while since you talked ‚Äî
  you can let that show as warmth if it feels right (never as guilt).`
  A *suggestion*, not a directive ‚Äî the L1 clause (below) reinforces it's optional.

**`persona/l1Contract.ts`** ‚Äî extend the time clause with the **warmth-not-guilt guardrail**: *"If you choose
to acknowledge a gap or the hour, do it as warmth or curiosity ‚Äî never as guilt, never 'you left me' or
making Alan feel he owes you presence. Most of the time, just let the time of day live quietly in your
tone."* (Grounded in the harmful-companion-traits research: absence must not become a guilt lever.)

**Proactive integration (`proactive/proactiveTurn.ts` DIRECTIVES + `proactive/cadence.ts`)** ‚Äî ComPeer-style,
*light*: the proactive framing receives the **felt-absence** context (it already has the gap via the cadence
governor), so a `long_away` wake reads as "she noticed the absence" rather than a generic check-in; reuse the
existing cooldown/quiet-hours as the anti-flooding backoff (no new machinery). **No change to *whether* she
wakes** ‚Äî only the *texture* of the framing when she does.

**Dream integration (optional, light)** ‚Äî the diary prompt may receive the daypart so a day's diary carries
its felt arc; gated, low-risk. (If it complicates, defer ‚Äî not load-bearing for C.)

**Measurement + default-flip** ‚Äî confirm on a representative session that **prompt-cache hit-rate is
unchanged** (time content is in the uncached tail ‚Äî expected exactly 0 impact; this is the gate). Record the
small added input-token cost. Then flip `LUNA_TIME_AWARE=1` + `LUNA_RECALL_TIME_LABELS=1`
(+`LUNA_TIME_SUBJECTIVE=1` if it reads well in testing).

**Env:** `LUNA_TIME_SUBJECTIVE` (off ‚Üí flipped), plus the A/B flags flipped on.

## Tests (`bun test`)

- `subjectiveTime`: each daypart ‚Üí a bounded mood string; each gap bucket ‚Üí the right feltness level; the
  output set is **finite + bounded** (no unbounded/escalating intensity).
- `buildTimeBlock` with `LUNA_TIME_SUBJECTIVE=1` appends exactly one suggestion line; off ‚Üí no subjective line.
- The L1 warmth-not-guilt clause is present when on; a snapshot asserts it forbids guilt framing.
- Proactive framing: a `long_away` proactive turn's assembled directive carries the felt-absence context; a
  `continuation` one does not (and the wake *decision* ‚Äî cadence/wake-gate ‚Äî is unchanged by C).
- **Cache invariant (the flip gate):** with all three flags on, the cached system block is still
  byte-identical to all-off (everything time-related remains in the uncached tail).

## What this version explicitly does NOT include

- **No re-timing of proactive wakes** ‚Äî C colors the *framing*, not the cadence/wake-gate decision.
- **No persistent mood state machine** ‚Äî the subjective signal is recomputed per turn from time, not stored
  (keeps it bounded + stateless).
- **No bi-temporal memory (D)** ‚Äî deferred; revisit only if fact-supersession becomes a felt need.

## Risk

- **Warmth ‚Üí guilt drift** (the headline risk) ‚Äî an absence acknowledgment sliding into emotional pressure.
  Mitigated by the explicit L1 guardrail + the snapshot test + keeping the signal a *suggestion*; if testing
  shows any guilt-tinge, dial the absence line down or behind a stricter gate.
- **Over-fixation / theatrical time** ‚Äî she narrates the hour too much. Mitigated by "let it live quietly,
  announcing is almost never needed" + bounded mood strings; tune the wording.
- **Subjective-time-from-gap is novel** (no companion product is documented doing it) ‚Äî so keep it bounded
  and suggestive, not authoritative; it's an enhancement, not a claim of correctness.
