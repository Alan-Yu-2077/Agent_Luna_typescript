# v0.15.3 — Repo map + hybrid symbol locator + plan tool

> **Status: PLANNED.** Initiative: Code-agent capability (Order 8, version 4/5). Risk: **Medium**.
> Depends: v0.15.0 (grep/list/read + sandbox). Flag: `LUNA_REPO_MAP` (default `0` until the cache +
> tree-sitter load are verified, then on).

## Goal

Fix the *targeting* half of "寻址能力差、目标定位弱" — v0.15.0 gave Luna eyes (read/grep/list); this gives
her a **map**. An Aider-style repo map (a ranked, token-bounded outline of the codebase's symbols) plus a
**hybrid symbol locator** (ripgrep candidates → tree-sitter verify — SICA's design) so she can answer
"where is `dispatchToolCalls` defined / who calls it" with a structural answer, not a guessed path. Plus a
lightweight `plan` tool so multi-step code work has a visible, revisable todo spine (the single most
common scaffold across Claude Code / SWE-agent / Aider).

## What ships

### New — `packages/server/src/code/repoMap.ts` (Aider-style ranked outline)
- Parse each source file with **tree-sitter WASM** (`web-tree-sitter` + per-language `.wasm` grammars,
  vendored) to extract definitions (functions, classes, types, exports) and references.
- **Rank** by a PageRank-ish graph over symbol references (Aider's `repomap.py` algorithm) so the most
  central symbols surface first; emit a **token-bounded** outline (`{ symbol, kind, file, line }[]` →
  rendered tree, default budget ~1–2k tokens).
- **Cache in SQLite, mtime-keyed** (a `repo_map` table: `path, mtime, symbols_json`) — re-parse only
  changed files. Reuses the existing memory `sql.ts` connection pattern (versioned migration, never
  in-place — luna-ts-orient caution).
- Tool **`repo_map`** (`builtin/repo_map.ts`): `{ focus?: path|symbol, max_tokens? }` →
  the ranked outline (optionally focused on a file/symbol's neighborhood). `proactiveRisk:'safe'`,
  `concurrency:'safe-parallel'`, jailed.

### New — `packages/server/src/code/symbolLocator.ts` (hybrid locate)
- Tool **`find_symbol`** (`builtin/find_symbol.ts`): `{ name, kind?: 'def'|'ref'|'any' }` →
  `{ definitions: {file,line,signature}[], references: {file,line}[] }`.
- **Hybrid** (SICA): ripgrep produces cheap candidate lines (reuse v0.15.0's `grep` runner) →
  **tree-sitter confirms** each candidate is a real def/ref (not a comment/string/substring), attaching
  the enclosing signature. Falls back to ripgrep-only (clearly marked `verified:false`) if a grammar is
  missing — never hard-fails. `proactiveRisk:'safe'`.

### New — `packages/server/src/tools/builtin/plan.ts` (the todo spine)
- `{ action: 'set'|'update'|'get', items?: {id?, text, status?}[] }` → the current plan. Session-scoped
  (lives on the session object, surfaced as a `tool.progress`/event so the web UI can show it). Mirrors
  Claude Code's TaskCreate/TaskUpdate. `proactiveRisk:'safe'`, `concurrency:'session-serial'`.
- L1 contract clause: "for multi-step code work, set a `plan` first, update as you go."

### Modified
- `packages/protocol/src/tools.ts` — add `'repo_map'`, `'find_symbol'`, `'plan'`.
- `registry.ts` — `repo_map`/`find_symbol` behind `LUNA_REPO_MAP`; `plan` ships on (cheap, safe).
- `package.json` — add `web-tree-sitter` + vendor the grammar `.wasm` files (ts/tsx/js to start;
  **Open Q #4** = which language grammars to ship — TS-first, others on demand).
- SQLite migration file for the `repo_map` cache table.
- L1 contract — locate via `find_symbol`/`repo_map` before reading whole files.

## Tests
- `repoMap.test.ts` — a small fixture tree parses to the expected symbol set; ranking puts the
  most-referenced symbol first; mtime cache returns cached on unchanged + re-parses on touch (inject
  clock/mtime); token budget truncates with a marker.
- `symbolLocator.test.ts` — `find_symbol` finds a def + its refs; a same-name substring in a comment is
  **excluded** by the tree-sitter pass (the thing ripgrep-only gets wrong); grammar-absent fallback marks
  `verified:false` + still returns candidates.
- `plan.test.ts` — set→get round-trips; update flips a status; emits the progress event.
- `tsc` + `bun test` green; with `LUNA_REPO_MAP=0` the map/locator tools are absent (`plan` still
  present).

## What this version explicitly does NOT include
No write/shell changes (done in v0.15.1/2). No semantic/embedding code search (text+AST only — scope bound
from v0.15.0). No LSP integration (tree-sitter is the chosen tier — no language-server lifecycle to
manage). No skill library / self-edit (v0.15.4).

## Risk
**Medium.** tree-sitter WASM adds a real dependency + vendored grammars (mitigated: graceful
ripgrep-only fallback, so a missing/broken grammar degrades instead of failing). The cache is a new
SQLite table — must be a versioned migration (in-place edit is the Python drift bug we're avoiding).
Ranking is heuristic — wrong ranking is a quality issue, not a safety one; the outline is advisory, every
claim is still verifiable via `read_file`.

## Acceptance criteria
- [ ] `repo_map` returns a ranked, token-bounded, mtime-cached outline (tested).
- [ ] `find_symbol` returns tree-sitter-verified defs/refs, excludes comment/string false positives,
      degrades to marked ripgrep-only (tested).
- [ ] `plan` set/update/get round-trips + surfaces to the UI (tested).
- [ ] cache is a versioned migration; `tsc` + `bun test` green; DEVELOPMENT.md v0.15.3 entry.

## Notes for v0.15.4 (don't foreclose)
The repo map is what makes a saved **skill** addressable ("the skill that touches `runTurn`"). `find_symbol`
is the locate primitive a self-edit proposal uses to point at exactly what it wants to change — keep its
output structured (file+line+signature), not prose.
