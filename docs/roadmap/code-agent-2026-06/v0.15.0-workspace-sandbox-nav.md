# v0.15.0 — Workspace sandbox + read/navigation

> **Status: PLANNED.** Initiative: Code-agent capability (Order 8, version 1/5). Risk: **Medium**.
> Depends: nothing. Flag: `LUNA_WORKSPACE_ROOT` (config, not a feature flag — these tools are read-only
> and ship on by default).

## Goal

Give Luna the **safe, read-only half** of a code agent first: a single workspace sandbox every file
tool routes through, plus the navigation primitives she completely lacks today — windowed line-range
reads, tree listing/glob, and regex search. This is the foundation the riskier write/shell tools
(v0.15.1/2) build on, and it alone fixes "寻址能力差、目标定位弱、锁在目录里": she can browse, search,
and read precisely, bounded to a known root. Nothing here mutates anything, so it ships on by default.

## What ships

### New — `packages/server/src/tools/workspace.ts` (the sandbox)
The load-bearing safety boundary, used by every file/shell tool from here on.
- `resolveInWorkspace(path, access: 'read'|'write'|'execute'): { ok; resolved } | { ok:false; reason }`
  — canonicalize (realpath where it exists), reject `..`-escape and symlink-escape of the root, reject
  the **sensitive-path blocklist** (`~/.ssh`, `~/.gnupg`, `~/.aws`, `~/.config/gcloud`, macOS Keychains,
  browser profiles — port of Python `filesystem.py` PathAccess sensitive list) even on override.
- Root = `Bun.env['LUNA_WORKSPACE_ROOT'] ?? <repo/launch cwd>`. Writes/execute are **hard-jailed** to
  the root; reads are jailed-by-default with an opt-in `LUNA_READ_OUTSIDE_WORKSPACE=1` (Open Q #1).
- Ignore-aware helpers (`.gitignore` + a built-in ignore set: `.git`, `node_modules`, `.venv`,
  `dist`, binaries) reused by `list_files`/`grep`.
- `contentHash(text): string` (sha256) for the optimistic-concurrency story v0.15.1 needs.

### Upgraded — `packages/server/src/tools/builtin/read_file.ts`
Today it's whole-file/any-path/32 KB (`read_file.ts:30-54`). Upgrade to the SWE-agent ACI + Python
`read_file` shape (`filesystem.py:1020-1110`):
- Input: `{ path, offset?: number, limit?: number }` (1-indexed line window; default `limit` ~800,
  hard cap so a huge file can't blow context).
- Output: `{ content (line-numbered), start_line, end_line, total_lines, truncated, content_hash }`.
- Route the path through `resolveInWorkspace(path,'read')`; ENOENT stays recoverable.

### New tools — `packages/server/src/tools/builtin/`
All `proactiveRisk: 'safe'` (read-only → may run silently in a proactive turn), `concurrency:
'safe-parallel'`, jailed via `workspace.ts`.
- **`list_files`** (`list_files.ts`) — `{ path?, recursive?, glob?, include_hidden?, max_entries? }` →
  `{ entries: {path, type:'file'|'dir'}[], truncated }`. Ignore-aware. Port of Python `list_files`.
- **`grep`** (`grep.ts`) — `{ query, path?, regex?, case_sensitive?, glob?, max_results? }` →
  `{ matches: {path, line, col, text}[], truncated, shown, total }`. Implemented via **ripgrep**
  subprocess (`rg --json`, `Bun.spawn`) with a graceful fallback to a JS scanner if `rg` is absent;
  results **capped + shaped** ("showing N of M"). Port of Python `search_code`.

### Modified
- `packages/protocol/src/tools.ts:3-10` — add `'list_files'`, `'grep'` to the `ToolName` enum.
- `packages/server/src/tools/registry.ts:15-30` — mount both in `builtinRegistry`.
- L1 thinking-contract clause (`packages/server/src/persona/l1Contract.ts`) — one line: "to work in
  code, locate first (list/grep) then read the exact lines; don't guess paths."
- `EMBODIMENT_BLOCK` / persona note that she now has a workspace she can browse + read.

## Tests
- `workspace.test.ts` — `resolveInWorkspace` rejects `../` escape, rejects a symlink pointing outside,
  rejects each sensitive path, accepts an in-root path; honors `LUNA_READ_OUTSIDE_WORKSPACE`.
- `read_file.test.ts` — line-window returns the right `start/end/total_lines`, `truncated` set past the
  cap, `content_hash` stable, out-of-workspace write-access path rejected.
- `list_files.test.ts` — glob filter, ignore-set honored, `max_entries` truncation flag.
- `grep.test.ts` — regex vs literal, `case_sensitive`, capped results report `shown/total`, the
  rg-absent JS fallback returns the same shape (inject the runner).
- `tsc --noEmit` (server + protocol) + `bun test` green.

## What this version explicitly does NOT include
No write/edit/create (v0.15.1). No `shell` (v0.15.2). No repo map / symbol locator / `find_relevant_files`
ranking (v0.15.3). No skill library / self-edit (v0.15.4). `grep` is text/regex only — semantic search
is out of scope for the whole initiative.

## Risk
**Medium** — the sandbox is security-load-bearing. Mitigation: it's pure + exhaustively unit-tested
(escape/symlink/blocklist), and these tools are read-only so a bug can't destroy anything; the worst
case is an over-broad read, which the blocklist + jail bound. The ripgrep dependency is mitigated by
the JS fallback (no hard native dep).

able to record "file X was read this session" (for read-before-edit) — note where to hang that
(per-session set keyed off `read_file` calls).
