# v0.15.2 Рђћ `shell` (sandboxed) + the verify loop

> **Status: PLANNED.** Initiative: Code-agent capability (Order 8, version 3/5). Risk: **High**.
> Depends: v0.15.0 (sandbox). Flag: `LUNA_SHELL` (default `0` until E2E-verified, then flipped on).

## Goal

Close the loop: let Luna **run things** Рђћ a sandboxed `shell` (the tool LD #10 reserved *"for v0.4+ when
concrete need arises"* Рђћ now) plus first-class `typecheck` / `run_tests` / `lint` verifiers so she can do
locate Рєњ edit Рєњ **verify** Рєњ iterate without the user driving every step. `shell` is the single most
dangerous surface in the whole rewrite, so it lands on its own flag, behind a deny-regex + inspection-mode
classifier + the existing proactive surface-gate, and also subsumes directory create/move/delete (per
LD #9's ТИЏУ▓а).

## What ships

### New Рђћ `packages/server/src/tools/builtin/shell.ts`
`concurrency: 'session-serial'`, `proactiveRisk: 'surface'`, **always-on + deny-regex inside execute**
(LD #10 Рђћ no `mountedWhen`).
- Input: `{ command: string (cap ~8000), cwd?: string, timeout_ms? }`. `cwd` via
  `resolveInWorkspace(_, 'execute')`; default cwd = workspace root.
- Output: `{ stdout, stderr, exit_code, timed_out }`, output **capped** (~120 KB, middle-elided) and
  streamed as `tool.progress`.
- **Safety (port of Python `exec_command.py:49-106, 240-252`):**
  - **deny-regex** blocklist: `rm -rf`, `dd if=`, `mkfs`, `:(){ :|:& };:`, `sudo`, `shutdown`/`reboot`,
    `curl|wget Рђд | sh`, writes to dotfiles/`~/.ssh`/keychains, etc. Рєњ refused (recoverable, with the
    matched pattern named).
  - **interactive-command block**: `vim`/`nano`/`less`/`top`/`ssh`/`tmux`/`screen`/`watch` Рєњ refused
    (no TTY).
  - run via `Bun.spawn` with `ctx.abortSignal` wired to the dispatcher timeout (`timeout_ms`, default
    120 s, hard max e.g. 1800 s); kill the process tree on timeout.
  - **Open Q #2** Рђћ optional WS human-approval prompt for non-allowlisted commands (deferred; v1 is
    deny-regex + surface-gate only).

### New verify tools Рђћ `packages/server/src/tools/builtin/`
Thin, *safe-ish* wrappers (they only run the project's own checkers) so the loop is first-class and the
model doesn't hand-roll shell for verification. `concurrency: 'session-serial'`, `proactiveRisk:'surface'`
(they execute), jailed cwd.
- **`typecheck`** (`typecheck.ts`) Рђћ `bun x tsc --noEmit -p <pkg>` (or whole-repo), parsed into
  `{ ok, diagnostics: {file,line,message}[] }`.
- **`run_tests`** (`run_tests.ts`) Рђћ `bun test [path]`, parsed into `{ pass, fail, failures[] }`.
- **`lint`** (`lint.ts`) Рђћ the project's linter (prettier `--check` / oxc / biome if present),
  structured result.

### `shell` subsumes fs-mutation (LD #9 ТИЏУ▓а)
Directory create/move/copy/delete are **commands through `shell`** (`mkdir -p`, `mv`, `cp -r`, `rm`
within the jail) Рђћ no separate tools, as the ТИЏУ▓а list locked. The deny-regex still blocks `rm -rf /`
etc.; in-workspace `rm <file>` is allowed.

## Tests
- `shell.test.ts` Рђћ deny-regex refuses each dangerous pattern (named); interactive commands refused;
  timeout kills + sets `timed_out`; output cap elides; out-of-workspace `cwd` rejected; a safe command
  (`echo`/`ls`) returns stdout + exit 0. (Inject the spawner so no real destructive command runs.)
- `typecheck.test.ts` / `run_tests.test.ts` / `lint.test.ts` Рђћ parse a known-good and known-bad run into
  the structured shape (inject the runner; no real toolchain in CI).
- With `LUNA_SHELL=0`, `shell` absent (registry test).
- `tsc` + `bun test` green.

## What this version explicitly does NOT include
No container/Docker isolation (path-confinement is the chosen tier for a local single-user agent;
REWRITE_CONTEXT). No `sandbox-exec` kernel wrap (a possible later hardening). No interactive shell / TTY.
No repo map (v0.15.3). No skill library (v0.15.4).

## Risk
**High Рђћ the highest-risk surface in the rewrite.** Mitigations stack: default-off flag E2E-verified
before flip; deny-regex (LD #10) tested per-pattern; jailed `cwd`; interactive-command block; timeout +
process-tree kill + output cap; `proactiveRisk:'surface'` (no silent shell in proactive turns) + the
`LUNA_PROACTIVE_MAX_ACTIONS` budget; full `tool` tracing. Residual: deny-regex is a blocklist (not a
jail) Рђћ a creative destructive command could slip; hence the surface-gate + budget + (Open Q #2) optional
approval prompt as defense-in-depth.
