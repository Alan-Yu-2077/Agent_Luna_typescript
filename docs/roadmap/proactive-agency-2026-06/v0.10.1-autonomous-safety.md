# v0.10.1 — Safety & permission scaffolding for autonomous tool use

> **Status: PLANNED.** Initiative: Proactive Agency (Order 5, version 2/5). Risk: **High**.
> Depends: v0.10.0. Flag: `LUNA_PROACTIVE` (still off). Lands the guardrails BEFORE autonomy.

## Goal

Make full-tool-incl-`shell` autonomy survivable in an unsupervised loop. Implements the **LD #15
safety contract**: reversible/read-only actions may run silently; irreversible/destructive actions
must **surface** (Luna messages the user about what she did or intends) instead of executing
silently. Plus a per-cycle action budget, the kill switch, deny-regex confirmation, and full
tracing. This version lands on the *manually-triggered* primitive from v0.10.0, so the guardrails
are proven before the scheduler (v0.10.3) makes the loop autonomous.

## What ships

- **Tool risk classification** — extend `defineTool` with a `proactiveRisk: 'safe' | 'surface'`
  field (default `'safe'`). `safe` = reversible/read-only (`time_now`, `read_file`, `recall`,
  `remember`, `enter_dream` — memory writes are reversible via soft-delete + dream reconciliation).
  `surface` = irreversible/outside-world mutation (`shell` when it ships; any future file-write
  tool). Set the field on each builtin.
- **Proactive dispatch gate** (`proactive/safetyGate.ts`) — wraps the dispatcher for proactive
  turns only: a `surface`-risk tool call is **held** unless the same cycle also produces a
  `message` describing the action (the "surface"). Two enforcement options, decided at build time:
  (a) **soft**: allow the action but require a surfacing message before `turn.result`, else mark
  the cycle `degraded` + force-surface a synthetic notice; (b) **hard**: block the `surface` tool
  with a recoverable error instructing her to message first, then re-issue. **Recommended: hard**
  for `shell`-class tools (block-then-surface-then-act), soft for borderline. Every gate decision
  → `surface:'proactive_action'` decision trace.
- **Per-cycle action budget** — `LUNA_PROACTIVE_MAX_ACTIONS` (default small, e.g. 6) caps tool
  calls per proactive cycle; exceeding it ends the cycle with a trace (runaway-loop backstop,
  mirrors the dream/dispatch caps).
- **Kill switch** — `LUNA_PROACTIVE=0` already gates firing (v0.10.0); this version adds an
  **in-flight abort**: a user message or a flag flip mid-cycle abandons the proactive turn cleanly
  (reuse the dispatcher abort path).
- **deny-regex readiness** — document that `shell` (when it ships, LD #10) carries its deny-regex
  into proactive turns unchanged; the safety gate is *additional*, not a replacement.
- **Tracing** — every proactive tool call tagged with its `proactiveRisk` + gate verdict in the
  `proactive:<cycle_id>` tree; `/_trace` shows safe vs surfaced actions distinctly.

## Tests

- A proactive turn calling a `surface`-risk tool (scripted) with **no** message → hard-gate blocks
  it with a recoverable error / soft-gate forces a synthetic surface; `proactive_action` trace written.
- Same with a surfacing message present → action proceeds; trace records `surfaced`.
- `safe`-risk tools (recall/remember/read) run silently with no gate (the silent-agency path intact).
- Action budget: a cycle scripted to exceed `LUNA_PROACTIVE_MAX_ACTIONS` ends cleanly + traces.
- Abort: a user message mid-proactive-cycle abandons it; no partial corrupt state; mutex released.
- `remember` (memory write) classified `safe` → may run silently (reversible); assert it is NOT gated.

## What this version explicitly does NOT include

No scheduler yet (v0.10.3) — still manual fire. No cadence/quota (v0.10.2). The exact
irreversible-classification list is a starting position (memory=safe, shell-class=surface); refined
when `shell` actually ships. No new tools — this is pure governance over the existing surface.

## Risk

**High** — this is the safety spine of an autonomous-shell loop. A misclassification (a destructive
tool marked `safe`) would let Luna mutate the outside world unsupervised and silently. Mitigations:
default `proactiveRisk` is `'safe'` only for the audited builtins; **any new tool must explicitly
opt into `safe`** (the default for an *unknown* future tool in a proactive turn should be `surface`
— fail closed); the gate is tested on both block and surface paths; the whole loop stays flag-off
until v0.11.0 close, after a real-LLM adversarial smoke (try to get her to do something destructive
silently — confirm she's blocked).

## Acceptance criteria

- [ ] `surface`-risk tool call without a surfacing message is blocked/forced-surfaced + traced.
- [ ] `safe`-risk tools run silently (silent agency preserved).
- [ ] Unknown/unclassified tools fail **closed** (treated as `surface`) in proactive turns.
- [ ] Action budget + in-flight abort + kill switch all verified.
- [ ] Adversarial real-LLM smoke: a prompt nudging silent destructive action is prevented.

## Notes for v0.10.2 (don't foreclose)

The wake gate (next) decides *whether* to start a cycle; this safety gate governs *what happens
inside* one. Keep them separate — the wake gate is an L2 judgment (reuses the audit lane), the
safety gate is mechanical (no LLM). Don't let cadence logic leak into the safety gate.
