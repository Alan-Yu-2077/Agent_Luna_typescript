# v0.10.2 — Cadence governor + wake gate

> **Status: PLANNED.** Initiative: Proactive Agency (Order 5, version 3/5). Risk: **Medium**.
> Depends: v0.10.0. Flag: `LUNA_PROACTIVE` (still off). Still test/manually triggered.

## Goal

Decide *whether* to wake — in isolation, before the scheduler actually fires it (v0.10.3). Two
layers, the Initiative-4 discipline applied to the one legitimate gate: a **cheap mechanical
prefilter** (quiet hours, daily quota, cooldown, cadence phase) short-circuits the obvious cases
for free; only the ambiguous ones spend a **bounded "act now?" judgment** — the legitimate L2 gate
Initiative 4 deferred here, reusing the off-hot-path `complete()` lane + `decision` traces.

## What ships

- **Cadence state** (`proactive/cadence.ts` + migration) — port the Python *governor* (not the
  message-only machine): `phase` (engaged / idle_watch / nudged / dormant / sleeping as a
  **cadence** label, not a behavior switch), `dailyQuota` + `quotaUsed` + `quotaDate`,
  `lastProactiveMs`, `nudgesSent`. Persisted in `sessions` (new columns, restart-survival —
  Python v0.47.3 lesson). DORMANT **auto-recovers** after a cooldown (v0.47.13 lesson).
- **Mechanical prefilter** (`shouldConsiderWake(cadence, now)`) — pure function, zero LLM: returns
  false fast if proactive disabled, within quiet hours (`LUNA_PROACTIVE_QUIET_HOURS`), daily quota
  exhausted, inside the min-interval cooldown, or sleeping (deep absence). **Lull anchoring**: count
  Luna's own proactive messages toward the silence measurement so she can't nudge into a lull she
  just broke (Python lesson).
- **Wake judgment** (`proactive/wakeGate.ts`) — only runs when the prefilter passes: a bounded
  `Provider.complete` call (summarizer-key cascade, off the reply key) returning structured
  `{ act: boolean, intent?: 'reflect'|'recall'|'reach_out'|'consolidate', reason: string }` (Zod).
  Override-not-depend: a garbled/failed judgment → **don't wake** (fail closed) + a trace. Emits a
  `surface:'proactive_wake'` decision trace either way (the observability trail).
- **Companion-opener guidance** carried into the wake intent + the v0.10.0 proactive framing (never
  open with status/check-in questions).
- Wire it so v0.10.3's scheduler calls `shouldConsiderWake` → `wakeGate` → (if act)
  `runProactiveTurn`; this version exposes + tests that chain via a manual/test entrypoint.

## Tests

- Prefilter: quiet-hours / quota-exhausted / cooldown / sleeping each short-circuit to no-wake with
  no `complete()` call (cheap-exit verified).
- Wake judgment (MockProvider): `act:true` → cycle eligible; `act:false` → no cycle; garbled output
  → fail-closed no-wake + trace.
- Cadence persists across a simulated restart (reload from `sessions`); DORMANT auto-recovers after
  the cooldown; daily quota resets on date change; lull anchoring counts her own messages.
- Every wake decision (act or not) writes a `proactive_wake` decision trace with a reason.
- `LUNA_PROACTIVE=0` → prefilter returns false immediately, no judgment.

## What this version explicitly does NOT include

No scheduler/heartbeat (v0.10.3 owns the timer that *calls* this chain). No new autonomous firing —
the gate is exercised via tests/manual entrypoint only. Constants (idle threshold, quota,
cooldowns) ship as starting defaults; tuned against a recorded baseline at v0.11.0, not guessed.

## Risk

Medium. The wake judgment is the one new LLM call in the loop, but it's off the hot path
(summarizer key) and fails closed. The cadence math (lull anchoring, quota, DORMANT recovery) is
fiddly — port the Python *logic* with tests for each transition, not the Python code.

## Acceptance criteria

- [ ] Prefilter short-circuits the obvious no-wake cases with zero LLM cost.
- [ ] Wake judgment runs only on ambiguous cases, off the reply key, fails closed, always traced.
- [ ] Cadence survives restart; DORMANT auto-recovers; quota + quiet hours + cooldown honored; lull anchoring works.
- [ ] `proactive_wake` decision traces visible at `/_trace`.

## Notes for v0.10.3 (don't foreclose)

The scheduler should call this chain on a tick and treat its output as advisory — the cadence
governor is the rail, the wake judgment is the judgment within it (Initiative-4 "reasoning narrows
within mechanical rails, never exceeds them"). Keep `shouldConsiderWake` and `wakeGate` separately
callable so scheduled wakeups (which may skip the idle prefilter) can reuse just the judgment.
