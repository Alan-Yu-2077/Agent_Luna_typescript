# v0.10.0 — Proactive turn primitive (manual trigger)

> **Status: PLANNED.** Initiative: Proactive Agency (Order 5, version 1/5). Risk: **Medium**.
> Depends: nothing. Flag: `LUNA_PROACTIVE` (default off).

## Goal

Establish the agency core in isolation: a proactive turn is a `runTurn` with a proactive opening
framing and the full tool registry, and — the key new capability — it **may complete with no
`message`** (act silently). Triggered **manually** this version (a WS `proactive.fire` event /
dev-chat button), so "can she take an autonomous tool-calling turn and optionally stay silent" is
proven before any scheduler decides *when*. Mirrors how Initiative 2 shipped dream: manual trigger
first, autonomy deferred.

## What ships

- **Protocol** — `ClientEvent` += `proactive.fire` (dev/manual trigger); `ServerEvent` +=
  `proactive.started` / `proactive.finished` (cycle markers, like `dream.status`). The proactive
  turn's `message` bubbles reuse the existing `tool.started/progress/finished` + `turn.result`
  events — no new message wire shape (LD #9 consistency).
- **`packages/server/src/proactive/proactiveTurn.ts`** — `runProactiveTurn(session, opts)`: builds
  a proactive **user-role stage direction** ("No one has spoken; you woke on your own. You may
  reflect, recall, consolidate, look something up, or reach out — or do nothing. Calling tools is
  acting; you do not have to speak.") + the companion-opener constraint (no 在吗/status questions),
  then runs the existing turn engine with the **full registry**. Traced under
  `proactive:<cycle_id>` (dream-cycle precedent).
- **`runTurn` change** — `finalize`'s empty-reply guard must NOT fire for proactive turns: thread a
  `proactiveTurn: boolean` flag into `TurnState`/`RunTurnOptions`; when set, a zero-`message` turn
  is a legitimate silent outcome (emit `turn.result` with empty text + a `proactive_silent` node
  trace, no corrective retry). The integrity guards (is_final/intent) still apply to any message
  she *does* send.
- **WS wiring** ([`ws.ts`](../../../packages/server/src/ws.ts)) — `proactive.fire` → take
  `Session.mutex` (so it never overlaps a user turn) → `runProactiveTurn` → push bubbles over WS.
  Rejected with a clear error if a user turn is active.
- **Dev chat** — a "🌱 主动" button to fire a proactive turn; proactive bubbles render with a subtle
  marker (distinct from reactive replies); silent cycles show a "(Luna 安静地做了点什么)" chip.

## Tests

- A proactive turn with a scripted silent outcome (tools called, no `message`) → `turn.result`
  empty text, `proactive_silent` trace, **no empty-reply retry** (the relaxation works).
- A proactive turn that DOES message → bubbles delivered; integrity guards still apply to it.
- A proactive turn that calls memory tools (scripted `remember`) → fact persisted; cycle traced
  under `proactive:<id>`.
- Mutex: `proactive.fire` while a user turn is active → rejected, no overlap; after it completes → runs.
- `LUNA_PROACTIVE=0` → `proactive.fire` is a no-op/error; reactive turns byte-identical.

## What this version explicitly does NOT include

No scheduler (manual trigger only — v0.10.3). No cadence/quota/wake-gate (v0.10.2). No safety
tier for irreversible actions yet (v0.10.1) — so this version's manual trigger is dev-only and the
flag stays off; do not enable autonomous firing until v0.10.1 lands the guardrails. No
self-continuation / dream auto-trigger (v0.11.0).

## Risk

Medium. The empty-reply-guard relaxation must be surgically scoped to proactive turns — a leak into
reactive turns would silently break the v0.6.2 "must speak" guarantee. Mitigation: the flag is a
per-turn `proactiveTurn` boolean, tested on both paths; reactive byte-identity asserted.

## Acceptance criteria

- [ ] `runProactiveTurn` runs the full registry and can complete silently (no message).
- [ ] Empty-reply guard does not fire for proactive turns; still fires for reactive turns (tested).
- [ ] Proactive turn takes `Session.mutex`; never overlaps a user turn.
- [ ] Cycle traced under `proactive:<cycle_id>`; dev chat can fire + render it.
- [ ] Real-LLM smoke: manual fire → she does something coherent (recalls / notes / or reaches out).

## Notes for v0.10.1 (don't foreclose)

Keep `runProactiveTurn` tool-dispatch interceptable — v0.10.1 needs to classify each tool call as
reversible/irreversible and gate irreversible ones behind surfacing. Don't hardwire the full
registry; pass it so the safety tier can wrap/restrict it.
