# v0.10.3 — Scheduler / heartbeat (idle ticks + scheduled wakeups)

> **Status: PLANNED.** Initiative: Proactive Agency (Order 5, version 4/5). Risk: **High**.
> Depends: v0.10.1 (safety), v0.10.2 (gate). Flag: `LUNA_PROACTIVE` (still off; flipped at v0.11.0).

## Goal

Make the loop **autonomous**: a server-side heartbeat that, on idle ticks and scheduled wakeups,
runs the wake chain (v0.10.2) and — when it says act — fires a proactive turn (v0.10.0) under the
safety tier (v0.10.1). This is the version where Luna acts with no human in the loop, so it lands
*after* both the guardrails and the gate exist.

## What ships

- **Heartbeat** (`proactive/scheduler.ts`, started from `main.ts`) — a single server-side timer
  (not per-session threads) that, every `LUNA_PROACTIVE_TICK_SECONDS` (default ~60s), iterates the
  active session(s) and runs `shouldConsiderWake → wakeGate → runProactiveTurn`. Pure backend, no
  UI-liveness dependency (Python v0.45.0 lesson — trivially true with persistent WS). Uses
  background-safe scheduling (no foreground sleep); survives across turns.
- **Idle trigger** — the prefilter's idle-threshold path: fire only after the user has been silent
  past `LUNA_PROACTIVE_IDLE_THRESHOLD`, with lull anchoring.
- **Scheduled wakeups** — a small declarative schedule (shape decided at build time: cron-ish
  strings or a `{ when, intent }` table in a config/migration) for time-anchored wakes ("morning
  hello", "deep-night dream"). The scheduler checks due entries each tick. Scheduled wakes may skip
  the idle prefilter but still pass the wake judgment + safety tier + quota.
- **Mutex + abort** — every proactive turn takes `Session.mutex`; a `chat.send` arriving mid-cycle
  wins (the proactive cycle aborts cleanly — v0.10.1 abort path). The scheduler never starts a
  cycle while a user turn is active.
- **Restart behavior** — cadence reloads from `sessions` (v0.10.2); the scheduler resumes without
  bursting (cooldown + quota already persisted; no outbox to replay — the Python v0.58.0.2
  backlog-burst class cannot occur here).
- **Observability** — `proactive.started/finished` over WS; a heartbeat heartbeat-tick trace
  (sampled) so the loop's liveness is visible without flooding `/_trace`.

## Tests

- Heartbeat fires the wake chain on tick; an `act:true` (MockProvider) → a proactive cycle runs and
  pushes bubbles/actions; `act:false` → nothing.
- Idle threshold respected (no fire before it); lull anchoring prevents back-to-back fires.
- Scheduled wakeup: a due entry fires at its tick even with no idle gap; not-due entries don't.
- Mutex: a `chat.send` during a scheduled tick that would fire → user turn wins, no overlap; a
  `chat.send` mid-cycle → cycle aborts cleanly, mutex released, no corrupt state.
- Restart: cadence reloads; the scheduler does not burst (cooldown/quota honored); no replay.
- `LUNA_PROACTIVE=0` → heartbeat is inert (kill switch).

## What this version explicitly does NOT include

No self-continuation or dream auto-trigger yet (v0.11.0 — they're *kinds* of scheduled/triggered
proactive turns, layered on this scheduler). No event-driven triggers (explicitly deferred — idle +
scheduled only). The flag stays **off** through this version; turned on only at v0.11.0 after the
end-to-end loop is proven and measured.

## Risk

**High** — this is the autonomy switch. Failure modes: a runaway timer firing too often (mitigated
by tick interval + cooldown + daily quota + per-cycle action budget), a proactive turn racing a
user turn (mitigated by the mutex + abort), a restart burst (structurally impossible — no outbox).
The kill switch (`LUNA_PROACTIVE=0`) is the ultimate backstop and stays default-on-for-off until
v0.11.0 close. A real-LLM soak (let the loop run idle for an extended period) is part of acceptance.

## Acceptance criteria

- [ ] Heartbeat autonomously fires proactive turns via the full wake chain + safety tier.
- [ ] Idle threshold + scheduled wakeups both work; lull anchoring + quota + quiet hours honored.
- [ ] No proactive/user-turn overlap; mid-cycle user message aborts cleanly.
- [ ] Restart resumes without bursting; kill switch fully inert.
- [ ] Real-LLM idle soak: the loop behaves (sane cadence, no runaway, everything traced).

## Notes for v0.11.0 (don't foreclose)

Self-continuation = schedule a near-immediate wakeup (a few seconds) right after a user turn, with
a "continuation" intent. Dream auto-trigger = a scheduled/idle wake whose chosen action is
`enter_dream`. Both should be expressible as scheduler inputs, not new subsystems — keep the
scheduled-wakeup entry shape general enough to carry an `intent`.
