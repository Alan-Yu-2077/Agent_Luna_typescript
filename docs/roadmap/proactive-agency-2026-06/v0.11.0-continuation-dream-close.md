# v0.11.0 — Self-continuation + dream auto-trigger + surfacing + close

> **Status: PLANNED.** Initiative: Proactive Agency (Order 5, capstone, version 5/5). Risk:
> **Medium**. Depends: all of v0.10.x. Flag: flips `LUNA_PROACTIVE` default **on**.

## Goal

Land the two behaviors that fall out of the scheduler for free, surface proactive activity in the
UI, measure the loop, flip it on, and close. Self-continuation and dream auto-trigger are not new
subsystems — they are *kinds of scheduled proactive turns*, proving the redesign's central claim.

## What ships

- **Self-continuation as a delayed micro-wake** — after a user turn finalizes, the scheduler may
  enqueue a near-immediate wakeup (a few seconds) with intent `continuation`. The proactive turn's
  framing quotes her just-sent reply as her own words (Python v0.27.1 lesson — user-role stage
  direction, low in the array, never system) and asks for at most one genuinely-new follow-up.
  Gated by a probability/mechanical rule (Python v0.28.1 lesson — never a model-emitted "more to
  say" flag). Reuses the proactive path entirely; `LUNA_SELFCONT` flag, default decided here
  (REWRITE_CONTEXT Open Q #2).
- **Dream auto-trigger** — a scheduled/deep-idle wake whose wake-judgment intent resolves to
  `enter_dream`; the proactive turn calls the existing `enter_dream` tool. This closes LD #11's
  deferred half (dream auto-idle) with **no new scheduler** — the proactive heartbeat is the idle
  scheduler dream was waiting for. Quiet-hours + "don't dream while a user turn is active" honored
  via the same gate.
- **Surfacing UI** (dev chat) — a proactive activity strip: silent cycles show what she did
  (recalled X, saved Y, dreamed), surfaced cycles show the message + the action. A "proactive log"
  view backed by the `proactive:<cycle_id>` traces, so the user can see her autonomous life.
- **Measurement sweep** (`scripts/proactive-soak.ts`) — an idle soak with the loop on: records
  wake-decision rate, action mix (silent vs surfaced, tool histogram), cadence sanity (no runaway,
  quota honored), and any safety-gate blocks. Recorded in the DEVELOPMENT entry like the
  Initiative 3/4 sweeps.
- **Default flip** — `LUNA_PROACTIVE` → default on (`!== '0'`), `=0` kill switch retained. Cadence
  constants tuned against the soak (not guessed). `.env.example` documents all proactive flags.
- **Docs close** — REWRITE_CONTEXT: LD #15 (proactive agency) marked LANDED, Open Q #2
  (self-continuation default) resolved, dream auto-trigger marked shipped; roadmap README →
  Initiative 5 ✅, head v0.11.0; orient skill gains the proactive file map.

## Tests

- Self-continuation: a user turn → a scripted continuation micro-wake fires once, adds one new
  line, never echoes/duplicates (v0.27.1/v0.28.1 guards); probability gate bounds it; off when flagged off.
- Dream auto-trigger: deep-idle wake → intent resolves to `enter_dream` → dream cycle runs; not
  during a user turn; not in quiet hours.
- Surfacing UI renders silent vs surfaced cycles from traces.
- Default-on: suite passes with no env set; tests pinning the off-path set `LUNA_PROACTIVE=0`
  explicitly (the Initiative-3/4 audit-don't-blanket-flip discipline).

## What this version explicitly does NOT include

No event-driven triggers (deferred beyond this initiative). No Hermes-style autonomous
skill-document authoring (a possible future "proactive self-improvement" initiative — the analog
here, proactive memory consolidation, is already partly covered by dream; named, not built). No
multi-session proactivity (single companion user).

## Risk

Medium — mostly integration + the default flip over proven pieces. The honest risk: the soak might
show the loop is too chatty or too quiet, or that silent autonomous actions feel uncomfortable.
That's an acceptable measured outcome — tune cadence, or keep the flag off and ship the capability
behind it. A null/negative result recorded honestly is part of acceptance (the Initiative-4 lesson).

## Acceptance criteria

- [ ] Self-continuation works as a delayed micro-wake (one new line, no echo, bounded).
- [ ] Dream auto-triggers on deep idle via the scheduler (LD #11 deferred half closed).
- [ ] Proactive activity (silent + surfaced) visible in the dev chat from traces.
- [ ] Soak recorded; cadence sane; safety gate holds; defaults flipped with `=0` kill switch.
- [ ] Initiative 5 acceptance criteria (README) all checked; LD #15 + Open Q #2 closed in docs.

## Notes for the future (don't foreclose)

The scheduler + wake-gate + safety-tier substrate generalizes: event-driven triggers and
Hermes-style proactive self-improvement (consolidate experience → write durable skill/notes) are
natural next initiatives that reuse this loop. Keep the scheduled-wakeup `intent` taxonomy open so
new proactive behaviors are new intents, not new subsystems.
