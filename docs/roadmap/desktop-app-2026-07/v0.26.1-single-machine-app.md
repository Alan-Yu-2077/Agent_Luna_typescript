## Risk

- **High â€” packaging + process supervision + signing** are the classic desktop pain. Mitigation: model
  the sidecar pattern on Open-LLM-VTuber (same backend-server-over-WS split as Luna, shipped on
  Electron); test the packaged artifact, not just `electron .`; explicit kill-on-quit test to prevent
  orphaned servers.
- **A migration/DB-path regression could touch Alan's data.** Mitigation: `LUNA_DB_PATH` defaults to
  today's behavior when unset; the desktop app uses a **fresh** app-data DB (or a one-time import), so
  the dev instance's `luna.sqlite` is never moved out from under him.
- **Secrets in the bundle** would be a leak. Mitigation: secrets load from app-data config only; a build
  check asserts no key string is present in the packaged resources.
