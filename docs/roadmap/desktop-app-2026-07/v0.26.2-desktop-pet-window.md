# v0.26.2 — Desktop-pet window + geo shim + polish

> **Status: PLANNED.** Initiative: Desktop app (Order 19, version 3/3). Risk: **Medium**. Depends:
> v0.26.1 (a working packaged app). Flag: a settings toggle for pet-mode (windowed ↔ floating pet).
> Initiative close.

## Goal

Make Luna a real **desktop companion**: a transparent, frameless, always-on-top window that shows her
over the desktop, keeps animating even when occluded, and lets clicks pass through around her body.
Add the one real API shim (geolocation) and verify the transparent-canvas correctness that only
surfaces after this port. This is where Init 18's collapsed companion mode gets its floating home.

## What ships

**`packages/desktop/main.ts`** (the pet window):
- The `BrowserWindow` gains `transparent:true`, `frame:false`, `alwaysOnTop:true` (macOS window level
  `'floating'`), `hasShadow:false`, `skipTaskbar` as appropriate. A settings toggle switches between a
  normal windowed mode and the floating pet.
- **Background-throttling OFF** — the sharpest pet failure mode: `backgroundThrottling:false` **and**
  handle the occlusion path (Electron flips `visibilityState→hidden` on occlusion, which re-throttles
  rAF; keep the pixi ticker alive so breathing/idle/lipsync never freeze behind other windows).
- **Click-through by region:** `setIgnoreMouseEvents(true, {forward:true})` globally, toggled `false`
  when the cursor is over an interactive region (Luna's body bbox, the input bar, a bubble). Renderer
  hit-tests the cursor and drives the toggle over IPC. Reconcile with `pixiLive2DSink`'s full-canvas
  `pointerdown`/`wheel`/`dblclick` drag/zoom handlers + the global `pointermove` gaze listener
  (`pixiLive2DSink.ts:163-238,208-217`) — clicks in transparent margins pass through; the model bbox
  still captures drag.

**Geo shim** (the one real API rewrite, `geo.ts:16-28`):
- Replace/augment `navigator.geolocation` with a desktop path: an Electron main-process location
  provider (native CoreLocation via a helper, or a configured provider) that calls the same `onFix`
  → `client.geo` path; **or** (zero-code option) drop live GPS on desktop and lean on the server's
  `LUNA_LAT_LON`. Keep `geo.ts`'s silent-no-op contract for the browser build.

**Transparent-canvas correctness (verify, fix if needed):**
- **Premultiplied-alpha:** Cubism draws premultiplied; confirm pixi's alpha/premultipliedAlpha context
  attributes match so there are **no dark halos** at Luna's edges over the desktop (surfaces only now,
  against a truly transparent background).
- **Packaged-build transparency:** verify transparency in the built `.app`/`.dmg`, not just `electron .`
  (a known both-shells failure class); avoid vibrancy; ensure `html/body` transparent before first
  paint.
- **Context-loss resilience:** wire a `webglcontextlost/restored` handler that re-uploads the model +
  Cubism textures, so an occlusion/GPU-reclaim wake doesn't leave a blank/garbled canvas.

**Polish (optional within this version):** tray icon + menu (show/hide, quit), launch-at-login,
remember window position.

## Tests

- `packages/desktop`: hit-test → `setIgnoreMouseEvents` toggle logic (unit, mocked IPC): cursor over the
  model bbox / input bar → interactive; over transparent margin → pass-through.
- Geo shim: the desktop provider calls the same `onFix({lat,lon})` contract `app.ts` already wires; the
  browser build's `geo.ts` behavior is unchanged (regression).
- **Manual (the real acceptance):** in the **packaged** app — transparent pet over the desktop; Luna
  keeps animating when another window covers her; clicks around her hit the desktop, her body + input
  bar stay interactive; no edge halos; weather resolves via the shim or `LUNA_LAT_LON`.

## What this version explicitly does NOT include

- Tauri (still deferred; if pursued later, it's its own initiative gated on a WKWebView go/no-go).
- Auto-update / public notarized distribution.
- Per-pixel (vs per-region) click-through refinement beyond what the pet needs.
- Multi-monitor pet placement niceties (can be a follow-on).

## Risk

- **Medium — macOS transparent-window + click-through gotchas** are real but well-charted (Electron
  issues #23042 click-through, #31862/#40515 transparency). Mitigation: model on shipped pets
  (Open-LLM-VTuber, WanNianYanHuo Electron+pixi-live2d pet); verify the packaged artifact; the windowed
  mode from v0.26.1 remains a working fallback if pet-mode has a platform bug.
- **Background-throttling freeze** would make her look dead behind other windows. Mitigation: explicit
  `backgroundThrottling:false` + occlusion handling + a manual occluded-animation check in acceptance.
- **Click-through vs the canvas drag/zoom handlers** could conflict. Mitigation: the hit-test owns the
  toggle; drag/zoom only active when cursor events are enabled (over the body).

## Acceptance criteria

- [ ] Packaged app shows a transparent, always-on-top Luna over the desktop; **animation continues when
      she is occluded** (no ticker freeze).
- [ ] Clicks around Luna pass to the desktop; her body + the input bar + bubbles stay interactive.
- [ ] No dark halos at her edges (premultiplied-alpha correct); transparency holds in the packaged build.
- [ ] Weather location resolves via the desktop geo shim or `LUNA_LAT_LON`.
- [ ] `packages/web` browser build unaffected; `bun test` + `tsc` green.
- [ ] Initiative 19 acceptance criteria (README) all check → mark ✅ in the master index +
      `DEVELOPMENT.md`.
