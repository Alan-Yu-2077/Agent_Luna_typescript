# v0.18.0 — web_search (the "look it up" capability)

> **Status: PLANNED.** Initiative: Web tools (Order 11, version 1/3). Risk: **Medium**.
> Depends: nothing. Flag: `LUNA_WEB_SEARCH` (default off).

## Goal

Give Luna a `web_search` tool that hits a live search API and returns ranked, citeable results — the
Python v0.58 capability, re-expressed on the TS dispatcher and **hardened against the "嘴上说手没动"
defection** that Python had to chase in three patches. This version also lays the shared **web infra**
(provider abstraction + `fetch`/env client + soft-fail discipline) that v0.18.1's `web_fetch` reuses. It
ships **off**; flipping it on is v0.18.2's job.

## What ships

**New module `packages/server/src/tools/web/`:**

- `provider.ts` — a `WebSearchProvider` interface (port of Python's Protocol):
  `search(query, {maxResults, timeRange?, includeDomains?, excludeDomains?}, signal) → Promise<SearchResult[]>`
  where `SearchResult = { title: string; url: string; snippet: string; score?: number; ageHint?: string }`.
  A `getProvider(name)` dispatch (`'tavily'` default; throws on unknown).
- `tavily.ts` — the default provider. POSTs to `https://api.tavily.com/search` via `fetch` with
  `authorization: Bearer ${LUNA_WEB_SEARCH_API_KEY}`, `signal: ctx.abortSignal`, the
  `fetchEmbedClient` error pattern (`embed.ts:1-69`): non-2xx ⇒ throw with status + body sliced to ~200
  chars; never leak the full payload. Per-result `snippet` clipped to `LUNA_WEB_SEARCH_RESULT_CHARS`.
- `web_search.ts` — the `defineTool` (mirrors `tools/builtin/*` shape):
  - `input`: `z.object({ query: z.string().min(2).max(400), max_results: z.number().int().min(1).max(10).default(5), time_range: z.enum(['day','week','month','year']).optional(), include_domains: z.array(z.string()).max(10).optional(), exclude_domains: z.array(z.string()).max(10).optional() })`.
  - `output`: `z.object({ query, results: z.array(z.object({title,url,snippet,score?,age_hint?})), provider, ts })`.
  - `concurrency: 'safe-parallel'` (read-only, no session race), `proactiveRisk: 'safe'` (LD #15 lists
    *searches* as silent-OK), `timeoutMs: LUNA_WEB_SEARCH_TIMEOUT_MS ?? 15000`.
  - `summarize(out)` → numbered citation line: `` `N results for "q": [1] url; [2] url; …` `` (so even if
    the model ignores structured data, plain-text citations survive — Python's discipline).
  - `execute`: yields `{kind:'progress', payload:{note:'正在查一下: '+query.slice(0,40)+'…'}}` first (the
    Python `interim_message_for` hook → a `tool.progress` UI line), then the provider call, then
    `{kind:'ok',data}` **or** `{kind:'err', code, message, recoverable:true}` for **every** failure
    (no key, empty query, unknown provider, timeout, HTTP error) — soft-fail; nothing throws past the tool.

**Registry + boot:** `registry.ts` gains `webSearchEnabled()` (`Bun.env['LUNA_WEB_SEARCH']==='1'`) and a
`withWebSearch(reg)` composer (mirrors `withCodeWrite`/`withShell`, `registry.ts:58,65`); `main.ts` wraps
the registry at boot. When off, the tool is never in the surface Luna sees.

**The defection guard (extends LD #14, `docs/roadmap/action-integrity-2026-06/`):**

- **L1 commitment clause** — a sentence added to the L1 thinking contract (the cached core block,
  `persona/l1Contract.ts`): *"If your thinking decides this turn needs a live lookup, you MUST emit the
  `web_search` (or `recall`) call in THIS SAME turn. Saying 'let me look that up / 我去查一下' and then
  ending the turn without the tool call is the failure mode, never the right move — calling the tool IS
  the act of searching."* Gated on `web_search` being mounted.
- **Intent-no-call audit** — extend the existing off-hot-path defection audit (`runTurn` finalize lane):
  when `web_search` is mounted, the turn emitted **no** `web_search` call, and the thinking contains a
  lookup-intent keyword (CN + EN set: 搜索/查一下/查询/去查/search the web/look it up/…), write a
  `surface:'web_search_intent_no_call'` decision trace (`matched_keyword`, thinking tail). Zero cost on
  success turns. **No forced-retry** (Python deferred it pending data — same here).

**L1 "when to reach for the web" clause** — a conservative directive: *default to NOT searching* (stable
knowledge, brainstorming, chitchat warrant none); reach for `web_search` only for current facts past
training or an explicit "look it up", and prefer `recall` first for things she may already know.

**Env (all read live, defaults shown):** `LUNA_WEB_SEARCH` (master gate, **off**), `LUNA_WEB_SEARCH_API_KEY`,
`LUNA_WEB_SEARCH_PROVIDER` (`tavily`), `LUNA_WEB_SEARCH_TIMEOUT_MS` (`15000`), `LUNA_WEB_SEARCH_MAX_RESULTS`
(`5`), `LUNA_WEB_SEARCH_RESULT_CHARS` (`800`).

## Tests (`packages/server/src/tools/web/*.test.ts`, `bun test`)

- **Provider→output shape**: a stub provider (injected `fetch` / injected provider) returns 3 results →
  `output.results` has the 3 `{title,url,snippet}`; `summarize` renders `[1] url; [2] url; [3] url`.
- **Soft-fail, every failure path**: no API key → `{kind:'err', code:'no_api_key', recoverable:true}`;
  injected `fetch` throwing / non-2xx → `err` with the sliced body, **no throw escapes** the generator;
  unknown provider → `err`. The dispatcher still records a clean tool result.
- **Abort/timeout**: an injected slow provider + a pre-aborted `ctx.abortSignal` → the tool ends promptly
  with an abort `err`, not a hang.
- **proactiveRisk**: `webSearchTool.proactiveRisk === 'safe'` (so it may run in a silent proactive turn).
- **Defection audit**: a turn whose thinking contains "我查一下" but emits no `web_search` call →
  a `web_search_intent_no_call` decision trace is written (assert via the trace store, as the
  action-integrity tests do); a turn that *does* call `web_search` writes none.
- **Registry gating**: `withWebSearch` adds `web_search` only when `LUNA_WEB_SEARCH==='1'`.

## What this version explicitly does NOT include

- **No `web_fetch` / page reading / extraction / SSRF guard** → v0.18.1.
- **No default-on** — `LUNA_WEB_SEARCH=0` ships; flip is v0.18.2 after cost is measured.
- **No citation UI card** (the `summarize` text + raw `url`s are enough for now) → rendering is v0.18.2.
- **No forced-tool retry** on defection (audit only) — deferred pending data, per Python's lesson.
- **No second provider built** — Brave/Exa are one-file additions behind the abstraction, not shipped here.

## Risk

- **Cost/abuse**: a search costs real credits. Mitigated by default-off + the conservative L1 clause +
  `max_results` cap; no automatic retry. The dispatcher `timeoutMs` + `ctx.abortSignal` bound a hung call.
- **Defection recurrence**: Python proved a directive alone is insufficient. Mitigated by shipping the
  commitment clause **and** the audit together (the audit gives the data to decide if forced-retry is ever
  needed).
- **Provider lock-in**: avoided by the `WebSearchProvider` abstraction from day one (native Anthropic
  `web_search` could even become a "provider" if Luna runs against first-party API).
- **Blocking I/O stall**: the `{kind:'progress'}` *正在查一下…* line keeps the UI honest during the 1–3 s call.

## Acceptance criteria

- [ ] `web_search` returns ranked `{title,url,snippet}` results; `summarize` emits the `[N] url` citation line.
- [ ] Every failure is a soft `err` (no throw escapes); abort/timeout ends promptly.
- [ ] `proactiveRisk:'safe'`; mounted only under `LUNA_WEB_SEARCH=1` via `withWebSearch`.
- [ ] The L1 commitment clause is in the cached contract; the intent-no-call audit fires on a defecting
      turn and stays silent on a calling turn (tested).
- [ ] `tsc` clean + `bun test` green; ships **off**.

## Notes for v0.18.1 (don't foreclose)

- Keep `provider.ts`'s `fetch`+env+AbortSignal client **generic enough to reuse** for `web_fetch` (same
  error-slice discipline, same `ctx.abortSignal` threading).
- The `tools/web/` module is the home for `web_fetch.ts` + `safeFetch.ts` + `extract.ts` next.
- `summarize`'s citation format should generalize to fetched-page citations (a stable `[id] Title — url`).
- Leave the `<untrusted_content>` decision to v0.18.1, but note: **search snippets are also untrusted web
  text** — v0.18.2's standing system rule must cover both tools, so don't bake "search is trusted" anywhere.
