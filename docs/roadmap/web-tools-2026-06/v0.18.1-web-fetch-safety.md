# v0.18.1 — web_fetch + SSRF / extraction safety core

> **Status: PLANNED.** Initiative: Web tools (Order 11, version 2/3). Risk: **High**.
> Depends: v0.18.0 (shared `tools/web/` infra + `fetch`/env client). Flag: `LUNA_WEB_FETCH` (default off).

## Goal

Give Luna the half Python never had: **read one web page, safely.** `web_fetch(url)` resolves the URL
through an **SSRF guard**, fetches with hard size/time caps, runs **boilerplate extraction**
(Readability → Turndown → markdown), and returns the cleaned text **wrapped as untrusted content**. This
is the security keystone of the initiative — the version where the URL-fetcher's attack surface is closed
*inside the tool* (LD #10), exactly as `shell`'s deny-regex closed its surface in Initiative 8. Ships off.

## What ships

**`packages/server/src/tools/web/safeFetch.ts` — the SSRF guard (the keystone):**

- `assertPublicUrl(rawUrl): { ok: true; url: URL } | { ok: false; reason: string }` — the URL analogue of
  `resolveInWorkspace` (`workspace.ts:195-238`):
  1. Parse + **canonicalize**: reject non-`http:`/`https:` schemes (`file:`, `ftp:`, `gopher:`, `data:`),
     reject embedded credentials (`user:pass@`), normalize the host (decode percent-encoding, collapse
     decimal/hex/octal IP forms, strip trailing dot), enforce `url.length ≤ 2048`.
  2. **Resolve DNS** (`node:dns/promises lookup`, all A/AAAA) and validate **every resolved IP** against a
     deny-list: loopback `127/8`+`::1`, RFC1918 `10/8`+`172.16/12`+`192.168/16`, link-local `169.254/16`
     (incl. cloud metadata `169.254.169.254`) + `fe80::/10`, ULA `fc00::/7`, `0.0.0.0/8`, broadcast,
     IPv4-mapped-IPv6 (`::ffff:…`), reserved/multicast. Any hit ⇒ `{ok:false}`.
- `safeFetch(url, {maxBytes, timeoutMs, signal}): Promise<{status, contentType, body}>`:
  - `redirect: 'manual'` — **never auto-follow**. On a 3xx, re-run `assertPublicUrl` on the `Location` and
    re-fetch manually, bounded to ≤5 hops (closes the public→`169.254.169.254` redirect bypass).
  - **DNS-rebinding pin**: resolve once, validate, and connect to the **validated IP** (Host header
    preserved) — or, minimally, re-resolve+re-check immediately before connect and treat a change as a
    failure. (Implementation note for build: Bun `fetch` has no IP-pin hook; use a `lookup`-checked
    `node:https` agent or re-validate-at-connect — decide at build, but the *test* asserts a rebinding
    host is rejected.)
  - Abort on `maxBytes` exceeded (stream + count; default `LUNA_WEB_FETCH_MAX_BYTES` = 3 MB) and on
    `timeoutMs` (default 15000, threaded from `ctx.abortSignal`). `content-type` must be `text/html` or
    `text/plain` (PDF/JS-rendered deferred — see scope).

**`packages/server/src/tools/web/extract.ts` — boilerplate stripping:**

- `extractMarkdown(html, url): { title: string; markdown: string }` — parse with **`linkedom`** (lighter
  than `jsdom`), run **`@mozilla/readability`** to isolate the main article, convert with **`turndown`** to
  markdown, collapse whitespace, truncate to `LUNA_WEB_FETCH_MAX_CHARS` (default 12000) with a `…[truncated]`
  marker. Empty/failed extraction falls back to a stripped-tags text slice (never throws).

**`packages/server/src/tools/web/web_fetch.ts` — the tool:**

- `input`: `z.object({ url: z.string().url(), max_chars: z.number().int().min(200).max(50000).optional() })`.
- `output`: `z.object({ url, final_url, title, content, truncated, fetched_ms })` — `content` is the
  extracted markdown, already wrapped (see below).
- `concurrency:'safe-parallel'`, `proactiveRisk:'safe'` (a read; see Open Q #2 — flip to `'surface'` if
  build review prefers conservative), `timeoutMs: LUNA_WEB_FETCH_TIMEOUT_MS ?? 15000`.
- `summarize(out)` → `` `read [title] (final_url) — N chars` ``.
- `execute`: `{kind:'progress', note:'正在读这一页…'}` → `assertPublicUrl` (fail ⇒ `err code:'blocked_url'`,
  recoverable) → `safeFetch` → `extractMarkdown` → `{kind:'ok'}`; **every** failure is a soft `err`.
- **Untrusted-content envelope**: `content` is returned wrapped —
  `` `<untrusted_content source="${final_url}">\n${markdown}\n</untrusted_content>` `` — so the delimiter is
  present from the first fetch (the *standing system rule* that tells the model what the delimiter means
  lands in v0.18.2, but the structural marker is intrinsic here).

**Registry + boot:** `webFetchEnabled()` + `withWebFetch(reg)` (mirrors v0.18.0), wired in `main.ts`.

**Dependencies added** (`package.json`): `@mozilla/readability`, `linkedom`, `turndown` — pure-Node, no
native build. (Open Q #6.)

**Env:** `LUNA_WEB_FETCH` (master gate, **off**), `LUNA_WEB_FETCH_TIMEOUT_MS` (`15000`),
`LUNA_WEB_FETCH_MAX_BYTES` (`3000000`), `LUNA_WEB_FETCH_MAX_CHARS` (`12000`).

## Tests (`bun test`)

- **SSRF deny-list (table-driven, the keystone)**: `assertPublicUrl` rejects `http://127.0.0.1`,
  `http://localhost`, `http://10.0.0.5`, `http://192.168.1.1`, `http://169.254.169.254` (metadata),
  `http://[::1]`, `http://[fc00::1]`, an octal/decimal-encoded private IP, `file:///etc/passwd`,
  `ftp://…`, a `user:pass@` URL, and a >2048-char URL — each with a specific `reason`. It **accepts** a
  normal public host (mock the resolver to a public IP).
- **Redirect re-validation**: a fetch whose first hop 302-redirects to `http://169.254.169.254` is
  rejected on the redirect (mock fetch + mock resolver), not followed.
- **DNS-rebinding**: a host that resolves to a public IP on the guard check but to a private IP at connect
  is rejected (mock a changing resolver).
- **Caps**: a body over `maxBytes` aborts mid-stream → `err`, no OOM; a slow fetch hits `timeoutMs` →
  abort `err`.
- **Extraction**: a fixture HTML page (article + nav + footer + script) → `extractMarkdown` returns the
  article body as markdown, nav/footer/script gone, title captured; an empty/garbage page → safe fallback,
  no throw; output truncates at `max_chars` with the marker.
- **Untrusted envelope**: `output.content` is wrapped in `<untrusted_content source=…>…</untrusted_content>`.
- **Soft-fail + proactiveRisk + gating** as in v0.18.0.

## What this version explicitly does NOT include

- **No JS-rendering / headless browser** and **no PDF** — `text/html` + `text/plain` only (matches
  Anthropic's native fetch). A Jina `r.reader` fallback + PDF text extraction are deferred (Open Q #4).
- **No standing injection system-rule yet** — only the structural `<untrusted_content>` delimiter ships
  here; the system-prompt rule that instructs the model how to treat it is v0.18.2.
- **No citation UI / L2 surfacing** → v0.18.2.
- **No fetch cache** (every call hits the network) → optional in v0.18.2 (migration `0012`).
- **No default-on** — `LUNA_WEB_FETCH=0` ships.

## Risk

- **SSRF is the whole ballgame** — a miss exposes the user's LAN + cloud metadata. Mitigated by the
  table-driven guard, **redirect re-validation**, the rebinding check, and exhaustive per-class tests; the
  guard is in the evaluator-firewall list philosophy (the agent must never weaken its own safety code —
  `safeFetch.ts` should join `workspace.ts`'s firewall set so a future `propose_self_edit` can't rewrite it).
- **Extraction fragility / ReDoS** — arbitrary HTML is hostile input. Mitigated by `linkedom` (no full
  browser), size caps **before** parsing, a never-throw fallback, and the dispatcher timeout.
- **Untrusted content in context** — the delimiter ships now; the behavioral rule + the read/write trust
  boundary land in v0.18.2. Until then `LUNA_WEB_FETCH` stays off, so no unguarded content reaches a real turn.
- **New deps** — small, pure-Node, widely used; pinned + `bun install` reproducible.

## Acceptance criteria

- [ ] `assertPublicUrl` blocks every private/loopback/link-local/metadata/ULA/encoded-IP/non-`http(s)`/
      credential/over-long case, and follows-then-re-validates redirects; rebinding rejected — all tested.
- [ ] `safeFetch` enforces byte + time caps and manual redirect; `extractMarkdown` returns clean article
      markdown with boilerplate gone, bounded + never-throwing.
- [ ] `web_fetch` returns `<untrusted_content>`-wrapped markdown + `final_url` + `title`; all failures soft.
- [ ] `safeFetch.ts` is added to the evaluator-firewall path (self-edit can't rewrite the SSRF guard).
- [ ] `tsc` clean + `bun test` green; ships **off**.

## Notes for v0.18.2 (don't foreclose)

- `output` carries `final_url` + `title` — the raw material for citation surfacing + L2 persistence.
- The `<untrusted_content source=…>` envelope is the hook the standing system rule will reference by name.
- Keep `safeFetch` cache-agnostic so v0.18.2 can wrap it with a TTL cache (migration `0012`) without
  touching the guard.
- Both `web_search` snippets **and** `web_fetch` bodies are untrusted — v0.18.2's system rule covers both.
