## Goal

Close the three items the PR #6 review **deferred** rather than papered over, so `web_fetch` can go
back to default-on honestly: (1) a **verified DNS-pinned fetch** that actually connects to the IP
`safeFetch` validated (closing the rebinding TOCTOU Bun's `fetch` left open), (2) the **live
latency/token measurement** the v0.18.2 "measure-before-flip" criterion required, and (3) the
**citation UI polish** (clickable + scheme-validated chips that survive a reload).

## Why this is a follow-up, not a v0.18.2 fix

The review found `safeFetch`'s rebinding defense is a window-narrowing re-check, not a pin ‚Äî Bun's
`fetch` exposes no IP-pin hook, and a from-scratch pinned HTTP client could not be verified against real
HTTPS in the review environment. Shipping an unverifiable security rewrite under review pressure would
violate "don't accept looks-right," so `web_fetch` was reverted to **opt-in (default OFF)** and the real
pin tracked here. `web_search` (no SSRF surface) stayed default-on.

## What ships

1. **Pinned-lookup fetch** (`safeFetch.ts`): resolve the host **once**, validate every IP, and force the
   connection onto a validated IP ‚Äî via a `node:undici` `Agent`/`Dispatcher` with a custom `connect`/
   `lookup` that returns only the pre-validated address (preserving the `Host` header + TLS SNI =
   hostname so HTTPS cert validation still uses the name), **or** a `node:https`/`node:http` `Agent` with a
   pinned `lookup`. `assertPublicUrl` returns the resolved IPs so a single resolution drives both the
   check and the connect (no TOCTOU). Re-pin on each redirect hop.
2. **Real rebinding test**: drive the resolution the *connect path actually uses* (assert the pinned IP),
   so the test fails on the pre-fix re-check-only code.
3. **Measurement**: a sweep with a real `LUNA_WEB_SEARCH_API_KEY` ‚Äî median + p90 latency and added input
   tokens for a search turn and a fetched turn; record in DEVELOPMENT.md. Then **re-flip `LUNA_WEB_FETCH`
   default ON**.
4. **Citation polish** (`packages/web` + `packages/protocol`): render `source` chips as real `<a>`
   anchors (`target=_blank rel=noopener noreferrer`) with `new URL()` **scheme-validation** (http/https
   only, else plain text) and tighten `Citation.url` to `z.string().url()` at the gathering point (not the
   wire boundary ‚Äî avoid breaking `turn.result` on an odd provider URL); **persist citations across
   reload** via a nullable `citations_json` L2 column replayed on connect (migration `0013`).

## Tests

- Pinned fetch connects to the validated IP; a host that resolves public-at-check / private-at-connect is
  rejected because the connect uses the pinned IP (not a re-resolve the socket ignores).
- HTTPS to a pinned IP still validates the cert against the hostname (SNI preserved).
- Citation chip renders as a clickable anchor; a `javascript:`/`data:` url falls back to text; citations
  re-render after a simulated reload (L2 replay).

## What this version explicitly does NOT include

- No JS-rendering / PDF / headless browser (still deferred).
- No change to `web_search` (already correct + default-on).
