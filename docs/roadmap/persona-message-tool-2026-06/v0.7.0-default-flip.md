## Design

**The A/B comparison (the reason the flag existed):** before flipping, run the same ~10-turn
conversation script through both modes against the real model and record in the DEVELOPMENT
entry:

- humanity compliance (violations per turn: text mode = prompt-only vs message mode = schema);
- empty-turn rate (text mode baseline vs message mode with guard);
- top-level-text leak rate in message mode (count from traces);
- first-visible-latency (`firstTokenMs` vs first `tool.progress` delta);
- subjective voice quality (persona adherence — manual judgment, recorded honestly).

This is a **dev-scale comparison, not a statistical claim** — its job is to catch "message
mode is obviously worse somehow" before it becomes the default, and to leave a written
baseline for Initiative 4 (reasoning rails) to compare against.

**Flip mechanics:**

- Default `LUNA_MESSAGE_TOOL` → `'1'` in the env read (single site); `=0` keeps the v0.3
  text path alive (cheap — both modes already share one runTurn; the flag stays supported
  through Initiative 6, after which removal is reconsidered).
- Dev chat page defaults to message-mode rendering.
- REWRITE_CONTEXT LD #9: introduction milestone marked **landed**, with the A1–A6 amendments
  folded into the detail block (the sketch table updates to the shipped schema).
- Roadmap master README: Initiative 3 row → ✅ shipped; Open Q #9 (model-callable `recall`
  tool) gets a one-line status note: now unblocked, queued for Initiative 4 planning.

**Deliberately NOT in this version:** any frontend beyond the dev page (Initiative 6), TTS
wiring (`voice_params` flows through untouched), proactive/self-continuation (Initiative 5 —
`is_final` semantics will be consumed there), removal of the text path.
