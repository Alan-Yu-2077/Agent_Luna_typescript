# v0.25.2 — Model glide (center↔right) + initiative close

> **Status: PLANNED.** Initiative: Collapsible companion UI (Order 18, version 3/3). Risk: **Medium**.
> Depends: v0.25.1 (needs the collapse/expand transition hook). Flag: none new (rides the collapse
> state; `.reduce-motion` gates the animation). **`packages/web` only.**

## Goal

Make Luna **glide** — smoothly slide to screen center when the window collapses and back to the right
region when it expands — instead of snapping. A pure client-side, vanilla eased tween on the pixi
ticker, on a dedicated mode-offset that never disturbs the user's saved drag. Close the initiative.

## What ships

**Modify — `packages/web/src/live2d/faceVm.ts`**: export the existing file-private `easeInOutSine` +
`lerp` + `clamp01` (`faceVm.ts:643-648`) — or extract them into a small shared `math` util — so the
glide reuses the exact easing Luna's other motion uses (`easeInOutSine` == CSS `ease-in-out`, keeping
the model glide visually consistent with the v0.25.1 CSS morph).

**Modify — `packages/web/src/live2d/modelDriver.ts`**: add a **dedicated mode-offset** channel
(separate from the persisted drag `off.dx/off.dy`), e.g. `setModeOffset(mx, my)` that composes with
`baseX/baseY + drag offset + mode offset` in the single `model.position.set`. This keeps the glide from
clobbering or fighting the user's saved drag (`luna:live2d:pos`) and the ±50%-host clamp
(`pixiLive2DSink.ts:176`).

**Modify — `packages/web/src/live2d/pixiLive2DSink.ts`** (the glide):
- Add a glide driver that eases the mode-offset from its current value to a target over
  ~`LUNA_GLIDE_MS` (default 450–600ms), stepped on the existing per-frame beat
  (`internal.on('beforeModelUpdate', …)`, `pixiLive2DSink.ts:147`) — **no animation library**. Targets:
  - **collapse → center:** the mode-offset that places the model at true screen center once
    `.model-stage` has grown to full width. Because `fit()` already centers within the host and the
    host becomes full-width on collapse, the mode-offset for "center" is near-zero after a re-fit; the
    glide covers the visual travel while the CSS morph resizes the region (drive the model motion in
    **one** system — the ticker — so it never disagrees with a `fit()` snap; call `fit()` once at rest,
    not mid-glide).
  - **expand → right:** ease the mode-offset back so the model reads as centered in the right region.
- Expose a glide method on the `Live2DSink` interface (`sinks.ts:15`) as an **optional** method
  (mirroring `setGazeFollow?`/`setIdleProfile?`), e.g. `glideTo?(region: 'center' | 'right')`; the
  console/no-Live2D sink no-ops it (`sinks.ts:44`). Gaze auto-follows since it re-reads `model.x` each
  pointermove (`pixiLive2DSink.ts:208`) — no extra wiring.
- **Reduce-motion:** when `.reduce-motion` is active, snap (single re-fit) instead of easing.

**Modify — `packages/web/src/app.ts`**: on the v0.25.1 collapse/expand hook, call
`sink.glideTo?('center' | 'right')` (guarded — the sink may be the console fallback when WebGL is
unavailable).

## Tests (`bun test packages/web/`)

- `modelDriver.test.ts`: `setModeOffset` composes with base + drag offset (position = base + drag +
  mode); it does **not** mutate or persist the drag offset (`luna:live2d:pos` untouched).
- Glide easing (unit, inject a clock): the mode-offset moves from start to target following
  `easeInOutSine` over `LUNA_GLIDE_MS` and settles exactly at the target; a reduce-motion path snaps in
  one step.
- Regression: after a glide, a subsequent `fit()` (window resize) re-centers correctly without a jump,
  and the user's saved drag offset is preserved.

## What this version explicitly does NOT include

- Any new easing/animation dependency (VANILLA constraint — reuse `easeInOutSine`/`lerp`).
- Any `packages/server` / `packages/protocol` change.
- Zoom/scale changes on collapse (out of scope; the model keeps its persisted zoom).

## Risk

- **Medium — two geometry owners** (CSS region resize + pixi model position). Mitigation: drive the
  model motion **only** on the ticker (not a competing CSS transform of the canvas host); call `fit()`
  at rest, not mid-glide, so the canvas resize and the model tween don't disagree (recon risk).
- **Clobbering the user's drag.** Mitigation: the dedicated mode-offset channel is separate from
  `off.dx/off.dy`; a test asserts the persisted drag is preserved across a glide.
- **DPR / coordinate mismatch.** Mitigation: author the glide in host/model-position pixels (autoDensity
  handles DPR); do not multiply by `devicePixelRatio`.

## Acceptance criteria
