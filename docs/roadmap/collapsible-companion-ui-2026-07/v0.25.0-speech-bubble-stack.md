# v0.25.0 — Beside-model speech-bubble stack

> **Status: PLANNED.** Initiative: Collapsible companion UI (Order 18, version 1/3). Risk: **High**.
> Depends: nothing. Flag: `luna:bubble-stack` (localStorage; default off — the stack renders only when
> enabled, so it is verified standalone before the collapse UI in v0.25.1 makes it the collapsed-mode
> renderer). **`packages/web` only.**

## Goal

Build the riskiest, most novel piece first, in isolation from any layout change: a **speech-bubble
stack that renders beside the Live2D model**, fed by the *same* `ServerEvent`-driven
`open/append/finalize` calls the chat window already consumes. A new `SpeechStackView` implements the
existing `BubbleView` interface; a `RouterBubbleView` sits between the controller and the concrete
views and fans calls to the window view and/or the stack based on a **live `collapsed` flag**
(default: window only). This lands + proves the stack mechanics (newest-at-bottom ordering, push-up,
speech-gated ~10s life, gradual fade, barge-in clear, proactive lifecycle) before v0.25.1 wires the
collapse UX to it.

## What ships

**New file — `packages/web/src/ui/speechStackView.ts`** (`class SpeechStackView implements BubbleView`):
- Mounts a `.speech-stack` container as an absolutely-positioned child of `.model-stage` (already
  `position:relative`, same containing block as `.mood-pip`/`.dream-btn` — `layout.ts:156-168`),
  anchored bottom, bubbles stacking upward (newest at the bottom, older pushed up).
- Implements `open/append/finalize/discard/chip/setThinking` (+ `renderHistory` as a **no-op** — a
  reconnect must not replay history as a bubble avalanche):
  - `open(id)`/`append(id, text)` — a live streaming bubble keyed by `id` (mirrors
    `cuteBubbleView.ts:85-101` Map pattern) **only if streaming into bubbles is enabled**; otherwise the
    stack waits for `finalize` (the validated per-message text) to spawn a bubble.
  - `finalize(id, text)` — commit the bubble with final text; start its life timer.
  - `discard(id)` — remove a preview without committing (validation failure / stutter must **not**
    leave a bubble).
  - `chip(kind, …)` — proactive/tool/dream chips: render minimally or drop (design: a small proactive
    seed chip may show; tool/source chips can be omitted in the stack — the window still has them).
- **Life + fade:** each committed bubble gets a ~`LUNA_BUBBLE_TTL_MS` (default 10_000) life that
  **starts at speech-begin**, not emit-time (see the audio wiring), then a CSS opacity/translateY
  fade-out (reuse the `dream-drift`/mood-pip fade idiom, `theme.css:197,212`) and DOM removal.
- **Overflow cap:** a max visible count (e.g. `LUNA_BUBBLE_MAX`, default 4) — the oldest fast-fades
  when exceeded, so a rapid multi-message turn can't grow an unbounded tower.

**New file — `packages/web/src/ui/routerBubbleView.ts`** (`class RouterBubbleView implements BubbleView`):
- Holds the window view (`CuteBubbleView`) + the `SpeechStackView` + a `collapsed: () => boolean`
  getter. Every `BubbleView` method forwards to the window view **always** (so the scrollback stays
  complete and expanding shows full history) and **additionally** to the stack when `collapsed()` is
  true. A live flag (not a swapped reference) so in-flight bubbles never strand (recon risk).
- This is the single place the "collapsed ⇒ also-bubbles" routing lives; the controller is unchanged
  (`controller.ts` still sees one `BubbleView`).

**Modify — `packages/web/src/app.ts`** (wiring only):
- Construct `SpeechStackView(refs.modelStage)` + wrap `CuteBubbleView` and it in a `RouterBubbleView`
  with a `collapsed` getter reading a client flag (in v0.25.0 the flag is driven only by the
  `luna:bubble-stack` localStorage toggle for standalone testing; v0.25.1 repoints it at the real
  collapse state). Pass the router as `deps.view` to `createController`.
- **Speech-gated life:** where the controller finalizes + speaks, the life clock needs the speech-begin
  signal. `audio.speak()` already accepts an unused `onStart` and resolves on end (`sinks.ts:36-42`).
  Thread a per-bubble begin/end signal to the stack **without changing the controller's wire seam** —
  e.g. the stack subscribes to a small client-side speech-event emitter that `app.ts` feeds from the
  `onStart`/resolve of the `speak()` call it already brokers. (No protocol/server change; `onStart` is
  a client callback.)
- **Barge-in:** on `turn.started` the controller calls `audio.stop()` (`controller.ts:64-76`) but does
  not clear bubbles; add a client-side hook so the stack fast-fades on a new user turn. Proactive turns
  do **not** emit `turn.started` — the stack keys its own lifecycle off `proactive.started/finished`
  where needed.

**Modify — `packages/web/src/ui/theme.css`**: `.speech-stack` + `.speech-bubble` styles (bottom-anchored
column, upward stack, tail toward the model), a `.fading` opacity/translateY transition, and matching
`.reduce-motion` / `@media (prefers-reduced-motion)` overrides (instant remove, no fade).

**Optionally export — `packages/web/src/live2d/faceVm.ts`**: not needed this version (no glide yet).

## Tests (`bun test packages/web/`)

- `speechStackView.test.ts` (jsdom-style host element):
  - `finalize` commits a bubble with the final text; a second `finalize` stacks it **below** the first
    (newest at bottom) and the first is still present (pushed up).
  - `discard` after `open` leaves **no** bubble (stutter/validation failure).
  - a committed bubble is removed after its life elapses (inject a clock / fake timer); `.fading` class
    applied before removal.
  - overflow: exceeding `LUNA_BUBBLE_MAX` fast-fades the oldest.
  - `renderHistory` is a no-op (no bubbles created).
- `routerBubbleView.test.ts`: with `collapsed()==false`, calls reach only the window view; with
  `collapsed()==true`, calls reach **both**; flipping the flag mid-stream doesn't strand an open bubble.
- `controller.test.ts` (regression): the controller still drives a single `BubbleView` (the router) and
  its existing assertions (finalize text, dedup, thinking) pass unchanged.

## What this version explicitly does NOT include

- **The collapse/expand UI + window morph** → v0.25.1 (this version's stack is exercised via the
  `luna:bubble-stack` toggle, not a real collapse).
- **The model glide** → v0.25.2.
- Any change to `controller.ts` logic, `packages/server`, or `packages/protocol`.

## Risk

- **High — new render surface + timing.** Mitigation: default-off flag; standalone tests with an
  injected clock; the stack is a pure additive view fed by the existing seam (no controller change).
- **TTL vs serialized audio** (recon): a blind 10s desyncs on multi-message turns. Mitigation: start the
  life at **speech-begin** via the existing `onStart`; fast-fade on barge-in.
- **Bubble spawned on a discarded preview.** Mitigation: commit only on `finalize`, never `append`;
  a test asserts `discard` leaves nothing.
- **Drift.** Mitigation: route through the `BubbleView` seam via `RouterBubbleView`, never a parallel
  `onEvent` listener.

## Acceptance criteria

- [ ] With `luna:bubble-stack` on, a reply produces a beside-model bubble; a second reply stacks below
      it and pushes the first up; each fades ~10s after she **speaks** it.
- [ ] A stutter/validation-failed preview leaves no bubble; a new user turn fast-fades the stack.
- [ ] `renderHistory` creates no bubbles; the window view is byte-unchanged when the flag is off.
- [ ] Proactive replies (no `turn.started`) still stack + expire correctly.
- [ ] `packages/server` + `packages/protocol` untouched; `bun test` + `tsc --noEmit` (web) green.

## Notes for v0.25.1 (don't foreclose)

- Keep the `collapsed` flag a **getter** the router reads live, so v0.25.1 can repoint it at the real
  collapse state without touching the router or the stack.
- Keep the stack's mount (`.model-stage` child) independent of the window's visibility so v0.25.1 can
  hide the window without disturbing the stack.
