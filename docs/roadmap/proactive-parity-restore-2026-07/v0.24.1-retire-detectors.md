# v0.24.1 — Flip default + retire the detector registry

> **Status: PLANNED.** Initiative: Proactive parity restore (Order 17, version 2/3). Risk: **Medium**.
> Depends: v0.24.0 (the ladder must exist and be live-verified behind the flag). Flag:
> `LUNA_PROACTIVE_LADDER` **default flips to on**; `=0` becomes the escape hatch back to nothing (the
> detector path is deleted, so `=0` means "no ladder" — the anti-spam rail + turn still stand).

## Goal

Make the silence ladder **the** proactive decision — flip `LUNA_PROACTIVE_LADDER` on by default,
then delete the detector registry and all scheduled-slot machinery that the ladder replaces. After
this version the proactive path is purely Alan's Python design: silence in, restraint-graded
scenario out. This is the version that makes the reversal real, so it also lands the **LD #15
amendment** in `REWRITE_CONTEXT.md`.

## What ships

**Default flip:**
- `LUNA_PROACTIVE_LADDER` defaults to **on** in `fire.ts` (the seam now selects the ladder unless
  explicitly `=0`). Remove the detector branch from `maybeFireProactive`.

**Delete — `packages/server/src/proactive/detectors.ts`** (entire file) and its test
`detectors.test.ts`:
- `afterNight`, `scheduledWindow`, `weatherShift`, `openThreadAged`, `promisedFollowThrough`, the
  `REGISTRY`, `evaluateDetectors`, `resetWeatherBaselineForTests`, the weather-bucket helpers, the
  promise-regex machinery — all gone.
- Update `fire.ts`: drop the `detectProactive` seam's `evaluateDetectors` default and
  `setProactiveDetectorForTests` (or repoint the test seam at `evaluateLadder`); `scheduler.ts`
  re-exports `setProactiveDetectorForTests` (`scheduler.ts:12`) — update or remove.

**Delete slot machinery — `packages/server/src/proactive/cadence.ts`:**
- Remove `scheduledSlots`, `isSlotConsumed`, `markSlotConsumed` (`cadence.ts:120-139`) and the
  `markSlotConsumed` call in `fire.ts:143`.
- Remove `slotsUsed`/`slotsDate` from the `Cadence` type, `DEFAULT_CADENCE`, `loadCadence`,
  `saveCadence`, and the `Row` shape.

**Migration — `packages/server/src/memory/migrations/`:**
- A new versioned migration retiring the `proactive_slots_used` / `proactive_slots_date` columns.
  SQLite can't `DROP COLUMN` cleanly on old versions → follow the repo's established table-rebuild
  migration pattern (the same discipline used for prior schema changes), or leave the columns
  physically present but **stop reading/writing them** and document that in the migration note. Choose
  at build time per the current SQLite/Bun version; either way `loadCadence`/`saveCadence` no longer
  reference them.

**Env cleanup:**
- `LUNA_PROACTIVE_SLOTS` is dead — remove its handling; note removal in `.env.example` if present.
  (Operationally: Alan's `.env` currently sets `LUNA_PROACTIVE_SLOTS=11,20,21,22` — those temp test
  slots become no-ops and should be deleted from `.env` on deploy; the ladder needs no slots.)

**Docs — `docs/REWRITE_CONTEXT.md` (the LD #15 amendment):**
- Append to LD #15 a dated amendment (2026-07-01): the escalation ladder (ambient → idle_nudge →
  renudge → leave_message, backoff, DORMANT-recovery) is **restored as the proactive decision layer**
  at Alan's direction, reversing the 2026-06-13 Hermes-paradigm framing for the *wake decision*.
  Explicitly preserve what still stands: the full-tool, message-optional, **safety-gated** proactive
  turn; the `LUNA_PROACTIVE` kill switch; the `.unref()`'d heartbeat; and the dropped
  outbox/cursor/TTL/SSE-replay delivery layer (still dropped — the single WS makes it moot).

## Risk

- **Medium — a live migration on Alan's running DB.** His stable instance persists cadence in
  `luna.sqlite` (already at migration 13 with the slot columns). Mitigation: the table-rebuild
  migration is atomic; `loadCadence` tolerates a row with or without the columns during the rollout;
  the ladder itself needs none of the slot state, so a failed/partial migration degrades to "no
  slots" (already the target). Verify on a DB copy before Alan restarts.
- **Removing the last non-ladder wake path.** After the flip, `LUNA_PROACTIVE_LADDER=0` means she
  never *proactively* opens (rail + turn still exist for continuation/dream). That is the intended
  "off" — documented so it isn't mistaken for a regression.
- **Losing the net-new triggers.** weather-shift / promise-follow-through / open-thread / after-a-night
  greeting are deleted (Alan accepted this for the clean ladder). Recorded in the README divergences so
  a future initiative can re-introduce any of them as *opportunistic content triggers layered on the
  ladder* if Alan later wants them back — the ladder seam leaves room.
