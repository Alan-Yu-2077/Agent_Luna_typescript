# v0.24.2 — Style self-tuning layer (`set_proactive_style`)

> **Status: PLANNED.** Initiative: Proactive parity restore (Order 17, version 3/3). Risk: **Low**.
> Depends: v0.24.1 (the ladder is the default and reads its cadence through one resolution point).
> Flag: `LUNA_PROACTIVE_STYLE` (default **on** — additive, bounded, and safe; `=0` pins
> `balanced`). Initiative close.

## Goal

Complete the faithful replica with the piece that makes the proactive voice *hers*: a **self-tuning
style layer**. Luna can move her own **activeness** (how eagerly she initiates) and record **voice
notes / exemplars** for her openers — but only within operator-owned **floor/ceiling** rails she
cannot exceed. This ports Python's two-layer split: an operator-owned mechanical config (the safety
kernel) + a Luna-writable style layer scaled *inside* it (`proactive_config.py:84-108`,
`proactive_tools.py`).

## What ships

**New file — `packages/server/src/proactive/style.ts`** (the two-layer resolver, ported from
`proactive_config.py`):
- `type Activeness = 'aloof' | 'balanced' | 'clingy'`.
- `type ProactiveStyle = { activeness: Activeness; voiceNotes: string; exemplars: Record<string, string[]> }`.
- `LEVEL_MULT` — the activeness → {cooldown, prob, quota} multipliers (`proactive_config.py:36-40`:
  aloof `{1.8, 0.45, 0.4}`, balanced `{1,1,1}`, clingy `{0.6, 1.35, 1.6}`).
- `resolveEffectiveCadence(style)` — the safety kernel (`proactive_config.py:84-108`): apply the
  lever, then **clamp inside** operator floor/ceiling. Returns the effective `minIntervalMs`,
  `renudgeBaseMs`, `dailyQuota` (clamped `[floor, ceiling]`), `nudgeProb`, `ambientProb`. The ladder
  (v0.24.0) reads its knobs from here instead of raw env, so the lever scales one resolution point.
- `loadStyle()` / `saveStyle(partial)` — persistence. **Reuse the existing SQLite store** (a small
  `proactive_style` table or a JSON blob on the session/settings row — pick the lightest that fits the
  memory schema; no new process-global file, unlike Python's `proactive_style.json`). Atomic write via
  the repo's existing DB write path.

**New tool — `packages/server/src/tools/proactiveStyle.ts`** via `defineTool` (ported from
`SetProactiveStyleTool`, `proactive_tools.py`):
- Name `set_proactive_style`; description makes clear it shapes *her own* outreach style and **cannot
  exceed** the system's fixed frequency bounds.
- Params (all optional): `activeness` (enum `aloof|balanced|clingy`), `voice_notes` (string).
  `exemplars` optional/deferred if the schema gets noisy.
- `concurrency`: `safe-parallel` (writes only her own style row); `proactiveRisk: 'safe'` (it's a
  reversible self-setting — she may call it silently in a proactive turn); short `summarize`.
- Returns `{ ok, summary: "proactive style updated (activeness=…)" }`; validates the enum
  (`bad_activeness` error) and the "provide at least one field" guard (`missing_args`).
- Register in the tool registry so it reaches both reactive and proactive turns.

**Modify — `packages/server/src/proactive/ladder.ts` + `cadence.ts`:**
- The ladder + rail resolve `minInterval` / `dailyQuota` / `renudgeBase` / `ambientProb` / `nudgeProb`
  through `resolveEffectiveCadence(loadStyle())` (guarded by `LUNA_PROACTIVE_STYLE`; off → `balanced`,
  i.e. today's raw defaults). Operator env knobs remain the floor/ceiling inputs.

**Modify — `proactiveTurn.ts`:** thread `voiceNotes` + `exemplars[scenario]` into the framing (the
"你给自己定的主动风格" + "你自己收藏的这类开场参考" lines, `proactive.py:418-426`), so her recorded
voice actually colors the opener.

## Tests

- `style.test.ts`: `resolveEffectiveCadence` — `clingy` shortens cooldown + raises quota but **never
  above ceiling**; `aloof` lengthens cooldown + lowers quota but **never below floor**; `balanced` ==
  raw config. Round-trip `loadStyle`/`saveStyle` (activeness + voiceNotes; bad activeness rejected).
- `proactiveStyle.test.ts` (tool): valid `activeness` persists + echoes; invalid → `bad_activeness`;
  empty args → `missing_args`; no field lets a stored value exceed the ceiling on the next resolve.
- `ladder.test.ts` (extend): with `clingy`, the ambient/nudge probabilities scale up but the fired
  cadence still respects the mechanical floor (a clamp test).
- `proactiveTurn.test.ts` (extend): a set `voice_notes` appears in the framing for a ladder scenario.
- `bun test` green; `tsc --noEmit` clean.

## What this version explicitly does NOT include

- Any change to the ladder's phase logic (v0.24.0) or the detector deletion (v0.24.1).
- A UI surface for style (it's a self-set tool + operator env; no `packages/web` change).
- Auto-learning exemplars from her own good openers (a possible future nicety, not this version).

## Risk

- **Low.** Additive + bounded: the lever only *scales inside* the operator floor/ceiling, so a
  runaway `clingy` can never breach the mechanical rail. `LUNA_PROACTIVE_STYLE=0` pins `balanced`
  (today's behavior). The only durable state is one small style row; a bad write degrades to
  `balanced` on read.
- Watch: the tool is model-writable, so its description must make the "cannot exceed system bounds"
  contract unambiguous — the clamp is enforced in `resolveEffectiveCadence`, never trusted from the
  tool input.
