# v0.24.0 — Silence-driven escalation ladder (core)

> **Status: PLANNED.** Initiative: Proactive parity restore (Order 17, version 1/3). Risk: **High**.
> Depends: nothing. Flag: `LUNA_PROACTIVE_LADDER` (default **off** — the detector path stays the
> default this version; the ladder runs only when the flag is on, so it is E2E-verified in isolation
> before it replaces anything).

## Goal

Restore Alan's Python silence-ladder as an alternate proactive **decision layer** that plugs into the
existing funnel at exactly one point — the `detectProactive` seam in `maybeFireProactive`
(`fire.ts:125`). When `LUNA_PROACTIVE_LADDER=1`, the wake decision is made by a phase machine driven
by `effective_gap` (silence since the last thing said in the channel, counting Luna's own outreach),
producing one of four restraint-graded scenarios — `ambient`, `idle_nudge`, `renudge`,
`leave_message` — each with its own USER-role framing and the full companion-opener constraint. The
model still has the last word (it may emit the silent sentinel), and it appends
`<proactive_meta>{follow_up}>` to steer the phase. The detector path is untouched and remains the
default; this version only *adds* the ladder behind the flag.

## What ships

**New file — `packages/server/src/proactive/ladder.ts`** (the ported phase machine; pure + clock-injectable):
- `export type ProactiveScenario = 'ambient' | 'idle_nudge' | 'renudge' | 'leave_message'`.
- `export function evaluateLadder(ctx: DetectorCtx): { scenario: ProactiveScenario; nextPhase: ProactivePhase; nudge: boolean } | null` — mirrors `proactive.py:282-320`:
  - Reads `ctx.cadence` (`phase`, `nudgesSent`, `lastProactiveMs`) + `ctx.session.lastUserMs` +
    `ctx.nowMs`. Computes `gap` (since last user msg) and `effective_gap = min(gap, nowMs −
    lastProactiveMs)` when she has spoken before (`proactive.py:277-281`).
  - **DORMANT recovery** (`proactive.py:288-295`): `phase==='dormant'` and `effective_gap ≥
    LUNA_PROACTIVE_DORMANT_RECOVERY_MS` (default 1h) → treat as `engaged`, `nudgesSent=0`.
  - `sleeping`/`dormant` (not recovered) → `null`.
  - `engaged`: `effective_gap ≥ idle_threshold` → `idle_nudge` (`nextPhase:'nudged'`); else
    `effective_gap ≥ ambient_min && rng() < ambient_prob` → `ambient` (phase unchanged); else `null`.
  - `idle_watch` (reserved for a future split; the TS rail collapses engaged→nudged directly like the
    hot path, so `idle_watch` maps through to the nudge branch — documented, not dead).
  - `nudged`: within `renudge_gap(nudgesSent, base)` (backoff `[1.0, 2.4, 6.0]`, `proactive.py:149-151`)
    → `null`; else `nudgesSent < max_nudges` → `renudge` (`nudge:true`); else → `leave_message`
    (`nextPhase:'dormant'`).
- `rng` injectable (default `Math.random`), like the detector tests inject determinism.
- Env knobs (all `num()`-guarded, read at call time — no restart to tune): `LUNA_PROACTIVE_IDLE_THRESHOLD_MS`
  (default 600_000, **deliberately > ambient_min so ambient is reachable** — see README "hard part"),
  `LUNA_PROACTIVE_AMBIENT_MIN_MS` (120_000), `LUNA_PROACTIVE_AMBIENT_PROB` (0.12),
  `LUNA_PROACTIVE_RENUDGE_BASE_MS` (300_000), `LUNA_PROACTIVE_MAX_NUDGES` (3),
  `LUNA_PROACTIVE_DORMANT_RECOVERY_MS` (3_600_000).

**Modify — `packages/server/src/proactive/cadence.ts`** (activate the dormant fields):
- `export function commitScenario(c: Cadence, scenario: ProactiveScenario, nowMs: number, followUp: boolean | null): Cadence` — the `commit_emission` port (`proactive.py:322-342`): bumps quota (reuses `commitProactive`'s date-roll logic), sets `phase`/`nudgesSent` from the scenario (`idle_nudge`/`renudge` → `nudged`, `nudgesSent+1`; `leave_message` → `dormant`; `ambient` → unchanged), and `followUp === false` forces `dormant`.
- **Wire `recordUserActivity`** (currently dead, `cadence.ts:143`): call it from the reactive path so a user turn resets `phase→engaged`, `nudgesSent→0` (Python `proactive.py:249-256`). Hook site: wherever a user turn finalizes cadence — if none exists, load→`recordUserActivity`→save in the WS user-message handler (server-internal, no wire change). Ladder-gated so the detector path is unaffected.

**Modify — `packages/server/src/proactive/proactiveTurn.ts`** (four scenario framings + full constraint):
- Add `COMPANION_OPENER_CONSTRAINT` — the full text ported from `proactive.py:367-378` (self-disclosure OR fresh topic; **vary every opener**; never 查岗). Supersede the partial inline version in the `spontaneous` DIRECTIVE.
- Add `SCENARIO_BODIES: Record<ProactiveScenario, string>` ported from `proactive.py:381-402` (the escalating-restraint bodies) + the `{seed}`-style continuity hint.

Absent → today's `intent` behavior (detector path unchanged).
- **Anti-repeat**: keep the last ≤5 spoken openers per session in memory (mirrors `recent_texts`, `proactive.py:204`) and inject "别重复你最近说过的开场" (`proactive.py:428-432`).

**New file — `packages/server/src/proactive/proactiveMeta.ts`** (the meta tail):

**Modify — `packages/server/src/proactive/fire.ts`** (the seam swap, flag-gated):

No slot marking on this path.

## Tests (`bun test packages/server/src/proactive/`)

- `ladder.test.ts` (inject `rng` + `nowMs` + a stub `Cadence`):
  - engaged + `effective_gap ≥ idle_threshold` → `idle_nudge`, `nextPhase:'nudged'`.
  - engaged + `ambient_min ≤ gap < idle_threshold` + `rng→0.0` → `ambient`; + `rng→0.99` → `null`.
  - `effective_gap` uses `min(userGap, sinceProactive)`: a recent self-outreach suppresses a nudge
    even when the *user* gap is large.
  - nudged within backoff window → `null`; past it, `nudgesSent<max` → `renudge`; at `max` →
    `leave_message`, `nextPhase:'dormant'`.
  - dormant + `gap ≥ recovery` → recovers (returns a scenario); + `gap < recovery` → `null`.
  - long absence handled by the rail/sleeping path → `null` (no ladder fire).
- `cadence.test.ts` (extend): `commitScenario` sets phase/nudges/quota per scenario; `followUp===false`
  forces `dormant`; quota still rolls on a new local day; `recordUserActivity` resets phase+nudges.

  a malformed/absent tail → `followUp:null`, text unchanged; the tag is stripped from *every* text

## Risk

- **High — activating a state machine that has only ever held defaults.** Mitigation: default-off
  flag; the ladder reads/writes the *existing* `phase`/`nudgesSent` columns (no migration); a restart
  mid-`nudged` rehydrates through `loadCadence` (already persisted). Isolated by the flag until a live
  smoke confirms the escalation sequence.
- **Double-gating the cooldown.** The rail's `min_interval` (`passesAntiSpam`) already fires before the
  ladder; the ladder must **not** re-check it (Python keeps them equal). Mitigation: `evaluateLadder`
  trusts the rail and layers only phase logic; a test asserts a nudge is not double-suppressed.
- **Meta tail leaking to the user.** If `parseProactiveMeta` runs after segmentation the tag reaches
  TTS/bubbles. Mitigation: strip in the emission path *before* segmentation, with a test on multi-block
  responses (the Python bug class, `proactive.py:467-482`).
- **Ambient unreachable** if `idle_threshold == ambient_min`. Mitigation: default `idle_threshold`
  (10m) > `ambient_min` (2m); documented; a test exercises the ambient band.
